## 定义

对集合最常见的三种遍历方式分别是：

- `iter()`：借用不可变元素
- `iter_mut()`：借用可变元素
- `into_iter()`：消费集合并产出拥有值

## 作用

- 用同一套命名表达三种不同所有权语义
- 让调用者显式选择“只读 / 可改 / 拿走”
- 避免模糊不清的遍历行为

## 原理

这三种方法返回的迭代器类型不同，所以后续闭包参数的类型和可做的事情也不同。看起来都在“遍历”，实际上是在走三条不同所有权路径。

## 最小示例

```rust
let borrowed_sum: i32 = numbers.iter().sum();

for score in &mut scores {
    *score += 5;
}

let consumed: Vec<i32> = scores.into_iter().collect();
```

## 注意点

- `iter` 不会拿走集合
- `iter_mut` 可以改元素，但仍不拿走集合本体
- `into_iter` 后原集合通常不能再用

## 常见错误

- 看见名字像，就以为行为差不多
- 想保留集合却用了 `into_iter`
- 在 `iter()` 闭包里试图按值拿走元素

## 我的理解

这三个方法是“所有权视角下的遍历 API”。真正学会它们，很多 borrow checker 报错会提前在脑子里消失。
