## 一分钟结论

- **结合性**决定“同优先级运算符如何分组”
- **运算符重载**决定“某个类型能不能使用某个运算符”
- Rust 的运算符重载不是任意魔改语法，而是通过 trait 控制

## 定义

### 结合性

当多个**同优先级**运算符放在一起时，解释器 / 编译器要决定先怎么分组。

### 运算符重载

让自定义类型也能使用某些运算符，例如：

```rust
Point + Point
```

## 作用

### 结合性的作用

- 决定表达式默认求值方式
- 影响最终结果

### 运算符重载的作用

- 让自定义类型拥有更自然的表达方式
- 让“领域对象的操作”更贴近直觉

## 原理

### 1. 结合性影响同优先级表达式怎么抱团

```rust
20 - 5 - 3
```

默认通常按左结合去理解，相当于：

```rust
(20 - 5) - 3
```

所以结果是 `12`。

如果改成：

```rust
20 - (5 - 3)
```

结果就是 `18`。

### 2. Rust 的运算符重载靠 trait

当前例子里：

```rust
use std::ops::Add;

impl Add for Point {
    type Output = Self;
    fn add(self, rhs: Self) -> Self::Output { ... }
}
```

这表示：

- `Point` 之所以能用 `+`
- 不是因为编译器“随便接受”
- 而是因为它实现了 `Add`

### 3. 运算符重载是受约束的，不是任意改语义

Rust 不鼓励让运算符变得完全不可预测。你只能在语言和标准库提供的 trait 范围内扩展它们。

## 最小示例

```rust
use std::ops::Add;

#[derive(Debug, Clone, Copy)]
struct Point {
    x: i32,
    y: i32,
}

impl Add for Point {
    type Output = Self;

    fn add(self, rhs: Self) -> Self::Output {
        Self {
            x: self.x + rhs.x,
            y: self.y + rhs.y,
        }
    }
}

fn main() {
    println!("20 - 5 - 3 = {}", 20 - 5 - 3);
    println!("20 - (5 - 3) = {}", 20 - (5 - 3));

    let p1 = Point { x: 1, y: 2 };
    let p2 = Point { x: 3, y: 4 };
    println!("{:?}", p1 + p2);
}
```

## 注意点

### 1. 结合性和优先级不是一个概念

- **优先级**：谁先算
- **结合性**：同一优先级时怎么分组

### 2. 不是所有运算符都适合重载

Rust 的设计是在“可扩展”和“可预测”之间做平衡，不是让你把符号语义随便改掉。

### 3. 自定义重载要保持语义直觉

如果 `+` 的行为完全违背读者预期，代码虽然能编译，但会非常难维护。

## 常见错误

### ❌ 错误 1：把结合性理解成优先级

两者相关，但不是同一层概念。

### ❌ 错误 2：以为 Rust 可以像某些动态语言一样随意定义运算符

Rust 是 trait 驱动的，扩展范围是受限制的。

### ❌ 错误 3：重载后忘了思考语义是否自然

“能写”不等于“该写”。

## 我的理解

- 结合性解决的是表达式解释问题
- 运算符重载解决的是类型表达问题
- 这两节放在一起学很合理，因为它们都和“符号背后真正发生了什么”有关
