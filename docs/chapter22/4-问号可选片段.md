## `?` 在宏里的含义

在 `macro_rules!` 的匹配器里，`$(...)?`  表示"这个片段可选（0 或 1 次）"。
**注意：这是宏语法里的 `?`，不是错误传播的 `?`！**

```rust
macro_rules! optional_default {
    ($name:ident $(= $default:expr)?) => {
        // $default 可能存在，也可能不存在
        struct $name {
            value: i32,
        }
        impl Default for $name {
            fn default() -> Self {
                Self {
                    value: 0 $( + $default )?,  // 如果 $default 存在就加上
                }
            }
        }
    };
}

optional_default!(NoDefault);
optional_default!(WithDefault = 42);

println!("{}", NoDefault::default().value);   // 0
println!("{}", WithDefault::default().value);  // 42
```

## 常见用途：可选尾分隔符

```rust
macro_rules! collect_items {
    ( $( $item:expr ),* $(,)? ) => {
        vec![ $( $item ),* ]
    };
}

let v1 = collect_items!(1, 2, 3);     // 无尾逗号
let v2 = collect_items!(1, 2, 3, );   // 有尾逗号（$(,)?）
// 两种都合法
```

## 宏的进阶：过程宏

`macro_rules!` 是声明式宏。更强大的是**过程宏**（proc macros）：

- **Derive 宏**：`#[derive(MyTrait)]`
- **属性宏**：`#[my_attribute]`
- **函数式宏**：`my_macro!(...)`

过程宏需要单独的 crate（`proc-macro = true`），可以接收和生成任意 Rust 代码。
