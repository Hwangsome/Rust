# 3. 双向链表（前端插入）

- Cargo package: `chapter09`
- Run chapter: `cargo run -p chapter09`
- Chapter entry: `chapters/chapter09/src/main.rs`
- Reference module: `chapters/chapter09/src/doubly_link_list_part1.rs`
- Chapter lab: `chapters/chapter09/src/lab.rs`

## 定义

双向链表中的每个节点既指向后一个节点，也指向前一个节点，链表本身通常还会保存 head 和 tail。

## 作用

- 支持双向遍历
- 支持更灵活的插入和删除
- 暴露 Rust 中共享可变结构的复杂度

## 原理

因为同一个节点需要被多个地方共同持有，而且 next / prev 都要能改，单纯的 Box 已经不够。教学示例里通常会切换到 `Rc<RefCell<Node>>`。

## 最小示例

```rust
type Link = Option<Rc<RefCell<Node>>>;

struct Node {
    element: i32,
    next: Link,
    prev: Link,
}
```

## 注意点

- 这里的复杂度不是“语法复杂”，而是“所有权关系复杂”
- `Rc` 解决共享所有权，`RefCell` 解决内部可变性
- 只要开始双向引用，就要提前想到引用环

## 常见错误

- 还沿用单向链表的思路硬写
- 只看能不能编译，不看节点之间的关系是否一致
- 用了 Rc 后忘了它不提供可变访问

## 我的理解

双向链表是个很好的分水岭：一旦结构需要多方共同拥有，Rust 的抽象成本马上就会抬头。
