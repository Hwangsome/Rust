# 12. 错误信息与错误码

> 类型：**Study note**
> 关键词：diagnostics、error code、`rustc --explain`
> 上一篇：[11. 编译器指令](./11-编译器指令.md)
> 下一篇：[13. Rust 中的运算符](./13-Rust中的运算符.md)

## 一分钟结论

- Rust 的错误信息通常不是一句“编译失败”就结束
- 它通常会给你：**错误码、出错位置、原因说明、修复建议**
- `rustc --explain <错误码>` 是非常值得养成的习惯
- 把编译器当成老师，比把它当成障碍更有效

## 证据来源

- 章节入口：[chapters/chapter01/src/main.rs](../../chapters/chapter01/src/main.rs)
- 对应模块：[topic_11_error_messages_and_error_codes.rs](../../chapters/chapter01/src/topic_11_error_messages_and_error_codes.rs)

章节里给出的错误片段：

```rust
let x: i32 = String::from("Hello");
```

这类问题通常会得到类似 **`E0308`** 的类型不匹配错误。

## 定义

Rust 编译错误信息一般包含：

- **错误码**
- **源码位置**
- **出错原因**
- **上下文说明**
- **help / note**

错误码是这套系统里最有价值的索引入口之一。

## 作用

错误码和详细诊断的作用不是“让报错更长”，而是：

- 让错误可搜索
- 让错误可复用
- 让团队能快速对齐同一类问题
- 让初学者知道问题发生在哪一层

## 原理

### 1. 错误码是分类入口

例如：

- `E0308` 常见于类型不匹配
- `E0382` 常见于 move 后继续使用
- `E0502` 常见于借用冲突

这意味着你以后看到错误码时，不需要从零开始猜。

### 2. `rustc --explain` 会展开详细说明

```bash
rustc --explain E0308
```

它通常会给你：

- 这个错误大概是什么意思
- 为什么会发生
- 常见代码形态
- 正确思路是什么

### 3. Rust 错误信息常带修复提示

很多时候编译器不仅指出错误，还会直接告诉你方向：

- 要不要加 `mut`
- 要不要 `clone`
- 是不是借用关系冲突

这也是 Rust 学习曲线虽然陡，但反馈质量很高的原因之一。

## 最小示例

```rust
fn main() {
    let x: i32 = String::from("Hello");
    println!("{}", x);
}
```

这类代码的问题不是语法错，而是：

- 左边期待 `i32`
- 右边实际给的是 `String`

对应学习动作：

```bash
rustc --explain E0308
```

## 注意点

### 1. 不要只看最后一行

Rust 错误信息常常是结构化的：

- 第一行说错误类型
- 中间几行说具体位置
- `help` 和 `note` 往往最值钱

### 2. 错误码很重要

如果你只截图“编译失败”，以后几乎无法复盘。把错误码一起记下来，后续定位会快很多。

### 3. 很多错误其实是“概念没对齐”

比如类型不匹配、move、借用冲突，这些都不是靠记命令解决的，而是靠补概念。

## 常见错误

### ❌ 错误 1：看到红字就只盯第一行

Rust 的诊断信息往往越往下越有用，特别是：

- `help`
- `note`
- 高亮代码位置

### ❌ 错误 2：不记错误码

没有错误码，你下次很难快速找到同类问题。

### ❌ 错误 3：把每个错误都当成新问题

很多报错其实只是同一个概念在不同位置的重复体现。

## 我的理解

- Rust 错误信息本身就是学习资料
- 错误码不是附带信息，而是索引入口
- 真正高效的学习方式不是“避开错误”，而是“学会拆解错误”
