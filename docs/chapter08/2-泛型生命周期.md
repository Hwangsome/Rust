## 定义

泛型生命周期是写在函数或类型签名里的生命周期参数，例如 `'a`。它用来描述多个引用之间的约束关系。

## 作用

- 告诉编译器返回值和输入引用的寿命关系
- 让函数签名表达得更完整
- 解决“编译器无法单独推断”的场景

## 原理

像 `fn longer<'a>(left: &'a str, right: &'a str) -> &'a str` 的含义不是“返回值能活得更久”，而是“返回值寿命不会超过两个输入里较短的那个”。

## 最小示例

```rust
fn longer<'a>(left: &'a str, right: &'a str) -> &'a str {
    if left.len() >= right.len() { left } else { right }
}
```

## 注意点

- 生命周期参数描述的是关系，不是时间机器
- `'static` 表示引用在整个程序期间都有效，常见于字符串字面量
- 生命周期写得多不代表更高级，能省时就交给省略规则

## 常见错误

- 以为 `'a` 会延长局部变量寿命
- 看到生命周期就本能地想“哪个活得更久”
- 为了“保险”乱写生命周期，结果让签名更难懂

## 我的理解

生命周期标注更像类型系统里的“关系说明书”。它写的是边界，而不是分配策略。
