## 这一节解决什么问题

`vec![1,2,3].iter().map(|x| x*2).filter(|x| x>3).collect::<Vec<_>>()` 这种链式调用是 Rust 代码里非常常见的风格。

方法链成立的条件是：每个方法的**返回类型**能让下一个方法继续调用。当链中某步可能失败时，链的形状就会改变。

---

## 一分钟结论

- `-> Self`：按值消耗并返回，链式调用常用（Builder 模式）
- `-> &mut Self`：原地修改并返回引用，最节省内存
- `-> Result<Self>`：带验证的步骤，需要 `?` 或 `match` 处理
- 链断掉的根本原因：返回类型变成了 `Result<T>` / `Option<T>`，不再是 `T` 本身
- 恢复链：用 `?` 操作符（函数返回 Result）或组合子（`and_then`）

---

## 详细原理

### 1. `-> Self`（消耗型 Builder）

```rust
struct QueryBuilder {
    table: String,
    conditions: Vec<String>,
    order_by: Option<String>,
    limit: Option<usize>,
}

impl QueryBuilder {
    fn new(table: &str) -> Self {
        Self { table: table.into(), conditions: vec![], order_by: None, limit: None }
    }

    fn where_clause(mut self, cond: &str) -> Self {
        self.conditions.push(cond.into());
        self  // 返回 self，链可以继续
    }

    fn order_by(mut self, col: &str) -> Self {
        self.order_by = Some(col.into());
        self
    }

    fn limit(mut self, n: usize) -> Self {
        self.limit = Some(n);
        self
    }

    fn build(self) -> String {
        let mut sql = format!("SELECT * FROM {}", self.table);
        if !self.conditions.is_empty() {
            sql += &format!(" WHERE {}", self.conditions.join(" AND "));
        }
        if let Some(col) = self.order_by {
            sql += &format!(" ORDER BY {col}");
        }
        if let Some(n) = self.limit {
            sql += &format!(" LIMIT {n}");
        }
        sql
    }
}

let sql = QueryBuilder::new("users")
    .where_clause("age > 18")
    .where_clause("active = 1")
    .order_by("name")
    .limit(10)
    .build();
println!("{sql}");
```

### 2. `-> &mut Self`（原地修改型）

```rust
struct Config {
    debug: bool,
    port: u16,
    log_level: String,
}

impl Config {
    fn new() -> Self {
        Config { debug: false, port: 8080, log_level: "info".into() }
    }

    fn debug(&mut self, enabled: bool) -> &mut Self {
        self.debug = enabled;
        self  // 返回 &mut Self，可以继续链
    }

    fn port(&mut self, port: u16) -> &mut Self {
        self.port = port;
        self
    }
}

let mut cfg = Config::new();
cfg.debug(true).port(9090);
println!("port={}, debug={}", cfg.port, cfg.debug);
```

### 3. `-> Result<Self>`（带验证）

```rust
struct Connection {
    host: String,
    port: u16,
}

impl Connection {
    fn new() -> Self { Connection { host: String::new(), port: 0 } }

    fn host(mut self, h: &str) -> Result<Self, String> {
        if h.is_empty() { return Err("主机不能为空".into()); }
        self.host = h.to_string();
        Ok(self)  // 链断了！需要 ?
    }

    fn port(mut self, p: u16) -> Result<Self, String> {
        if p == 0 { return Err("端口不能为 0".into()); }
        self.port = p;
        Ok(self)
    }

    fn connect(self) -> String {
        format!("连接 {}:{}", self.host, self.port)
    }
}

fn create_connection(host: &str, port: u16) -> Result<String, String> {
    Ok(Connection::new()
        .host(host)?  // ? 处理 Err，Ok 时解包继续链
        .port(port)?
        .connect())
}
```

## 完整运行示例

```rust
fn main() {
    println!("=== 消耗型 Builder ===");
    let sql = QueryBuilder::new("orders")
        .where_clause("status = 'pending'")
        .where_clause("created_at > '2024-01-01'")
        .order_by("created_at DESC")
        .limit(50)
        .build();
    println!("{sql}");
    println!();

    println!("=== 带验证的 Builder（Result）===");
    let cases = [("localhost", 8080), ("", 8080), ("localhost", 0)];
    for (host, port) in cases {
        match create_connection(host, port) {
            Ok(msg) => println!("  ✅ {msg}"),
            Err(e) => println!("  ❌ {e}"),
        }
    }
    println!();

    println!("=== 方法链断裂时的处理 ===");
    // and_then 不需要函数返回 Result
    let result = Connection::new()
        .host("example.com")
        .and_then(|conn| conn.port(443))
        .map(|conn| conn.connect());
    println!("  {result:?}");
}

// 为了演示 and_then 的用法
impl Result<Connection, String> {}  // 标准 Result 自带 and_then

struct QueryBuilder {
    table: String,
    conditions: Vec<String>,
    order_by: Option<String>,
    limit: Option<usize>,
}

impl QueryBuilder {
    fn new(table: &str) -> Self { Self { table: table.into(), conditions: vec![], order_by: None, limit: None } }
    fn where_clause(mut self, cond: &str) -> Self { self.conditions.push(cond.into()); self }
    fn order_by(mut self, col: &str) -> Self { self.order_by = Some(col.into()); self }
    fn limit(mut self, n: usize) -> Self { self.limit = Some(n); self }
    fn build(self) -> String {
        let mut sql = format!("SELECT * FROM {}", self.table);
        if !self.conditions.is_empty() { sql += &format!(" WHERE {}", self.conditions.join(" AND ")); }
        if let Some(col) = self.order_by { sql += &format!(" ORDER BY {col}"); }
        if let Some(n) = self.limit { sql += &format!(" LIMIT {n}"); }
        sql
    }
}

struct Connection { host: String, port: u16 }
impl Connection {
    fn new() -> Self { Connection { host: String::new(), port: 0 } }
    fn host(mut self, h: &str) -> Result<Self, String> {
        if h.is_empty() { Err("主机不能为空".into()) } else { self.host = h.into(); Ok(self) }
    }
    fn port(mut self, p: u16) -> Result<Self, String> {
        if p == 0 { Err("端口不能为 0".into()) } else { self.port = p; Ok(self) }
    }
    fn connect(self) -> String { format!("连接 {}:{}", self.host, self.port) }
}

fn create_connection(host: &str, port: u16) -> Result<String, String> {
    Ok(Connection::new().host(host)?.port(port)?.connect())
}
```
