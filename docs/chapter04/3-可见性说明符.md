## 这一节解决什么问题

Rust 的默认是**私有**，只有明确标注 `pub` 的才对外可见。这是"最小权限原则"的体现——不小心暴露内部细节会导致 API 耦合过深。

---

## 一分钟结论


| 修饰符            | 可见范围                       |
| -------------- | -------------------------- |
| 无修饰            | 只在定义它的模块（及子模块）内            |
| `pub`          | 完全公开（任何能访问该模块的地方）          |
| `pub(crate)`   | 当前 crate 内部（对外部 crate 不可见） |
| `pub(super)`   | 只对父模块可见                    |
| `pub(in path)` | 只对 `path` 指定的模块可见（高级用法）    |


---

## 详细原理

### 1. 默认私有

```rust
mod auth {
    pub fn login(user: &str) -> bool {
        validate_password(user)  // 内部调用私有函数，合法
    }

    fn validate_password(user: &str) -> bool {
        // 私有：外部不能直接调用
        user.len() > 3
    }
}

// auth::login("alice");  ✅
// auth::validate_password("alice");  ❌ 私有
```

### 2. `pub(crate)` 和 `pub(super)`

```rust
mod library {
    // pub(crate)：整个 crate 内部可用，对外部用户不可见
    pub(crate) fn internal_utility() { }

    mod implementation {
        // pub(super)：只对父模块 library 可见
        pub(super) fn helper() { }

        pub fn uses_helper() {
            helper(); // 自身可用
        }
    }

    pub fn public_api() {
        implementation::helper(); // 父模块可用（pub(super)）
        internal_utility();        // 同 crate 可用（pub(crate)）
    }
}
```

## 完整运行示例

```rust
mod school {
    pub fn open_class() {
        prepare_room();  // 调用私有函数（同模块内合法）
        println!("课程开始");
    }

    fn prepare_room() {
        println!("  私有: 准备教室");
    }

    pub(crate) fn faculty_meeting() {
        println!("  教职工会议（仅 crate 内）");
    }

    pub mod departments {
        pub fn list() -> Vec<&'static str> {
            vec!["数学", "英语", "科学"]
        }

        pub(super) fn admin_only() {
            println!("  管理部门访问（仅父模块 school）");
        }
    }
}

pub fn run() {
    school::open_class();

    // pub(crate) - 同 crate 内可调用
    school::faculty_meeting();

    // pub - 完全公开
    println!("科系: {:?}", school::departments::list());

    // pub(super) - 只有 school 模块内能调用，这里调用会失败
    // school::departments::admin_only(); // ❌ E0603
}
```

## 注意点

### struct 字段默认私有（即使 struct 是 pub）

```rust
pub struct Config {
    pub host: String,    // pub：外部可读写
    port: u16,           // 私有：外部不能直接访问
}

// 外部代码：
let c = Config { host: "localhost".into(), port: 8080 };
// ❌ 因为 port 是私有的！
// 只能通过 Config::new() 等方法创建
```
