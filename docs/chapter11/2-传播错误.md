## 定义

传播错误是指函数在遇到失败时，不自己决定最终处理策略，而是把错误返回给调用者。

## 作用

- 保持函数职责单一
- 让高层统一决定恢复策略
- 减少重复的 match 样板代码

## 原理

`?` 会在 `Err` 或 `None` 出现时提前返回当前函数，在成功时继续向下执行。因此它本质上是“提前 return 的语法糖”。

## 最小示例

```rust
fn read_number(input: &str) -> Result<i32, ParseIntError> {
    let number = input.trim().parse::<i32>()?;
    Ok(number)
}
```

## 注意点

- 用 `?` 的前提是当前函数返回兼容的 `Result` 或 `Option`
- `?` 没有处理错误，只是传播错误
- 什么时候在本层处理，什么时候往上传，是 API 设计问题

## 常见错误

- 把 `?` 理解成“自动吞掉错误”
- 当前函数返回类型不兼容，还强行使用 `?`
- 在底层函数里过早决定展示或记录错误

## 我的理解

错误传播的本质是职责分层。不是每一层都该做最终处理，有些层只需要把上下文干净地传上去。
