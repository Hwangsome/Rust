## 这一节解决什么问题

没有泛型时，你为 `i32`、`f64`、`String` 分别写了三套几乎一样的 `Stack<i32>`、`Stack<f64>`、`Stack<String>`——代码量三倍，逻辑完全重复。

有了泛型，你写一次 `Stack<T>`，编译器替你生成所有具体版本。这是**不重复自己（DRY）**原则在类型系统层面的体现。

关键是：Rust 的泛型和 Java 不同。Java 的泛型在运行时被**类型擦除**（ArrayList 的 Integer 信息在字节码里消失了）。Rust 的泛型是**单态化（monomorphization）**——编译器为每个实际使用的类型组合生成一份专属代码，**运行时零开销**，可以内联，可以优化。

---

## 一分钟结论

- 泛型用 `<T>` 声明类型参数，`T` 是占位符，编译时被替换成具体类型
- **单态化**：编译器为 `Vec<i32>` 和 `Vec<String>` 生成两份独立的代码，零运行时开销
- 泛型函数：`fn max<T: PartialOrd>(a: T, b: T) -> T`
- 泛型 struct：`struct Pair<T> { first: T, second: T }`
- 泛型 impl：`impl<T, U> Point<T, U>` 对所有组合实现；`impl Point<i32, i32>` 只对特定类型实现（**部分特化**）
- 方法可以引入自己的类型参数：`fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W>`
- 代价：每种使用的类型都会生成一份代码，可能**增加二进制体积**

## 与其他语言对比


| 语言   | 泛型/模板的实现方式                                              | 运行时开销        |
| ---- | ------------------------------------------------------- | ------------ |
| Java | 类型擦除（Type Erasure）：编译后 `List<Integer>` 变 `List<Object>` | 装箱拆箱开销       |
| C#   | 运行时具化（Reification）：.NET JIT 动态生成具体类型代码                  | 极小           |
| C++  | 模板（Template）：编译期展开，每种类型一份代码                             | 零（和 Rust 相同） |
| Go   | 1.18+ 泛型：实现方式介于前两者之间                                    | 较小           |
| Rust | 单态化（Monomorphization）：编译期为每种类型生成专属代码                    | **零**        |


Java 程序员来学 Rust 泛型最大的认知跳跃：Java 的 `List<Integer>` 在运行时其实是 `List<Object>`，需要装箱；Rust 的 `Vec<i32>` 是真正存放 `i32` 的专用代码，无装箱，可内联。

## 核心概念与心智模型

### 单态化：编译器替你生成多份代码

```rust
fn largest<T: PartialOrd>(a: T, b: T) -> T {
    if a > b { a } else { b }
}

// 调用方：
largest(5_i32, 10_i32); // 编译器生成 largest_i32(a: i32, b: i32) -> i32
largest(5.0_f64, 3.0);  // 编译器生成 largest_f64(a: f64, b: f64) -> f64

// 生成的代码（概念上）：
fn largest_i32(a: i32, b: i32) -> i32 {
    if a > b { a } else { b }
}
fn largest_f64(a: f64, b: f64) -> f64 {
    if a > b { a } else { b }
}
```

你写了一个函数，但编译后二进制里有多个版本。好处是零运行时开销；代价是二进制体积增加（但 LTO 可以部分抵消）。

### 泛型 struct 的字段类型

```
struct Point<T, U> {
    x: T,
    y: U,
}

T 和 U 是独立的类型参数，可以相同也可以不同：
Point<i32, i32>  { x: 1, y: 2 }       → 两个整数
Point<f64, f64>  { x: 1.0, y: 2.0 }   → 两个浮点
Point<i32, f64>  { x: 1, y: 2.0 }     → 整数 + 浮点
Point<&str, Vec<u8>> { x: "hello", y: vec![1,2] } → 完全不同类型
```

## 详细原理

### 1. 泛型函数

```rust
// 最小约束：T 只需要能比较大小
fn max_of<T: PartialOrd>(a: T, b: T) -> T {
    if a > b { a } else { b }
}

// 多个 bound：T 需要能比较且能打印
fn max_and_print<T: PartialOrd + std::fmt::Display>(a: T, b: T) -> T {
    let result = if a > b { a } else { b };
    println!("max = {result}");
    result
}

// where 子句：更可读的写法（等价）
fn max_and_print_v2<T>(a: T, b: T) -> T
where
    T: PartialOrd + std::fmt::Display,
{
    let result = if a > b { a } else { b };
    println!("max = {result}");
    result
}
```

### 2. 泛型 struct

```rust
#[derive(Debug)]
struct Point<T, U> {
    x: T,
    y: U,
}

// impl 块也需要声明泛型参数
impl<T, U> Point<T, U> {
    fn new(x: T, y: U) -> Self {
        Self { x, y }
    }

    fn x_ref(&self) -> &T {
        &self.x
    }
}

// 部分特化：只为特定类型组合实现额外方法
impl Point<f64, f64> {
    fn distance_from_origin(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

// 方法自己的泛型参数（mixup）
impl<T, U> Point<T, U> {
    fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {
        Point {
            x: self.x,    // 保留 self 的 x（类型 T）
            y: other.y,   // 采用 other 的 y（类型 W）
        }
    }
}
```

### 3. 泛型枚举（标准库中大量使用）

```rust
// 这就是标准库里 Option 和 Result 的定义
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}

// 你的业务代码里也可以用泛型枚举：
#[derive(Debug)]
enum Tree<T> {
    Leaf(T),
    Node(Box<Tree<T>>, Box<Tree<T>>),
}

let tree: Tree<i32> = Tree::Node(
    Box::new(Tree::Leaf(1)),
    Box::new(Tree::Leaf(2)),
);
```

## 完整运行示例

```rust
use std::fmt::{Display, Debug};
use std::mem::size_of;

// ===== 1. 泛型函数 =====
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}

// ===== 2. 泛型 struct =====
#[derive(Debug)]
struct Point<T, U> {
    x: T,
    y: U,
}

impl<T: Display, U: Display> Point<T, U> {
    fn new(x: T, y: U) -> Self { Self { x, y } }
    fn display(&self) { println!("  ({}, {})", self.x, self.y); }
}

// 部分特化
impl Point<f64, f64> {
    fn distance(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

// 方法级泛型参数
impl<T: Display, U: Display> Point<T, U> {
    fn mixup<V: Display, W: Display>(self, other: Point<V, W>) -> Point<T, W> {
        Point { x: self.x, y: other.y }
    }
}

// ===== 3. 泛型枚举 =====
#[derive(Debug)]
enum Pair<T> {
    Two(T, T),
    Single(T),
    Empty,
}

impl<T: Display + PartialOrd> Pair<T> {
    fn show_max(&self) {
        match self {
            Pair::Two(a, b) if a > b => println!("  max = {a}"),
            Pair::Two(_, b) => println!("  max = {b}"),
            Pair::Single(x) => println!("  only = {x}"),
            Pair::Empty => println!("  empty"),
        }
    }
}

fn main() {
    println!("=== 1. 泛型函数 ===");
    let integers = [34, 50, 25, 100, 65];
    let floats = [1.1, 2.2, 0.5, 9.9];
    let words = ["banana", "apple", "cherry"];

    println!("  largest int:   {}", largest(&integers));
    println!("  largest float: {}", largest(&floats));
    println!("  largest word:  {}", largest(&words));
    println!();

    println!("=== 2. 泛型 struct ===");
    let int_pt = Point::new(1_i32, 2_i32);
    let str_float = Point::new("hello", 3.14_f64);
    let float_pt = Point::new(3.0_f64, 4.0_f64);

    int_pt.display();
    str_float.display();
    println!("  distance from origin: {:.2}", float_pt.distance());

    // mixup：交叉类型
    let mixed = int_pt.mixup(Point::new("world", 99_u32));
    mixed.display(); // (1, 99)（x 来自 int_pt，y 来自 new Point 的 y）
    println!();

    println!("=== 3. 泛型枚举 ===");
    let pair1: Pair<i32> = Pair::Two(10, 30);
    let pair2: Pair<&str> = Pair::Two("rust", "go");
    let empty: Pair<f64> = Pair::Empty;

    pair1.show_max(); // max = 30
    pair2.show_max(); // max = rust（字典序）
    empty.show_max(); // empty
    println!();

    println!("=== 4. 单态化的内存证明 ===");
    println!("  Point<i32,i32> 大小 = {} bytes", size_of::<Point<i32,i32>>());     // 8
    println!("  Point<f64,f64> 大小 = {} bytes", size_of::<Point<f64,f64>>());     // 16
    println!("  Point<i8,i64>  大小 = {} bytes", size_of::<Point<i8,i64>>());      // 16（含对齐）
    println!("  不同类型实例大小不同，说明是真正的不同代码，不是擦除");
}
```

**预期输出**：

```text
=== 1. 泛型函数 ===
  largest int:   100
  largest float: 9.9
  largest word:  cherry

=== 2. 泛型 struct ===
  (1, 2)
  (hello, 3.14)
  distance from origin: 5.00
  (1, 99)

=== 3. 泛型枚举 ===
  max = 30
  max = rust
  empty

=== 4. 单态化的内存证明 ===
  Point<i32,i32> 大小 = 8 bytes
  Point<f64,f64> 大小 = 16 bytes
  Point<i8,i64>  大小 = 16 bytes（含对齐）
  不同类型实例大小不同，说明是真正的不同代码，不是擦除
```

## 编译器错误分析

### ❌ E0277：T 不满足 trait bound

```rust
fn max<T>(a: T, b: T) -> T { // 没有约束！
    if a > b { a } else { b } // ❌ T 不一定能比较
}
```

```text
error[E0369]: binary operation `>` cannot be applied to type `T`
  |
3 | if a > b { a } else { b }
  |    ^^^^^ T doesn't implement `PartialOrd`
  |
help: consider restricting type parameter `T`
  |
1 | fn max<T: std::cmp::PartialOrd>(a: T, b: T) -> T {
  |         ++++++++++++++++++++++++
```

### ❌ E0308：类型不匹配（错误使用部分特化）

```rust
let p = Point::new(1_i32, 2_i32); // Point<i32, i32>
```

## 实际工程场景

### 1. 通用缓存（Cache<K, V>）

```rust
use std::collections::HashMap;

struct Cache<K, V> {
    store: HashMap<K, V>,
}

impl<K: Eq + std::hash::Hash, V: Clone> Cache<K, V> {
    fn new() -> Self { Cache { store: HashMap::new() } }

    fn get_or_insert(&mut self, key: K, default: V) -> &V {
        self.store.entry(key).or_insert(default)
    }
}

let mut cache: Cache<String, i32> = Cache::new();
cache.get_or_insert("answer".into(), 42);
```

### 2. 标准库里无处不在的泛型

```rust
// 这些都是泛型：
let v: Vec<i32> = Vec::new();
let h: HashMap<String, Vec<u8>> = HashMap::new();
let r: Result<i32, String> = Ok(42);
let o: Option<&str> = Some("hello");

// 方法也是泛型的：
let parsed: i32 = "42".parse().unwrap(); // parse::<i32>()
let collected: Vec<String> = vec!["a", "b"].iter().map(|s| s.to_string()).collect();
```

## 性能影响

### 单态化的代价与收益


| 方面           | 内容                                                                             |
| ------------ | ------------------------------------------------------------------------------ |
| ✅ 运行时零开销     | `Vec<i32>` 和 `Vec<f64>` 各自是专用代码，可内联，可优化                                        |
| ✅ 类型安全       | 编译期而不是运行时检查                                                                    |
| ❌ 编译时间增加     | 每种组合都要编译一遍                                                                     |
| ❌ 二进制体积可能增大  | 多个具体版本共存于二进制（LTO 可以消除死代码）                                                      |
| ✅ vs Java 装箱 | Rust `Vec<i32>` 存储实际 i32；Java `ArrayList<Integer>` 存 Integer 对象（8字节指针+16字节对象头） |

## 注意点与陷阱

### 陷阱 1：`impl<T>` 和 `impl ConcreteType` 不能有重名方法

```rust
impl<T> Point<T, T> {
    fn add_x(&self) { /* ... */ }
}

impl Point<i32, i32> {
    // fn add_x(&self) { /* ... */ } // ❌ 和上面重名，ambiguous
}
```

### 陷阱 2：泛型类型默认不实现任何 trait

```rust
fn print<T>(x: T) {
    println!("{x}"); // ❌ T 没有 Display 约束！
}
```

## 我的理解与记忆方法

**单态化心智模型**：

> 你写了一个 `fn max<T>` 函数的"模板"。编译器把你每次调用 `max` 的地方都看一下，收集所有用到的 T 类型，然后为每种类型各生成一份具体函数。运行时就像你自己手写了多份一样，没有任何额外开销。

**什么时候加 trait bound**：

```
需要对 T 做操作 → 该操作对应哪个 trait → 加那个 bound
  比较大小   → PartialOrd / Ord
  打印       → Display / Debug
  加减乘除   → Add / Sub / Mul / Div
  默认值     → Default
  克隆       → Clone
```
