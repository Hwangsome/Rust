
## 定义

迭代器是按需逐个产出元素的对象。Rust 里最核心的接口是 `Iterator` trait，而它真正必须实现的方法只有 `next()`。

## 作用

- 统一“逐个处理元素”的接口
- 为过滤、映射、收集等组合子提供基础
- 支持惰性求值

## 原理

`for` 循环、`map`、`filter` 等写法，底层都建立在 `Iterator::next()` 之上。只要一个类型能不断给出 `Option<Item>`，它就能融入这套生态。

## 最小示例

```rust
impl Iterator for EmployeeRecords {
    type Item = String;

    fn next(&mut self) -> Option<Self::Item> {
        // 返回 Some(item) 或 None
    }
}
```

## 注意点

- 迭代器描述的是“产出过程”，不是“容器本身”
- `next()` 每次调用都会推进内部状态
- 很多迭代器适配器都是惰性的，不消费就不会真正执行

## 常见错误

- 以为 `for` 循环和迭代器是两套系统
- 忽略 `next()` 会改变迭代器状态
- 只会用现成迭代器，不理解自定义类型如何接入

## 我的理解

Iterator 像一份“逐项产出协议”。理解它之后，很多集合 API 都会从“语法记忆”变成“协议推导”。
