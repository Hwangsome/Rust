---

## 这一节解决什么问题

单元测试测的是内部细节——但用户看到的是公开 API。集成测试从"用户视角"测试，验证不同模块协作的行为是否正确。

---

## 一分钟结论

- 集成测试放在 `tests/` 目录，和 `src/` 同级
- 每个 `tests/*.rs` 文件是一个独立的 crate，只能访问**公开 API**
- 通过 `use package_name::...` 引入被测 crate
- `cargo test --test smoke` 只运行 `tests/smoke.rs` 里的测试
- 共享测试辅助代码放在 `tests/common/mod.rs`（不能用 `tests/common.rs`！）

---

## 单元测试 vs 集成测试对比

| 特性 | 单元测试 | 集成测试 |
|-----|---------|---------|
| 位置 | `src/` 内，`#[cfg(test)]` | `tests/` 目录 |
| 能访问私有 API | ✅ | ❌（只有 pub）|
| 每个文件是独立 crate | ❌ | ✅ |
| 测试的对象 | 函数/模块内部逻辑 | 公开 API 的行为 |

---

## 目录结构

```
src/
├── lib.rs       ← library crate（集成测试的目标）
└── main.rs      ← binary crate（集成测试测不到 binary！）

tests/           ← 集成测试目录
├── common/      ← 共享辅助代码
│   └── mod.rs   ← 必须用这个路径（不是 common.rs）
├── api_tests.rs ← 一个测试文件
└── smoke.rs     ← 另一个测试文件
```

## 完整运行示例

```rust
// 这是 chapters/chapter05/tests/smoke.rs 的内容
// 展示集成测试的标准写法

// 集成测试可以 use 本 package 的公开内容
// use chapter05::add; // ← 假设 src/lib.rs 暴露了 add 函数

// 简单的烟雾测试
#[test]
fn smoke_arithmetic() {
    assert_eq!(2 + 2, 4);
    assert_eq!(10 - 3, 7);
}

#[test]
fn smoke_string_operations() {
    let s = String::from("hello");
    assert_eq!(s.to_uppercase(), "HELLO");
    assert_eq!(s.len(), 5);
    assert!(s.contains("ell"));
}

#[test]
fn smoke_vec_operations() {
    let mut v: Vec<i32> = vec![1, 2, 3, 4, 5];
    v.push(6);
    assert_eq!(v.len(), 6);
    assert_eq!(v.iter().sum::<i32>(), 21);

    let evens: Vec<_> = v.iter().filter(|&&x| x % 2 == 0).collect();
    assert_eq!(evens.len(), 3);
}
```

## 共享辅助代码

```rust
// tests/common/mod.rs
pub fn setup_test_data() -> Vec<i32> {
    vec![1, 2, 3, 4, 5]
}

pub fn assert_sorted(v: &[i32]) {
    for i in 1..v.len() {
        assert!(v[i-1] <= v[i], "not sorted at index {i}: {} > {}", v[i-1], v[i]);
    }
}

// tests/sorting_tests.rs
mod common;

#[test]
fn test_sort() {
    let mut data = common::setup_test_data();
    data.sort();
    common::assert_sorted(&data);
}
```
