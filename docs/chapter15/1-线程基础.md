## OS 线程 vs 绿色线程

Rust 的 `std::thread` 是**真正的操作系统线程**：
- 重量级（约几 MB 内存栈）
- 由操作系统调度（抢占式）
- 适合 CPU 密集型工作或少量长期任务

对比 `async/await` + tokio 的**绿色线程**（协程）：
- 轻量级（几 KB）
- 由 runtime 调度（协作式）
- 适合大量 IO 密集型任务

## 基础 API

```rust
use std::thread;
use std::time::Duration;

// 启动线程，返回 JoinHandle
let handle = thread::spawn(|| {
    println!("子线程运行中，id={:?}", thread::current().id());
    42  // 线程的返回值
});

// join() 等待线程完成，返回 Result<T, Box<dyn Any>>
let result = handle.join().expect("线程 panic 了");
println!("子线程返回: {result}");

// sleep
thread::sleep(Duration::from_millis(100));
```

## 完整运行示例

```rust
use std::thread;
use std::time::{Duration, Instant};

pub fn run() {
    println!("=== 线程基础 ===");
    let start = Instant::now();

    // 启动多个线程并行工作
    let handles: Vec<_> = (0..4).map(|i| {
        thread::spawn(move || {
            thread::sleep(Duration::from_millis(100));
            println!("  线程 {i} 完成");
            i * i  // 返回平方
        })
    }).collect();

    // 等待所有线程，收集结果
    let results: Vec<u32> = handles.into_iter()
        .map(|h| h.join().unwrap())
        .collect();

    println!("  结果: {results:?}");
    println!("  总耗时: {:?} (4个线程并行，约100ms)", start.elapsed());
}
```
