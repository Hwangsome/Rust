## 这一节解决什么问题

一个常见困惑：

```rust
fn increase(mut value: i32) -> i32 {
    value += 1;
    value
}

let score = 10;
let new_score = increase(score);
println!("{score}"); // 10，不是 11！为什么？
```

参数写了 `mut`，但调用方的 `score` 没有变——这和很多人的直觉不符。

这是因为**参数的 `mut` 只影响函数内部的绑定，和调用方的变量无关**。

---

## 一分钟结论

- `fn f(mut x: T)` 中的 `mut` 只是说"函数内部可以修改这个参数绑定"
- 对 `Copy` 类型（如 `i32`）：函数拿到的是副本，修改副本不影响原值
- 对非 `Copy` 类型（如 `String`）：函数拿到了所有权，但调用方的变量已经失效
- 如果想让函数修改调用方的数据，必须传 **`&mut T`**（可变引用）
- 这个设计使得 Rust 中的函数调用默认是"隔离的"——不会意外改变调用方数据

---

## 与其他语言对比

| 语言 | `f(x)` 能改调用方的 `x` 吗？ |
|-----|---------------------------|
| C | 不能（值传递复制）；指针才能 |
| C++ | 不能（值传递）；引用 `T&` 才能 |
| Java | 基本类型：不能；对象：能（通过引用改字段）|
| Python | 不可变对象（int/str）：不能；可变对象（list）：能 |
| Rust | Copy 类型：不能；非 Copy：所有权转移；`&mut T`：才能 |

## 详细原理

### 1. Copy 类型：函数拿到的是副本

```rust
fn double(mut n: i32) -> i32 {
    n *= 2;  // 修改的是函数内部的副本
    n
}

let original = 5;
let result = double(original);

println!("original = {original}");  // 5（未变）
println!("result   = {result}");    // 10
```

**内存图**：

```
调用前：
  original: [5] （栈上）

调用 double(original) 时：
  original: [5] （栈上，不变）
  n: [5] ← 复制的副本（因为 i32 是 Copy）

函数内 n *= 2：
  n: [10]（仅副本变了）

返回后：
  original: [5]（没被动）
  result: [10]
```

### 2. 非 Copy 类型：所有权转移，调用方失效

```rust
fn process(mut s: String) -> String {
    s.push_str(" processed");
    s  // 返回修改后的 String
}

let greeting = String::from("hello");
let result = process(greeting);

// println!("{greeting}"); // ❌ greeting 已被 move 进函数
println!("{result}");  // "hello processed"
```

### 3. 想真正修改调用方的数据：用 `&mut T`

```rust
fn add_one(x: &mut i32) {
    *x += 1;  // 通过引用修改调用方的值
}

let mut count = 0;
add_one(&mut count);
add_one(&mut count);
println!("count = {count}");  // 2（被修改了！）
```

## 完整运行示例

```rust
// 1. mut 参数：只影响函数内部
fn modify_inside(mut value: i32, mut label: String) -> (i32, String) {
    value *= 10;
    label.push_str(" [modified]");
    (value, label)
}

// 2. &mut 参数：影响调用方
fn modify_outside(value: &mut i32, label: &mut String) {
    *value *= 10;
    label.push_str(" [modified]");
}

fn main() {
    // 测试 1：mut 参数（不影响调用方）
    let num = 7;
    let text = String::from("hello");
    let (new_num, new_text) = modify_inside(num, text.clone());

    println!("=== mut 参数：不影响调用方 ===");
    println!("num   原: {num}  (text 已 move, 无法打印原值)");
    println!("结果: num={new_num}, text={new_text}");
    println!();

    // 测试 2：&mut 参数（影响调用方）
    let mut counter = 5;
    let mut message = String::from("start");

    println!("=== &mut 参数：影响调用方 ===");
    println!("修改前: counter={counter}, message={message}");
    modify_outside(&mut counter, &mut message);
    println!("修改后: counter={counter}, message={message}");
    println!();

    // 测试 3：实际场景
    println!("=== 实际场景：累加器 ===");
    let scores = vec![10, 20, 30, 40, 50];
    let mut total = 0;

    fn accumulate(total: &mut i32, score: i32) {
        *total += score;
    }

    for &s in &scores {
        accumulate(&mut total, s);
    }
    println!("总分: {total}");
}
```

**预期输出**：

```text
=== mut 参数：不影响调用方 ===
num   原: 7  (text 已 move, 无法打印原值)
结果: num=70, text=hello [modified]

=== &mut 参数：影响调用方 ===
修改前: counter=5, message=start
修改后: counter=50, message=start [modified]

=== 实际场景：累加器 ===
总分: 150
```

## 编译器错误分析

### ❌ 忘记 `mut`：试图修改不可变绑定

```rust
fn bad(n: i32) {
    n += 1; // ❌ 参数默认是不可变绑定
}
```

```text
error[E0384]: cannot assign twice to immutable variable `n`
  |
2 |     n += 1;
  |     ^^^^^^ cannot assign to immutable variable
help: consider making this binding mutable
  |
1 | fn bad(mut n: i32) {
  |        +++
```

### ❌ 用 `mut T` 试图影响调用方（常见误解）

```rust
fn try_modify(mut n: i32) {
    n = 100; // 只改了副本，调用方看不到
}

let x = 5;
try_modify(x);
println!("{x}"); // 仍然是 5，不是 100
// 没有报错，但行为不符合预期
```

## 注意点与陷阱

### 陷阱：`mut` 修饰参数绑定 vs `&mut T` 是两个概念

```rust
// mut n: i32 → 参数绑定可变（函数内部可以改 n 的值）
fn with_mut_binding(mut n: i32) { n += 1; }

// n: &mut i32 → 参数是可变引用（可以改调用方的值）
fn with_mut_ref(n: &mut i32) { *n += 1; }

// mut n: &mut i32 → 参数绑定可变 + 可变引用（可以既改调用方值，也能让 n 指向别处）
fn both(mut n: &mut i32) {
    *n += 1;      // 改调用方的值
    // n = &mut other; // 还可以让 n 指向其他 &mut i32
}
```

## 我的理解与记忆方法

**核心直觉**：

> 函数参数就像"你把东西借给/送给了朋友"。
> - `T`（Copy）：你复制了一份给朋友，你手上的还在
> - `T`（non-Copy）：你把原件送给了朋友，你手上没有了
> - `&mut T`：你把东西借给朋友，让他可以修改，改完你拿回来

**什么时候用 `mut n: T` vs `n: &mut T`**：

```
只需要在函数内部临时改变它（结果不返回给调用方）→ mut n: T
需要让调用方看到修改（原地修改） → n: &mut T
需要改完了返回新值给调用方     → mut n: T + return n
```
