# 3. 可见性说明符

> 类型：**Study note**
> 关键词：`pub`、private、visibility、E0603
> 上一篇：[2. 模块基础](./2-模块基础.md)
> 下一篇：[4. 模块中的隐私与重导出](./4-模块中的隐私与重导出.md)

## 一分钟结论

- Rust 默认更保守：模块成员默认私有
- `pub` 是显式开放访问权限的关键字
- “能在模块内用”和“能在模块外用”是两回事

## 证据来源

- 对应模块：[topic_03_visibility.rs](../../chapters/chapter04/src/topic_03_visibility.rs)
- 运行章节：`cargo run -p chapter04`
- 补充实验：私有函数从模块外调用会报 `E0603`

关键输出：

```text
私有函数只能在当前模块内部使用。
公开函数可以被模块外调用。
```

补充实验错误：

```text
error[E0603]: function `prepare_room` is private
```

## 扩展演示输出（当前代码已升级）

`topic_03_visibility.rs` 现在把 4 种可见性都走一遍：默认私有（`prepare_room`）→ `pub`（`open_class`, `public_notice`）→ `pub(crate)`（`announce`，在 crate 内随处可调但不会被公开到 library 外）→ `pub(super)`（`internal_policy`，只对父模块 `school` 可见）。

```text
-- (1) pub 函数入口 --
  私有 prepare_room()：排桌椅、擦黑板
  pub(crate) announce()：crate 内部任何模块都可访问
公开入口完成上课前准备。

-- (2) pub(crate) --
  pub(crate) announce()：crate 内部任何模块都可访问

-- (3) pub(super) 只对父模块可见 --
  pub(super) internal_policy()：只有 school 内部能调用
  pub public_notice()：对外的公告
```

取消 `school::administration::internal_policy();` 的注释会触发 **E0603**：pub(super) 对外部模块不可见。

## 定义

可见性说明符决定一个项能被哪些作用域访问。

## 作用

- 控制 API 暴露面
- 隐藏内部实现细节
- 保持模块边界清晰

## 原理

当前模块示例：

```rust
mod school {
    pub fn open_class() {
        prepare_room();
    }

    fn prepare_room() { ... }
}
```

这表示：

- `open_class` 对模块外公开
- `prepare_room` 只在模块内可见

## 最小示例

```rust
mod school {
    pub fn open_class() {
        prepare_room();
    }

    fn prepare_room() {
        println!("private");
    }
}

fn main() {
    school::open_class();
}
```

## 注意点

### 1. Rust 默认私有

这和一些默认公开的语言风格不同，是非常有意的设计。

### 2. `pub` 是 API 承诺的一部分

一旦公开出去，后面修改成本通常会更高。

## 常见错误

### ❌ 错误 1：为了方便一路全 `pub`

这样会快速失去模块边界。

### ❌ 错误 2：以为模块里能访问，模块外也自然能访问

这正是可见性机制要区分的。

### ❌ 错误 3：报了 `E0603` 只会加 `pub`

正确问题应该是：这东西真的该暴露吗？

## 我的理解

- Rust 的默认私有很“工程”
- 它逼你先想清楚谁是内部实现，谁是对外接口
