## 定义

函数指针写作 `fn(...) -> ...`，表示“指向一个普通函数项的值”。它和闭包相关，但不是同一个概念。

## 作用

- 在 API 明确只接收普通函数时使用
- 避免引入更宽的闭包 trait 约束
- 在简单回调场景里保持签名直观

## 原理

普通函数项拥有固定类型，可以转成函数指针。零捕获闭包因为不依赖环境，也可以退化成函数指针；一旦闭包捕获了外部状态，就不再满足这个条件。

## 最小示例

```rust
fn apply_twice(value: i32, operation: fn(i32) -> i32) -> i32 {
    operation(operation(value))
}
```

## 注意点

- `fn` 是具体函数指针类型，不等于 `Fn`
- 零捕获闭包能传给 `fn`，捕获环境的闭包不行
- 如果想让 API 更灵活，通常还是 trait bound 更常见

## 常见错误

- 以为所有闭包都能自动转成 `fn`
- 混淆“函数项”“函数指针”“闭包值”
- 本来需要 `FnMut`，却把参数写死成 `fn`

## 我的理解

函数指针更像“最窄的可调用接口”。当你明确知道不需要捕获环境时，用它会让签名更简单。
