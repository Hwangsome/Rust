## 这一节解决什么问题

双向链表里，如果 `prev` 用 `Rc` 而不是 `Weak`，就会形成**引用环**：

```
A.next → B（Rc，A 强引用 B，B 的 strong_count = 1）
B.prev → A（Rc，B 强引用 A，A 的 strong_count = 1）

当外部把 A 和 B 都丢掉时：
  A.strong_count 降到 1（B.prev 还在引用）
  B.strong_count 降到 1（A.next 还在引用）
  → 都无法降到 0 → 内存永远不会释放 → 内存泄漏！
```

---

## `Weak<T>` 的作用

```
Rc::clone(&rc)    → 增加 strong_count（保持数据存活）
Rc::downgrade(&rc) → 创建 Weak，增加 weak_count（不保持数据存活）

Weak::upgrade()  → Option<Rc<T>>
  - 如果原数据还存活（strong_count > 0），返回 Some(Rc<T>)
  - 如果原数据已被 drop，返回 None
```

---

## 树结构的经典用法

```rust
use std::{cell::RefCell, rc::{Rc, Weak}};

#[derive(Debug)]
struct TreeNode {
    value: i32,
    parent: RefCell<Weak<TreeNode>>,     // Weak：不拥有父节点（防止环）
    children: RefCell<Vec<Rc<TreeNode>>>, // Rc：拥有子节点
}
```

- 父节点通过 `Rc` 拥有子节点（父 drop 时子跟着 drop）
- 子节点通过 `Weak` 引用父节点（不影响父节点的生命周期）

---

## 完整运行示例

```rust
pub fn run() {
    let leaf = Rc::new(TreeNode {
        value: 3,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![]),
    });

    let branch = Rc::new(TreeNode {
        value: 5,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![Rc::clone(&leaf)]),
    });

    // leaf 的 parent 用 Weak 引用 branch
    *leaf.parent.borrow_mut() = Rc::downgrade(&branch);

    println!("branch strong_count = {}", Rc::strong_count(&branch)); // 1
    println!("branch weak_count = {}",   Rc::weak_count(&branch));   // 1

    // 通过 Weak 访问父节点
    if let Some(parent) = leaf.parent.borrow().upgrade() {
        println!("leaf 的父节点值: {}", parent.value); // 5
    }

    println!("leaf strong_count = {}", Rc::strong_count(&leaf)); // 2（leaf 本身 + branch.children）
    println!("leaf weak_count = {}",   Rc::weak_count(&leaf));   // 0
}
```
