## 这一节解决什么问题

在 Java 里，要把"一段代码"传给另一个函数，需要创建匿名内部类或 lambda 表达式；在 Python 里有 `lambda`，但很受限。

Rust 的**闭包**（closure）是一种可以**捕获周围环境变量**的匿名函数。它比普通函数更灵活，因为：

```rust
let threshold = 5;  // 环境中的变量

let is_big = |x| x > threshold;  // 闭包捕获了 threshold
                                   // 普通函数做不到这一点
println!("{}", is_big(10));  // true
println!("{}", is_big(3));   // false
```

闭包在 Rust 里无处不在：`map`、`filter`、`sort_by`、线程闭包、回调函数——全都用到它。

---

## 一分钟结论

- 闭包语法：`|参数| 表达式` 或 `|参数| { 多行语句 }`
- 闭包**自动捕获**周围环境的变量（普通函数不能）
- 三种捕获方式对应三个 trait：`Fn`（不可变借用）、`FnMut`（可变借用）、`FnOnce`（获取所有权）
- `move` 关键字强制闭包通过**取得所有权**来捕获，常用于线程
- 闭包的类型由编译器推断，每个闭包是**唯一的匿名类型**
- 参数类型和返回类型可以省略，编译器从上下文推断

## 与其他语言对比

| 特性 | Java lambda | Python lambda | Go func literal | Rust closure |
|-----|------------|--------------|----------------|-------------|
| 捕获环境 | 是（effectively final 变量）| 是 | 是 | 是 |
| 捕获方式 | 只读（不可改变外部变量）| 读写（但有复杂语义）| 读写（任意） | 显式：`Fn`/`FnMut`/`FnOnce` |
| 多次调用 | 是 | 是 | 是 | `Fn`/`FnMut` 可以；`FnOnce` 只能一次 |
| 类型 | `Function<T,R>` 接口 | 任意函数对象 | 函数值 | 唯一匿名类型，实现 `Fn*` trait |
| 大小 | 无（栈上或堆上）| 对象 | 函数指针+捕获变量 | **零开销**（仅含捕获变量）|

## 核心概念与心智模型

### 闭包 vs 函数

```
普通函数：
  fn add(x: i32) -> i32 { x + 1 }
  ↑ 不能访问函数外的变量（只能通过参数或全局变量）

闭包：
  let delta = 5;
  let add_delta = |x| x + delta;  // delta 被"封闭"进闭包
  ↑ 通过捕获机制可以访问外部变量
  ↑ "closure" 的名字来自 "close over"（封闭变量）
```

### 三种 Fn* trait

```
Fn（不可变借用，可多次调用）
  let msg = String::from("hello");
  let greet = || println!("{msg}");
  greet(); // ✅
  greet(); // ✅（msg 只被借用，可以多次）

FnMut（可变借用，可多次调用但需要 &mut 闭包）
  let mut count = 0;
  let mut inc = || { count += 1; count };
  inc(); // 1
  inc(); // 2

FnOnce（取得所有权，只能调用一次）
  let name = String::from("rust");
  let consume = move || { drop(name); };
  consume(); // ✅
  // consume(); // ❌ name 已被消耗，闭包不能再调用

三者关系：Fn ⊂ FnMut ⊂ FnOnce
（实现了 Fn 的也实现了 FnMut 和 FnOnce）
```

## 详细原理

### 1. 闭包的基本语法

```rust
// 最简形式
let add = |a, b| a + b;
println!("{}", add(3, 4));  // 7

// 带类型注解（通常不需要，编译器推断）
let multiply = |a: i32, b: i32| -> i32 { a * b };

// 多行闭包
let complex = |x: i32| {
    let doubled = x * 2;
    let result = doubled + 1;
    result
};
```

### 2. 自动捕获（三种方式）

```rust
let text = String::from("hello");
let number = 42;

// Fn：不可变借用（默认，当闭包只读取时）
let print_text = || println!("{text}");
print_text();  // hello
println!("{text}");  // 仍然有效（只是借用）

// FnMut：可变借用（当闭包修改捕获变量时）
let mut count = 0;
let mut increment = || {
    count += 1;
    count
};
println!("{}", increment());  // 1
println!("{}", increment());  // 2
// println!("{count}"); // ❌ 此时 count 还被 increment 可变借用

// FnOnce：获取所有权（当闭包消耗捕获变量时）
let s = String::from("world");
let take_string = || {
    let result = s + "!";  // s 被 move 进这个操作
    result
};
let result = take_string();  // 只能调用一次
// take_string(); // ❌ 已经消耗了 s
println!("{result}");
```

### 3. `move` 关键字

```rust
let data = vec![1, 2, 3];

// 不加 move：闭包借用 data（生命周期受限）
let print_data = || println!("{data:?}");
print_data();
println!("{data:?}");  // data 仍然有效

// 加 move：闭包取得 data 的所有权（跨线程必须）
let thread_fn = move || {
    println!("thread sees: {data:?}");
};
std::thread::spawn(thread_fn).join().ok();
// println!("{data:?}"); // ❌ data 已被 move 进线程
```

### 4. 函数接收闭包

```rust
// 用泛型参数接收（最灵活，静态分派）
fn apply<F: Fn(i32) -> i32>(f: F, value: i32) -> i32 {
    f(value)
}

// 用 impl Trait（等价简写）
fn apply_impl(f: impl Fn(i32) -> i32, value: i32) -> i32 {
    f(value)
}

// 用 Box<dyn Fn>（动态分派，可存储在集合里）
fn apply_dyn(f: Box<dyn Fn(i32) -> i32>, value: i32) -> i32 {
    f(value)
}

// 返回闭包
fn make_adder(delta: i32) -> impl Fn(i32) -> i32 {
    move |x| x + delta  // move 是必需的，否则 delta 的生命周期不够
}
```

## 完整运行示例

```rust
fn apply<F>(list: &[i32], f: F) -> Vec<i32>
where
    F: Fn(i32) -> i32,
{
    list.iter().map(|&x| f(x)).collect()
}

fn apply_if<F, G>(list: &[i32], condition: F, transform: G) -> Vec<i32>
where
    F: Fn(i32) -> bool,
    G: Fn(i32) -> i32,
{
    list.iter()
        .filter(|&&x| condition(x))
        .map(|&x| transform(x))
        .collect()
}

fn make_multiplier(factor: i32) -> impl Fn(i32) -> i32 {
    move |x| x * factor
}

fn main() {
    let nums = vec![1, 2, 3, 4, 5, 6];

    println!("=== 基础闭包 ===");
    let doubled = apply(&nums, |x| x * 2);
    println!("doubled: {doubled:?}");

    let offset = 10;
    let with_offset = apply(&nums, |x| x + offset); // 捕获 offset
    println!("with_offset (+{offset}): {with_offset:?}");
    println!();

    println!("=== FnMut：有状态的闭包 ===");
    let mut sum = 0;
    let mut running_sum = Vec::new();
    for &n in &nums {
        sum += n;
        running_sum.push(sum);
    }
    println!("running sum: {running_sum:?}");
    println!();

    println!("=== 工厂函数返回闭包 ===");
    let triple = make_multiplier(3);
    let times_seven = make_multiplier(7);
    println!("triple(5) = {}", triple(5));
    println!("times_seven(5) = {}", times_seven(5));
    println!();

    println!("=== 复合条件过滤 ===");
    let threshold = 3;
    let factor = 100;
    let result = apply_if(
        &nums,
        |x| x > threshold,   // 捕获 threshold
        |x| x * factor,       // 捕获 factor
    );
    println!("大于{threshold}的元素 * {factor}: {result:?}");
    println!();

    println!("=== move 闭包（线程场景）===");
    let data = vec!["rust", "is", "amazing"];
    let handle = std::thread::spawn(move || {
        println!("thread: {data:?}");
        data.len()  // 返回值
    });
    println!("main: waiting for thread...");
    let len = handle.join().unwrap();
    println!("thread returned: {len}");
}
```

**预期输出**：

```text
=== 基础闭包 ===
doubled: [2, 4, 6, 8, 10, 12]
with_offset (+10): [11, 12, 13, 14, 15, 16]

=== FnMut：有状态的闭包 ===
running sum: [1, 3, 6, 10, 15, 21]

=== 工厂函数返回闭包 ===
triple(5) = 15
times_seven(5) = 35

=== 复合条件过滤 ===
大于3的元素 * 100: [400, 500, 600]

=== move 闭包（线程场景）===
main: waiting for thread...
thread: ["rust", "is", "amazing"]
thread returned: 3
```

## 编译器错误分析

### ❌ E0525：`FnOnce` 闭包不能多次调用

```rust
let s = String::from("hello");
let consume = || drop(s); // FnOnce（drop 消耗了 s）
consume();
consume(); // ❌
```

```text
error[E0525]: expected a closure that implements the `Fn` trait,
              but this closure only implements `FnOnce`
  |
3 | consume();
4 | consume(); // ❌
  | ^^^^^^^^^ cannot call `FnOnce` closure more than once
```

**修复**：如果需要多次调用，不要消耗捕获的变量；或者用 `clone()`。

### ❌ E0597：闭包借用了不够长寿的变量

```rust
let result;
{
    let local = String::from("hello");
    result = || println!("{local}"); // ❌ local 会在花括号后被 drop
}
result();
```

```text
error[E0597]: `local` does not live long enough
```

**修复**：用 `move` 把 `local` 的所有权搬进闭包：

```rust
let result;
{
    let local = String::from("hello");
    result = move || println!("{local}"); // ✅ local 被 move 进闭包
}
result();
```

## 实际工程场景

### 1. 事件回调（GUI/异步框架）

```rust
struct Button {
    label: String,
    on_click: Box<dyn Fn()>,
}

impl Button {
    fn new(label: &str, on_click: impl Fn() + 'static) -> Self {
        Button { label: label.into(), on_click: Box::new(on_click) }
    }
    fn click(&self) { (self.on_click)(); }
}

let click_count = std::sync::Arc::new(std::sync::Mutex::new(0));
let count_clone = click_count.clone();

let btn = Button::new("OK", move || {
    let mut c = count_clone.lock().unwrap();
    *c += 1;
    println!("点击次数: {c}");
});

btn.click();
btn.click();
```

### 2. 策略模式

```rust
fn sort_by_strategy<T, F>(data: &mut Vec<T>, strategy: F)
where
    F: Fn(&T, &T) -> std::cmp::Ordering,
{
    data.sort_by(strategy);
}

let mut products = vec![("apple", 1.5), ("banana", 0.5), ("cherry", 3.0)];

// 按名字排序
sort_by_strategy(&mut products, |a, b| a.0.cmp(b.0));
println!("{products:?}");

// 按价格排序
sort_by_strategy(&mut products, |a, b| a.1.partial_cmp(&b.1).unwrap());
println!("{products:?}");
```

## 性能影响

- 闭包的内存布局：只含被捕获的变量，大小等于捕获变量之和
- 没有捕获变量的闭包大小为 0（和函数指针一样高效）
- `impl Fn`（静态分派）：可以内联，零开销
- `Box<dyn Fn>`（动态分派）：一次间接跳转（约 2-4ns 额外开销）

```rust
let lambda = || 42;  // 没有捕获，大小 = 0
println!("{}", std::mem::size_of_val(&lambda)); // 0

let x = 100_i64;
let capturing = || x + 1;  // 捕获了 x（8字节），大小 = 8
println!("{}", std::mem::size_of_val(&capturing)); // 8
```

## 注意点与陷阱

### 陷阱 1：闭包捕获是按需的，不是全量的

```rust
let a = String::from("a");
let b = String::from("b");
let c = 100_i32;

let f = || println!("{a} {c}");  // 只捕获了 a 和 c，没有 b
f();
println!("{b}");  // ✅ b 仍然有效（没被捕获）
```

### 陷阱 2：循环中的闭包捕获变量（经典陷阱）

```rust
let mut closures = Vec::new();

// ❌ 所有闭包都捕获同一个 i 的引用
for i in 0..3 {
    closures.push(move || println!("{i}")); // move 才对！
}
// 如果不加 move，i 是循环变量，生命周期有问题

for f in &closures {
    f();  // 0, 1, 2（每个闭包独立捕获了不同的 i）
}
```

## 我的理解与记忆方法

**Fn* trait 的选择规则**：

```
接收闭包时：
  只需要只读 → Fn（最宽松，接受最多类型）
  需要可变状态 → FnMut
  只调用一次 → FnOnce（最宽，因为 Fn ⊂ FnMut ⊂ FnOnce）

返回闭包时：
  不消耗外部变量 → impl Fn
  需要跨线程/跨生命周期 → move + impl Fn + 'static
```

**快速区分**：

```
闭包用了什么操作？
  读取外部变量   → Fn（不可变借用）
  修改外部变量   → FnMut（可变借用）
  消耗外部变量   → FnOnce（获取所有权）
```
