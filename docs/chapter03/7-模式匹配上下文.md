## 定义

模式匹配是用某种结构去描述“这个值该如何被识别或拆解”的机制。

## 作用

- 按分支处理不同形态的数据
- 直接解构数据
- 在语法上减少样板代码

## 原理

### 1. `match` 是最完整的模式匹配入口

适合需要覆盖多个分支的场景。

### 2. `if let` 适合只关心一个分支

当你只想处理某个成功模式时，它通常比完整 `match` 更简洁。

### 3. `while let` 适合“匹配成功就继续”

当前例子里不断 `pop()` 栈就是最典型场景。

### 4. `let` 本身也支持解构

```rust
let (a, b) = (10, 20);
```

### 5. 函数参数也是模式位置

```rust
fn print_coords((x, y): (i32, i32)) { ... }
```

## 最小示例

```rust
fn main() {
    let maybe_value = Some(5);

    if let Some(value) = maybe_value {
        println!("{}", value);
    }

    let (a, b) = (10, 20);
    println!("{} {}", a, b);
}
```

## 注意点

### 1. `if let` 更简洁，但也更少信息

如果你需要处理多个分支，`match` 通常更清楚。

### 2. 模式匹配不是“只在 enum 上用”

tuple、struct、字面量、函数参数都能参与模式。

### 3. 解构会影响绑定方式

你拆出来的每个名字，本质上都在创建新的绑定。

## 常见错误

### ❌ 错误 1：把模式匹配只理解成 `match`

这会错过很多 Rust 语法的一致性。

### ❌ 错误 2：该用 `match` 的地方硬用 `if let`

简洁如果让分支意图变模糊，就不值。

### ❌ 错误 3：只看表面语法，不看“这其实是在解构值”

这会让很多绑定行为显得突然。

## 我的理解

- 模式匹配不是一个语句，而是一套贯穿语法的思想
- 一旦理解了“参数、`let`、`if let` 都在做模式匹配”，很多代码会突然顺起来
