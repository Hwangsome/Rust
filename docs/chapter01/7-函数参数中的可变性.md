## 一分钟结论

- 参数默认也是不可变绑定
- `mut` 作用在函数内部这个参数绑定上
- 参数是否可变，不等于调用方的变量就会被原地修改
- 对 `Copy` 类型和借用类型，要分开理解


## 最小示例

```rust
fn increase(mut value: i32) -> i32 {
    value += 1;
    value
}
```

这里的 `mut` 只表示：函数体内部允许修改 `value` 这个参数绑定。

## 核心理解

### 1. 参数是新的绑定

函数被调用时，参数名不是“沿用调用方变量名”，而是在函数作用域里创建的新绑定。

### 2. `mut` 只影响函数内部

```rust
fn increase(mut value: i32) -> i32 {
    value += 1;
    value
}
```

这不会把外部变量自动改掉。  
如果你想让外部状态也被修改，通常要进一步学习可变借用。

## 常见误解

### ❌ “参数写了 mut，外部变量一定会变”

这不成立。  
对于 `i32` 这类 `Copy` 类型，函数里改的是参数副本。
