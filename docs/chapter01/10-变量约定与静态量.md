# 10. 变量约定与静态量

> 类型：**Study note**
> 关键词：`snake_case`、`SCREAMING_SNAKE_CASE`、`const`、`static`
> 上一篇：[9. 注释与打印](./9-注释与打印.md)
> 下一篇：[11. 编译器指令](./11-编译器指令.md)

## 一分钟结论

- 局部变量通常使用 **`snake_case`**
- 常量和静态量通常使用 **`SCREAMING_SNAKE_CASE`**
- `const` 更像“编译期常量”
- `static` 更像“有固定存储位置的全局值”
- `_unused` 这类前导下划线，不是装饰，而是在和编译器沟通“这个变量我暂时故意不用”

## 证据来源

- 章节入口：[chapters/chapter01/src/main.rs](../../chapters/chapter01/src/main.rs)
- 对应模块：[variable_conventions_and_statics.rs](../../chapters/chapter01/src/variable_conventions_and_statics.rs)

当前示例里出现了这些点：

```rust
static WELCOME: &str = "Welcome to Rust";
const PI: f32 = 3.14;

let _unused_number = 45;
let large_value = 40_000;
let radius = 2.0_f32;
```

## 定义

这一节包含两类内容：

1. **命名约定**
   让代码更容易读、更容易被团队统一维护。
2. **全局命名值**
   也就是 `const` 和 `static`，它们都能在更大作用域下被使用，但语义不同。

## 作用

### 命名约定的作用

- 让代码一眼看出“这是什么类型的名字”
- 降低阅读成本
- 减少团队里“每个人写一套风格”的混乱
- 更好地配合 Rust 的 lint 和编译器提示

### `const` / `static` 的作用

- `const` 适合表示**逻辑上不会变化的固定值**
- `static` 适合表示**程序整个生命周期里都存在的固定全局数据**

## 原理

### 1. 命名约定本身就是一种低成本元信息

Rust 没有强制你只能用某一种命名风格，但社区和工具链有稳定共识：

| 场景 | 常见写法 | 例子 |
| --- | --- | --- |
| 局部变量 / 函数名 | `snake_case` | `large_value` |
| 常量 / 静态量 | `SCREAMING_SNAKE_CASE` | `WELCOME`, `MAX_SIZE` |
| 类型名 | `PascalCase` | `Point`, `User` |

这种约定带来的收益很直接：看到名字就先知道它大概处于哪一层语义。

### 2. 前导下划线是在“压制未使用告警”

```rust
let _unused_number = 45;
```

Rust 编译器默认会提醒未使用变量，因为：

- 这通常是 Bug 的信号
- 也可能说明你写了一半逻辑没接完

但有时你就是临时不想用，这时前导下划线是在明确告诉编译器：

- 我知道它现在没用
- 这是有意的

### 3. `const` 和 `static` 不是一回事

#### `const`

```rust
const PI: f32 = 3.14;
```

特点：

- 必须显式标注类型
- 必须在编译期可求值
- 更像“一个到处都能引用的固定值”

#### `static`

```rust
static WELCOME: &str = "Welcome to Rust";
```

特点：

- 有固定存储位置
- 生命周期通常覆盖整个程序运行期
- 更接近真正的全局数据

所以：

| 对比项 | `const` | `static` |
| --- | --- | --- |
| 语义重点 | 固定值 | 全局存储 |
| 是否更像内联常量 | 是 | 否 |
| 是否有固定地址感 | 弱 | 强 |

## 最小示例

```rust
static WELCOME: &str = "Welcome to Rust";
const PI: f32 = 3.14;

fn main() {
    let _unused_number = 45;
    let radius = 2.0_f32;
    let large_value = 40_000;

    println!("large_value = {}", large_value);
    println!("circle area ≈ {}", PI * radius * radius);
    println!("WELCOME = {}", WELCOME);
}
```

## 注意点

### 1. `const` 必须写类型

```rust
const PI: f32 = 3.14;
```

这里不能像普通 `let` 那样完全依赖推断。

### 2. `static` 不等于“到处乱放全局状态”

当前章节只展示只读 `static`。如果你把全局共享状态做成可变，复杂度会快速升高，通常也会引入额外约束。

### 3. 数字里的下划线只是可读性技巧

```rust
let large_value = 40_000;
```

这和 `40000` 没有语义差别，只是更容易读。

## 常见错误

### ❌ 错误 1：把 `const` 和普通变量当成一样

`const` 不是“写在函数外的 `let`”，它有更严格的规则。

### ❌ 错误 2：把前导下划线当成命名习惯，而不是语义信号

如果变量本来就会使用，不要为了“安静”随手都加 `_`。这会降低代码可读性。

### ❌ 错误 3：所有全局固定值都用 `static`

很多场景里，`const` 已经足够，而且语义更清楚。

## 我的理解

- 命名约定不是形式主义，它是在给阅读者减少理解负担
- `const` 解决“固定值”
- `static` 解决“全局存在的值”
- `_unused` 是和编译器协作，而不是在逃避规则
