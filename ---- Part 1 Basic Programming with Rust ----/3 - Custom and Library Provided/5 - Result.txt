fn check_student_get_grade(
    student_name: &String,
    student_db: &Vec<Student>,
) -> Result<Option<u32>, String> {
    for student in student_db {
        if student.name == *student_name {
            return Ok(student.grade);
        }
    }
    Err(String::from("Student not found"))
}

/*
--------------------------------------------------------------------------------
Concept                   | Explanation                                                                                                                                                                                                                                                                                                             |
------------------------- | ----------------------------------------------------
Result Enum               | Represents success (Ok) or failure (Err) outcomes.
                          | Encourages explicit and safe error handling.
                          | Reducing relaince on exceptions handling or error codes.
                          | Can be used in functions that may fail.

Type Parameters in Result | Result<T, E> has two Generic type parameters.
                          | T for success and E for error.
                          | Ok(T) carries the success value.
                          | Err(E) carries error information.

Option vs Result          | Option represents presence or absence of a value.
                          | Result represents success or failure.
--------------------------------------------------------------------------------
 */
