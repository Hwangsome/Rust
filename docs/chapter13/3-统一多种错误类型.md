## 这一节解决什么问题

一个函数调用了多个库，它们返回不同类型的错误。如何让 `?` 正常工作？

```rust
fn load_and_parse(path: &str) -> Result<i32, ???> {
    let content = fs::read_to_string(path)?;  // io::Error
    let n: i32 = content.trim().parse()?;      // ParseIntError
    // 这里 ??? 应该是什么类型？
}
```

三种解决方案，各有适用场景：

1. **自定义错误 enum + `From`**：精确类型，库代码推荐
2. **`Box<dyn Error>`**：快速原型，应用代码可用
3. **`anyhow::Error`**（crate）：应用层最省心

---

## 方案 A：自定义错误 enum

```rust
use std::{io, num::ParseIntError};

#[derive(Debug)]
enum MyError {
    Io(io::Error),
    Parse(ParseIntError),
    Custom(String),
}

// 实现 From，让 ? 自动转换
impl From<io::Error> for MyError {
    fn from(e: io::Error) -> Self { MyError::Io(e) }
}

impl From<ParseIntError> for MyError {
    fn from(e: ParseIntError) -> Self { MyError::Parse(e) }
}

impl std::fmt::Display for MyError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            MyError::Io(e) => write!(f, "IO 错误: {e}"),
            MyError::Parse(e) => write!(f, "解析错误: {e}"),
            MyError::Custom(s) => write!(f, "错误: {s}"),
        }
    }
}

fn load_number(path: &str) -> Result<i32, MyError> {
    let content = std::fs::read_to_string(path)?;  // → MyError::Io
    let n: i32 = content.trim().parse()?;           // → MyError::Parse
    if n > 100 {
        return Err(MyError::Custom(format!("{n} 超过了最大值 100")));
    }
    Ok(n)
}
```

## 方案 B：`Box<dyn Error>` 快速方案

```rust
use std::error::Error;

fn load_number_quick(path: &str) -> Result<i32, Box<dyn Error>> {
    let content = std::fs::read_to_string(path)?;  // 任何 Error → Box<dyn Error>
    let n: i32 = content.trim().parse()?;
    Ok(n)
}

// 优点：简单，不需要定义错误 enum
// 缺点：调用方无法按错误类型分支处理
```

## 完整运行示例

```rust
use std::{io, num::ParseIntError};

#[derive(Debug)]
enum TemperatureError {
    InvalidFormat(ParseIntError),
    OutOfRange { value: i32, min: i32, max: i32 },
    IoError(io::Error),
}

impl From<ParseIntError> for TemperatureError {
    fn from(e: ParseIntError) -> Self { TemperatureError::InvalidFormat(e) }
}

impl From<io::Error> for TemperatureError {
    fn from(e: io::Error) -> Self { TemperatureError::IoError(e) }
}

impl std::fmt::Display for TemperatureError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            Self::InvalidFormat(e) => write!(f, "格式无效: {e}"),
            Self::OutOfRange { value, min, max } =>
                write!(f, "温度 {value} 超出范围 [{min}, {max}]"),
            Self::IoError(e) => write!(f, "IO 错误: {e}"),
        }
    }
}

fn parse_temperature(s: &str) -> Result<i32, TemperatureError> {
    let temp: i32 = s.trim().parse()?;  // ParseIntError → TemperatureError
    if temp < -100 || temp > 100 {
        return Err(TemperatureError::OutOfRange { value: temp, min: -100, max: 100 });
    }
    Ok(temp)
}

fn main() {
    println!("=== 统一多种错误类型 ===");
    let test_cases = ["25", "  -5  ", "abc", "200", "0"];

    for input in test_cases {
        match parse_temperature(input) {
            Ok(t) => println!("  ✅ {:?} → {}°C", input, t),
            Err(TemperatureError::InvalidFormat(_)) =>
                println!("  ❌ {:?} → 不是有效数字", input),
            Err(TemperatureError::OutOfRange { value, min, max }) =>
                println!("  ❌ {:?} → {value} 超出 [{min}, {max}]", input),
            Err(e) => println!("  ❌ {:?} → {e}", input),
        }
    }
    println!();

    println!("=== map_err：手动转换错误类型 ===");
    fn parse_with_context(s: &str, field: &str) -> Result<i32, String> {
        s.trim().parse::<i32>()
            .map_err(|e| format!("字段 '{field}' 解析失败: {e}"))
    }

    for (field, val) in [("age", "30"), ("score", "bad"), ("count", "-5")] {
        match parse_with_context(val, field) {
            Ok(n) => println!("  {field} = {n}"),
            Err(e) => println!("  错误: {e}"),
        }
    }
}
```

## 选型建议

```
库代码（让调用方处理）：
  → 自定义错误 enum + impl Error
  → 可以用 thiserror 减少样板代码

应用代码（只打印/记录错误）：
  → Box<dyn Error> 或 anyhow::Error
  → 用 .context("...") 添加上下文

两者都有：
  → 内部用精确类型，对外层返回 Box<dyn Error>
```
