## 定义

基准测试（benchmark）是对性能特征做测量，而不是验证逻辑是否正确。

## 作用

- 比较不同实现的速度
- 观察优化是否有效
- 为性能问题提供量化依据

## 原理

当前最小示例使用：

```rust
let start = Instant::now();
let sum: u64 = (1..=10_000).sum();
let elapsed = start.elapsed();
```

这是最朴素的计时流程：

1. 记录开始时间
2. 执行工作负载
3. 计算耗时

## 最小示例

```rust
use std::time::Instant;

fn main() {
    let start = Instant::now();
    let sum: u64 = (1..=10_000).sum();
    let elapsed = start.elapsed();

    println!("sum = {}", sum);
    println!("elapsed = {:?}", elapsed);
}
```

## 注意点

### 1. benchmark 不是普通测试

功能测试问“对不对”，benchmark 问“快不快”。

### 2. 单次计时只能算粗略观察

更正式的性能分析通常需要：

- 多次重复
- 预热
- 统计分析

### 3. `Instant` 是起点，不是终点

它很适合教学和最小实验，但不等于完整 benchmark 方案。

## 常见错误

### ❌ 错误 1：拿单次计时结果当最终结论

### ❌ 错误 2：功能还没稳定就急着 benchmark

### ❌ 错误 3：把 benchmark 和 correctness test 混成同一类东西

## 我的理解

- benchmark 的真正价值不是给出一个漂亮数字
- 而是建立“性能也是可验证、可比较、可复盘的”工程习惯
