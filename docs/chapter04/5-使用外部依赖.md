## 一分钟结论

- 在 `Cargo.toml` 的 `[dependencies]` 段添加依赖
- `cargo add package_name` 命令行快速添加
- SemVer：`"1"` 允许 1.x.y 任意版本；`"=1.2.3"` 锁死精确版本
- `features = ["feature_name"]` 启用可选功能
- `dev-dependencies`：只在测试和开发时用的依赖

---

## Cargo.toml 依赖写法

```toml
[dependencies]
# 基础写法（接受兼容版本）
serde = "1"                    # 接受 1.0.0 及以上（不跨大版本）
tokio = "1.0"                  # 同上（1.0.x 开始）

# 精确版本
regex = "=1.10.2"              # 只能用这个精确版本

# 范围
log = ">=0.4, <0.5"

# 启用 features
serde = { version = "1", features = ["derive"] }
tokio = { version = "1", features = ["full"] }

# 可选依赖（需要通过 feature 启用）
serde_json = { version = "1", optional = true }

# 从 Git
my-lib = { git = "https://github.com/user/repo", branch = "main" }

# 本地路径（workspace 内常用）
utils = { path = "../utils" }

[dev-dependencies]
# 只在 cargo test / cargo bench 时使用
criterion = "0.5"
tempfile = "3"
```

## 依赖版本的含义（SemVer）

```
"1"    = "^1"  → 允许 1.0.0 ~ <2.0.0
"1.2"  = "^1.2" → 允许 1.2.0 ~ <2.0.0
"0.4"  = "^0.4" → 允许 0.4.0 ~ <0.5.0（0.x 更保守）
"=1.0" → 严格等于 1.0
"~1.2" → 允许 1.2.x（只更新 patch 版本）
```

## 添加依赖的命令

```bash
# 添加最新兼容版本
cargo add serde

# 添加并启用 feature
cargo add serde --features derive

# 添加特定版本
cargo add tokio@1.35

# 添加为 dev dependency
cargo add --dev criterion

# 移除依赖
cargo rm serde
```

## 完整运行示例

```rust
pub fn run() {
    println!("=== 外部依赖使用流程 ===");
    println!();
    println!("1. 添加依赖到 Cargo.toml:");
    println!("   cargo add serde --features derive");
    println!("   或手动编辑 Cargo.toml:");
    println!("   [dependencies]");
    println!("   serde = {{ version = \"1\", features = [\"derive\"] }}");
    println!();
    println!("2. 在代码里 use:");
    println!("   use serde::{{Serialize, Deserialize}};");
    println!();
    println!("3. 使用:");
    println!("   #[derive(Serialize, Deserialize)]");
    println!("   struct Config {{ name: String, port: u16 }}");
    println!();
    println!("本 chapter 为了保持简单没有引入外部依赖。");
    println!("chapter11 引入了 anyhow 和 thiserror，可以参考那里的使用方式。");
}
```

## 版本管理最佳实践

```
库（library crate）：
  - 依赖版本范围尽量宽松（让用户选择具体版本）
  - Cargo.lock 不提交到 Git

应用（binary crate）：
  - 可以指定较精确的版本
  - Cargo.lock 提交到 Git（保证构建可复现）

安全更新：
  cargo update            # 更新到允许的最新版本
  cargo update -p serde   # 只更新 serde
  cargo audit             # 检查已知安全漏洞
```
