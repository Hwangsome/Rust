## 这一节解决什么问题

Rust 的借用检查器如何知道某个引用是否"悬垂"（指向已经被释放的内存）？它靠的是追踪每个值的**生命周期**——从创建到被 drop 的这段时间。

这是理解 Rust 内存安全的根基。生命周期不是 Rust 发明的概念——每种语言里值都有生命周期，只不过 Rust 把它显式化，让编译器来验证"引用不能活过它指向的值"。

---

## 一分钟结论

- 每个值都有生命周期：从创建到离开作用域（被 drop）
- 引用的生命周期必须**不超过**被引用值的生命周期
- **NLL（Non-Lexical Lifetimes）**：引用的有效期到"最后一次使用"为止
- 悬垂引用（dangling reference）= 引用的值已经被 drop → 编译器通过 E0597 阻止

---

## 与其他语言对比

| 语言 | 如何处理悬垂引用 |
|-----|-------------|
| C | 不管：释放后仍可用（未定义行为）|
| C++ | 析构函数 + 智能指针部分解决；仍可能出错 |
| Java | GC：只要有引用，对象就不被回收（不可能悬垂）|
| Go | GC：同 Java |
| Rust | 编译期检查：**不可能在运行时出现悬垂引用** |

## 核心概念与心智模型

```
生命周期：值从"创建"到"drop"的时间段

fn main() {
    let x = 5;        // ← x 的生命周期开始
    let r = &x;       // ← r 是对 x 的借用，r 的生命周期必须 ≤ x 的
    println!("{r}");  // r 最后一次使用（NLL：r 的生命周期到这里结束）
    // ← x 的生命周期继续到这里结束
}                     // ← x 被 drop，生命周期结束

悬垂引用（Rust 会在编译期阻止）：
fn bad() -> &i32 {
    let x = 5;
    &x           // ← x 在函数结束时 drop，返回悬垂引用
}               // ← x 的生命周期结束，但引用还没消失
```

## 详细原理

### 1. 基本生命周期规则

```rust
fn main() {
    let r;                    // r 声明，但未初始化

    {
        let x = 5;            // x 的生命周期开始（内层块）
        r = &x;               // r 借用 x
    }                         // x 的生命周期结束，x 被 drop

    // println!("{r}"); // ❌ E0597: x 已经被 drop，r 是悬垂引用
}
```

### 2. NLL（Non-Lexical Lifetimes）

```rust
let mut data = vec![1, 2, 3];
let first = &data[0];     // 不可变借用开始
println!("{first}");       // first 最后一次使用（NLL：借用到这里结束）
// ↑ 注意：first 的借用在 println! 之后就结束了（NLL）

data.push(4);             // ✅ NLL 允许这里修改（first 的借用已结束）
println!("{data:?}");     // [1, 2, 3, 4]
```

### 3. 所有权转移结束生命周期

```rust
let s1 = String::from("hello");
let r = &s1;              // r 借用 s1
println!("{r}");          // r 最后一次使用（NLL）

let s2 = s1;              // ✅ s1 被 move 给 s2，r 的借用已结束
println!("{s2}");
```

## 完整运行示例

```rust
fn main() {
    println!("=== 基本生命周期 ===");
    {
        let value = 42;
        let reference = &value;
        println!("reference = {reference}");
        // reference 和 value 在这个块结束时一起失效
    }

    println!();

    println!("=== NLL：借用活到最后一次使用 ===");
    let mut v = vec![10, 20, 30];

    let first = &v[0];
    println!("first element = {first}");
    // first 最后一次使用，借用结束

    v.push(40);  // ✅ 允许，因为 first 的借用已经结束
    println!("after push: {v:?}");

    println!();

    println!("=== 生命周期与所有权转移 ===");
    let s = String::from("original");
    {
        let borrow = &s;
        println!("借用: {borrow}");
        // borrow 最后一次使用，借用结束
    }
    let s2 = s;  // ✅ 可以 move，借用已经结束
    println!("新所有者: {s2}");

    println!();

    println!("=== Drop 顺序（后声明的先 drop）===");
    struct Announce(i32);
    impl Drop for Announce {
        fn drop(&mut self) {
            println!("  dropping Announce({})", self.0);
        }
    }

    let _a = Announce(1);
    let _b = Announce(2);
    let _c = Announce(3);
    println!("  三个对象创建完毕");
    // 函数结束时：先 drop _c（3），再 _b（2），再 _a（1）
}
```

## 编译器错误分析

### ❌ E0597：引用的值比引用先消失

```rust
fn main() {
    let r;
    {
        let x = 5;
        r = &x;  // ❌ x 会在这个块结束时被 drop
    }
    println!("{r}"); // r 指向已释放的内存
}
```

```text
error[E0597]: `x` does not live long enough
  |
4 |         r = &x;
  |              ^ borrowed value does not live long enough
5 |     }
  |     - `x` dropped here while still borrowed
6 |     println!("{r}");
  |               - borrow later used here
```

**修复**：让 x 的生命周期与 r 一样长：

```rust
fn main() {
    let x = 5;  // 移到外层，生命周期覆盖 r 的使用范围
    let r = &x;
    println!("{r}");
}
```
