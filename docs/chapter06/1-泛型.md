## 定义

泛型是把“具体类型”变成“类型参数”的机制，例如 `Point<T, U>`。同一套结构和逻辑可以在编译期被替换成多个具体版本。

## 作用

- 消除重复代码
- 让 API 更通用
- 保留静态类型检查，不退化成动态类型容器

## 原理

Rust 的泛型不是运行时反射。编译器会在需要时把泛型实例化成具体类型版本，这通常被称为单态化。对使用者来说写的是一套模板，对编译器来说最后仍然会落成具体代码。

## 最小示例

```rust
#[derive(Debug)]
struct Point<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point<T, U> {
    fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {
        Point { x: self.x, y: other.y }
    }
}
```

## 注意点

- 泛型解决的是“类型可变”，不是“行为自动存在”
- 一旦泛型函数需要调用某些方法，通常就要继续加 trait bound
- 不同字段位置可以绑定不同类型，不一定只能写一个 `T`

## 常见错误

- 以为泛型等于运行时多态
- 以为用了泛型就不需要 trait
- 把所有类型参数都写成一个 `T`，结果无意中限制了灵活性

## 我的理解

泛型本质上是在说：这段代码的结构是稳定的，变化的是“里面装的类型”。它先解决“重复”的问题，后面 trait 再解决“这些类型至少得会什么”的问题。
