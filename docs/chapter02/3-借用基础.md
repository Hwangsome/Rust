# 3. 借用基础

> 类型：**Study note**
> 关键词：borrow、reference、`&T`、`&mut T`
> 上一篇：[2. 函数中的所有权](./2-函数中的所有权.md)
> 下一篇：[4. 函数中的借用](./4-函数中的借用.md)

## 一分钟结论

- 借用就是：**使用值，但不拿走它**
- `&T` 是不可变借用
- `&mut T` 是可变借用
- Rust 的核心规则可以记成：
  - **多个不可变借用**
  - **或一个可变借用**
  - **但不能同时混用**

## 证据来源

- 对应模块：[borrowing_basics.rs](../../chapters/chapter02/src/borrowing_basics.rs)
- 运行章节：`cargo run -p chapter02`

关键输出：

```text
immutable refs => [4, 5, 6], [4, 5, 6]
mutable ref after immutable refs end => [4, 5, 6, 7]
```

补充实验：借用冲突会报错：

```text
error[E0502]: cannot borrow `v` as mutable because it is also borrowed as immutable
```

## 定义

借用是通过引用访问值的一种方式。它允许你：

- 看值
- 或修改值
- 但不取得所有权

## 作用

借用解决的问题是：

- 不想 move，又想访问数据
- 不想 clone，又想复用数据
- 希望多个地方协作访问同一份值

## 原理

### 1. 不可变借用可以有多个

```rust
let ref1 = &values;
let ref2 = &values;
```

因为它们都只是读，不会互相破坏状态。

### 2. 可变借用只能独占

```rust
let ref3 = &mut values;
ref3.push(7);
```

因为它可能修改底层数据，所以 Rust 要求这一刻不能再有其他引用同时使用这份值。

### 3. 规则本质上是在避免“读写混乱”

这套规则背后的直觉非常工程化：

- 多个读者没问题
- 一个写者也没问题
- 读写同时发生，最危险

## 最小示例

```rust
fn main() {
    let mut values = vec![4, 5, 6];

    let ref1 = &values;
    let ref2 = &values;
    println!("{:?} {:?}", ref1, ref2);

    let ref3 = &mut values;
    ref3.push(7);
    println!("{:?}", ref3);
}
```

## 注意点

### 1. 借用规则看的是“使用区间”，不是肉眼上的几行距离

如果一个不可变借用后面还在使用，那么新的可变借用就可能冲突。

### 2. 可变借用不是“可变变量”的翻版

- `mut` 修饰绑定
- `&mut` 修饰借用方式

这是两层不同概念。

### 3. 借用不是复制

引用只是临时访问入口，不会把底层值复制一份出来。

## 常见错误

### ❌ 错误 1：同时保留不可变借用和可变借用

```rust
let r1 = &v;
let r2 = &mut v;
```

常见报错：

```text
cannot borrow as mutable because it is also borrowed as immutable
```

### ❌ 错误 2：把引用当成独立值副本

引用不是 clone，它只是访问路径。

### ❌ 错误 3：把 `mut` 和 `&mut` 混成一回事

二者都和“可变”有关，但语义层不同。

## 我的理解

- 借用本质上是在所有权规则上加了“临时通行证”
- 这套规则严格，但直觉其实很像多人协作：
  - 大家一起看可以
  - 一个人改也可以
  - 一边看一边改最危险
