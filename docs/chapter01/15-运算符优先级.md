# 15. 运算符优先级

> 类型：**Study note**
> 关键词：precedence、parentheses、expression
> 上一篇：[14. 结合性与运算符重载](./14-结合性与运算符重载.md)

## 一分钟结论

- 运算符优先级决定表达式默认先算哪一部分
- 不确定时，**优先加括号**
- “能靠脑补读懂”和“代码可维护”不是一回事

## 定义

运算符优先级是表达式里的默认求值顺序规则。

比如：

- 乘除通常优先于加减
- 括号可以显式改写默认顺序

## 作用

优先级规则的作用是：

- 让表达式在没有括号时也能有确定含义
- 避免每个式子都必须手工标出完整分组

但从工程实践看，括号通常仍然值得多用一点。

## 原理

### 1. 默认规则会先算更高优先级运算

```rust
2 + 3 * 4
```

默认相当于：

```rust
2 + (3 * 4)
```

所以结果是 `14`。

### 2. 括号优先级最高，能强制分组

```rust
(2 + 3) * 4
```

这里会先算括号里的 `2 + 3`，结果变成 `20`。

### 3. 优先级解决“谁先算”，括号解决“我想怎么读”

即使你知道默认规则，为了让读者更快理解，很多时候仍值得加括号。

## 最小示例

```rust
fn main() {
    let default_order = 2 + 3 * 4;
    let grouped_order = (2 + 3) * 4;

    println!("2 + 3 * 4 = {}", default_order);
    println!("(2 + 3) * 4 = {}", grouped_order);
}
```

## 注意点

### 1. 优先级和结合性不是一个概念

- 优先级：先算谁
- 结合性：同优先级时怎么分组

### 2. 可读性优先于“少写括号”

工程代码不是数学竞赛题。多几个括号，常常能省掉很多脑力。

### 3. 一旦表达式变复杂，建议主动分步

如果你已经需要解释半天一个表达式，通常说明它可以拆成中间变量。

## 常见错误

### ❌ 错误 1：默认脑补错了求值顺序

尤其是在加减和乘除混写时。

### ❌ 错误 2：为了“简洁”拒绝加括号

简洁如果建立在读者要停下来心算的前提上，通常不值。

### ❌ 错误 3：把优先级、结合性、重载混在一起

这三者都影响表达式，但层次不同：

- 优先级：先后
- 结合性：分组
- 重载：符号对类型意味着什么

## 我的理解

- 优先级是语言默认规则
- 括号是程序员主动表达意图的工具
- 真正成熟的写法不是“记住一整张优先级表”，而是知道什么时候别让别人猜
