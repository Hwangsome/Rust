## 这一节解决什么问题

表达式 `2 + 3 * 4` 等于 `20` 还是 `14`？取决于 `*` 比 `+` 优先级更高——结果是 `14`。

运算符优先级决定了"没有括号时，哪个运算符先做"。掌握最常用的几条规则，遇到复杂表达式时加括号明确意图。

## 一分钟结论

**记忆规则（从高到低）**：

1. 函数调用、方法调用、字段访问（`.`、`()`、`[]`）
2. 一元运算符（`-x`、`!x`、`*x`、`&x`）
3. `as`（类型转换）
4. `*`、`/`、`%`（乘除余）
5. `+`、`-`（加减）
6. `<<`、`>>`（位移）
7. `&`（按位与）
8. `^`（按位异或）
9. `|`（按位或）
10. `==`、`!=`、`<`、`>`、`<=`、`>=`（比较）
11. `&&`（逻辑与）
12. `||`（逻辑或）
13. `..`、`..=`（范围）
14. `=`、`+=` 等（赋值）

**实践原则：不确定就加括号，比记规则更可靠。**

## 关键优先级陷阱

### 陷阱 1：`&&` 比 `||` 优先级高

```rust
// 这两个是不同的！
let a = true || false && false;
// 等价于：true || (false && false) = true || false = true

let b = (true || false) && false;
// = true && false = false

println!("{}", a);  // true
println!("{}", b);  // false
```

### 陷阱 2：位运算低于比较运算（经典陷阱！）

```rust
let x = 5_u8;

// ❌ 这不是你想要的！
let wrong = x & 1 == 0;
// 等价于：x & (1 == 0) = x & false → 类型错误！
// 实际上编译器会报错

// ✅ 正确写法
let correct = (x & 1) == 0;  // 先位与，再比较
println!("x={x} 是偶数: {correct}");
```

### 陷阱 3：一元 `-` 不是最高优先级

```rust
let x = 2_i32;
let y = -x.pow(2);  // -(x.pow(2)) = -(4) = -4，不是 (-x).pow(2) = 4
println!("{y}");    // -4
```

## 完整运行示例

```rust
fn main() {
    println!("=== 基础优先级 ===");
    println!("2 + 3 * 4 = {}   (先*后+)", 2 + 3 * 4);      // 14
    println!("(2+3) * 4 = {}   (括号改变)", (2 + 3) * 4);   // 20
    println!("10 - 6 / 2 = {} (先/后-)", 10 - 6 / 2);      // 7
    println!();

    println!("=== 一元运算符 ===");
    let x = 3_i32;
    println!("-x * 2 = {}   (-x 先算)", -x * 2);   // -6
    println!("-(x * 2) = {}", -(x * 2));            // -6（相同）
    println!("-x.pow(2) = {}", -x.pow(2));          // -9（方法调用优先）
    println!("(-x).pow(2) = {}", (-x).pow(2));      // 9（括号改变）
    println!();

    println!("=== 逻辑运算符 ===");
    println!("true || false && false = {} (&&先算)", true || false && false);  // true
    println!("(true || false) && false = {}", (true || false) && false);  // false
    println!();

    println!("=== 位运算和比较 ===");
    let n: u8 = 6;
    println!("n = {n} = {:08b}", n);
    println!("(n & 1) == 0: {}  (是偶数)", (n & 1) == 0);  // true

    // 使用位运算的实际场景
    let flags: u8 = 0b0000_0111;  // 低3位是权限
    let read  = (flags >> 0) & 1;
    let write = (flags >> 1) & 1;
    let exec  = (flags >> 2) & 1;
    println!("读权限: {}, 写权限: {}, 执行权限: {}", read, write, exec);
    println!();

    println!("=== 实战建议：遇到复杂表达式加括号 ===");
    let a = 5;
    let b = 3;
    let c = 2;

    // 不清楚时，明确写出括号
    let result1 = (a + b) * c;        // 明确：先加后乘
    let result2 = a + (b * c);        // 明确：先乘后加
    let result3 = (a & b) | c;        // 明确：位运算顺序

    println!("(a+b)*c   = {result1}");  // 16
    println!("a+(b*c)   = {result2}");  // 11
    println!("(a&b)|c   = {result3}");  // 3
}
```

## 完整优先级参考

参考 [Rust Reference - Expression Precedence](https://doc.rust-lang.org/reference/expressions.html#expression-precedence)（官方完整表）

**最实用的三条规则**：

1. `*`、`/`、`%` 先于 `+`、`-`（基础数学）
2. `&&` 先于 `||`（逻辑）
3. 比较运算（`==`、`<` 等）先于 `&&` 和 `||`
4. 位运算（`&`、`|`、`^`）低于比较运算（常忘记！）
