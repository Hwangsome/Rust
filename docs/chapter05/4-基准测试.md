## 这一节解决什么问题

"这个函数快不快？"——直觉往往不靠谱，需要测量。但测量本身也有陷阱：编译器可能优化掉你想测量的代码！

---

## 一分钟结论

- **不要在 debug 模式下 benchmark**——编译器优化被关闭，结果无意义
- `std::hint::black_box(x)`：阻止编译器优化掉 x
- `std::time::Instant::now()` + `.elapsed()` = 粗略计时
- **正式 benchmark 用 `criterion` crate**（稳定版，有统计学分析）
- nightly Rust 有 `#[bench]` + `test::Bencher`，但不稳定

---

## 测量时常见陷阱

### 陷阱 1：编译器把常量计算优化掉

```rust
// ❌ 错误：编译器发现 sum 结果是常量，直接优化成 result = 5050
let result = {
    let sum: i32 = (1..=100).sum();
    sum
};
// 测量的是 0ns（已被优化）！

// ✅ 正确：用 black_box 阻止优化
use std::hint::black_box;
let result = black_box((1..=black_box(100)).sum::<i32>());
```

### 陷阱 2：结果被丢弃

```rust
// ❌ 函数被调用了，但结果没用，编译器可能整个函数都不执行
for _ in 0..10000 {
    compute_something(42);  // 返回值被丢弃
}

// ✅ 用 black_box 使用返回值
for _ in 0..10000 {
    let _ = black_box(compute_something(black_box(42)));
}
```

## 粗略计时

```rust
use std::time::Instant;
use std::hint::black_box;

fn measure<F: Fn() -> T, T>(name: &str, f: F) {
    const ITERATIONS: u32 = 1000;

    // 热身（让 CPU 缓存和 JIT 稳定）
    for _ in 0..100 {
        black_box(f());
    }

    let start = Instant::now();
    for _ in 0..ITERATIONS {
        black_box(f());
    }
    let elapsed = start.elapsed();

    println!("{name}: {:.2} ns/op",
        elapsed.as_nanos() as f64 / ITERATIONS as f64);
}
```

## 完整运行示例

```rust
use std::time::Instant;
use std::hint::black_box;

fn sum_loop(n: u64) -> u64 {
    let mut sum = 0u64;
    for i in 1..=n { sum += i; }
    sum
}

fn sum_formula(n: u64) -> u64 {
    n * (n + 1) / 2
}

pub fn run() {
    println!("=== 基准测试基础 ===");
    println!("注意: 这里是 debug 模式，结果仅供演示");
    println!("      生产 benchmark 用 cargo run --release 或 criterion");
    println!();

    let n: u64 = 100_000;
    let iterations = 100u32;

    // 循环求和
    let start = Instant::now();
    for _ in 0..iterations {
        let _ = black_box(sum_loop(black_box(n)));
    }
    let loop_time = start.elapsed();

    // 公式求和
    let start = Instant::now();
    for _ in 0..iterations {
        let _ = black_box(sum_formula(black_box(n)));
    }
    let formula_time = start.elapsed();

    println!("sum_loop({n}) × {iterations} 次: {:?}", loop_time);
    println!("sum_formula({n}) × {iterations} 次: {:?}", formula_time);
    println!("公式快了约 {:.1}×",
        loop_time.as_nanos() as f64 / formula_time.as_nanos() as f64);
    println!();

    println!("=== 生产级 benchmark 用 criterion ===");
    println!("在 Cargo.toml 添加:");
    println!("  [dev-dependencies]");
    println!("  criterion = \"0.5\"");
    println!();
    println!("创建 benches/my_bench.rs 并写:");
    println!("  use criterion::{{criterion_group, criterion_main, Criterion}};");
    println!("  fn bench(c: &mut Criterion) {{");
    println!("      c.bench_function(\"sum\", |b| b.iter(|| sum_loop(black_box(100_000))));");
    println!("  }}");
    println!("  criterion_group!(benches, bench);");
    println!("  criterion_main!(benches);");
    println!();
    println!("然后 cargo bench 运行");
}
```

## 使用 criterion 的完整示例（benches/my_bench.rs）

```rust
use criterion::{black_box, criterion_group, criterion_main, Criterion};

fn fibonacci(n: u64) -> u64 {
    match n {
        0 => 0,
        1 => 1,
        n => fibonacci(n - 1) + fibonacci(n - 2),
    }
}

fn criterion_benchmark(c: &mut Criterion) {
    c.bench_function("fib 20", |b| b.iter(|| fibonacci(black_box(20))));
}

criterion_group!(benches, criterion_benchmark);
criterion_main!(benches);
```
