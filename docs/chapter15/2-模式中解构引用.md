## `&` 与 `ref` 在模式中

```rust
// & 在模式里：解开引用（match 外层的引用）
let r = &42;
let &x = r;     // x = 42（解包了 &）
println!("{x}");

// ref 在模式里：创建引用
let s = String::from("hello");
let ref y = s;  // y 是 &String（而不是消耗 s）
println!("{y}");
println!("{s}"); // s 仍然有效

// 在 match 里
let maybe = Some(String::from("value"));
match &maybe {              // 注意是 &maybe
    Some(s) => println!("{s}"),  // s 是 &String，不消耗 maybe
    None => {}
}
println!("{maybe:?}"); // maybe 仍然有效
```

## 为什么 match &collection

```rust
let v = vec![1, 2, 3];

// ❌ 消耗 v
match v {
    _ => {}
}
// println!("{v:?}"); // v 已被 move

// ✅ 保留 v
let v = vec![1, 2, 3];
match &v {
    _ => {}
}
println!("{v:?}"); // v 仍有效
```
