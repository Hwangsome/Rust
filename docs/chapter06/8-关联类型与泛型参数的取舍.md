## 这一节解决什么问题

你设计一个 `Convert` trait：

```rust
// 方案 A：关联类型
trait Convert {
    type Output;
    fn convert(&self) -> Self::Output;
}

// 方案 B：泛型参数
trait Convert<Output> {
    fn convert(&self) -> Output;
}
```

这两种方案在用法上有实质区别——选错了会导致 API 要么太宽泛要么太受限。

---

## 一分钟结论

**选关联类型（`type Item`）当**：

- 对每个实现者，该类型有且只有一种"自然"的选择
- 调用方不需要"指定"这个类型
- 例：`Iterator::Item`（每个迭代器只产出一种元素类型）

**选泛型参数（`trait T<U>`）当**：

- 同一个类型可以多次实现 trait（针对不同的 U）
- 调用方需要选择不同版本
- 例：`Add<Rhs>`（`i32 + i32`、`i32 + f64` 是不同实现）

---

## 详细对比

```
关联类型：一对一
  impl Iterator for MyIter { type Item = i32 }
  ↑ 这个绑定是固定的：MyIter 只有一种 Item

  调用方写法：
  fn use_iter<I: Iterator>(i: I) { ... }   ← 简洁
  fn use_iter<I: Iterator<Item=i32>>(i: I) { ... }  ← 可以限定 Item

泛型参数：一对多
  impl Add<f64> for MyType { type Output = f64 }
  impl Add<i32> for MyType { type Output = i32 }
  ↑ MyType 对不同 Rhs 有不同实现！

  调用方写法：
  fn add_f64(a: impl Add<f64>) { ... }  ← 指定 Rhs = f64
  fn add_any<T, Rhs: Add<T>>(a: Rhs, b: T) { ... }
```

## 完整运行示例

```rust
use std::ops::{Add, Mul};
use std::fmt::Display;

// ===== 关联类型：每种容器只有一种 Item =====
trait Container {
    type Item;
    fn first(&self) -> Option<&Self::Item>;
    fn count(&self) -> usize;
}

struct Stack<T> { data: Vec<T> }

impl<T> Container for Stack<T> {
    type Item = T;  // ← 固定：Stack<T> 的 Item 就是 T
    fn first(&self) -> Option<&T> { self.data.first() }
    fn count(&self) -> usize { self.data.len() }
}

fn print_first<C: Container>(c: &C)
where
    C::Item: Display,
{
    match c.first() {
        Some(x) => println!("  first: {x}"),
        None => println!("  empty"),
    }
}

// ===== 泛型参数：同一类型多种转换 =====
trait CastTo<T> {
    fn cast(&self) -> T;
}

struct Pixel { r: u8, g: u8, b: u8 }

impl CastTo<f64> for Pixel {
    fn cast(&self) -> f64 {
        (self.r as f64 + self.g as f64 + self.b as f64) / 3.0
    }
}

impl CastTo<String> for Pixel {
    fn cast(&self) -> String {
        format!("#{:02X}{:02X}{:02X}", self.r, self.g, self.b)
    }
}

impl CastTo<(u8, u8, u8)> for Pixel {
    fn cast(&self) -> (u8, u8, u8) { (self.r, self.g, self.b) }
}

fn main() {
    println!("=== 关联类型（一对一关系）===");
    let stack = Stack { data: vec![10, 20, 30] };
    print_first(&stack);
    println!("  count: {}", stack.count());
    println!();

    println!("=== 泛型参数（一对多关系）===");
    let pixel = Pixel { r: 255, g: 128, b: 64 };
    let gray: f64 = pixel.cast();
    let hex: String = pixel.cast();
    let tuple: (u8, u8, u8) = pixel.cast();
    println!("  灰度: {gray:.1}");
    println!("  十六进制: {hex}");
    println!("  元组: {tuple:?}");
}
```

## 选型决策树

```
设计 trait 时，某个类型参数...
  │
  ├── 对每个实现者是"唯一确定"的？
  │       是 → 关联类型 (type Item)
  │             例：Iterator::Item, Deref::Target
  │
  └── 同一个类型可能以不同方式实现（对不同 Rhs）？
          是 → 泛型参数 (trait T<Rhs>)
               例：Add<Rhs>, From<T>, Into<T>
```

---

## 我的理解与记忆方法

**一句话记忆**：

> 关联类型是"这个类型只有一个身份"；泛型参数是"这个类型可以扮演多个角色"。
