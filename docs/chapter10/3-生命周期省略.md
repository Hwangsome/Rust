## 这一节解决什么问题

如果每个函数都需要手写生命周期标注，Rust 代码会变得极其冗长。实际上，编译器有一套**省略规则**，能在绝大多数情况下自动推断生命周期，让你不需要手写。

```rust
// 有了省略规则，这样写就够了：
fn first_word(s: &str) -> &str { ... }

// 而不需要写这么长：
fn first_word<'a>(s: &'a str) -> &'a str { ... }
```

---

## 一分钟结论

三条省略规则（按顺序应用）：

1. **每个引用参数**各自分配一个独立生命周期
2. **如果只有一个输入引用参数**，输出引用的生命周期 = 输入引用的生命周期
3. **如果有 `&self` 或 `&mut self`**，输出引用的生命周期 = self 的生命周期

全部三条都无法推断时，编译器报错，需要手写标注。

---

## 三条规则详解

### 规则 1：每个输入引用参数分配独立生命周期

```rust
// 你写的：
fn f(s: &str) -> &str
// 规则 1 应用后：
fn f<'a>(s: &'a str) -> &str  // 输出还不确定
```

### 规则 2：只有一个输入引用 → 输出继承它

```rust
// 应用规则 1 和 2：
fn f(s: &str) -> &str
// 展开后：
fn f<'a>(s: &'a str) -> &'a str  // ✅ 推断完成
```

### 规则 3：方法有 &self → 输出继承 self

```rust
// 应用规则 1 和 3：
impl Config {
    fn description(&self) -> &str { &self.desc }
}
// 展开后：
fn description<'a>(&'a self) -> &'a str  // ✅ 推断完成
```

### 三条规则都无法推断的情况

```rust
// ❌ 需要手写：两个输入引用，编译器不知道返回值和哪个绑定
fn longer(s1: &str, s2: &str) -> &str {
    if s1.len() > s2.len() { s1 } else { s2 }
}
// 规则 1：fn longer<'a,'b>(s1: &'a str, s2: &'b str) -> &? str
// 规则 2 和 3 都不适用（两个输入引用，无 self）
// ❌ 无法推断，必须手写：
fn longer<'a>(s1: &'a str, s2: &'a str) -> &'a str { ... }
```

## 完整运行示例

```rust
struct Article {
    title: String,
    content: String,
}

impl Article {
    fn new(title: &str, content: &str) -> Self {
        Article { title: title.to_string(), content: content.to_string() }
    }

    // 规则 3 应用：&self → 返回值生命周期 = self 的
    fn title(&self) -> &str {
        &self.title
    }

    // 规则 3：多个参数时，&self 决定返回值生命周期
    fn title_or_default<'a>(&'a self, default: &'a str) -> &'a str {
        if self.title.is_empty() { default } else { &self.title }
    }
}

// 规则 2：单输入引用
fn first_word(s: &str) -> &str {
    s.split_whitespace().next().unwrap_or("")
}

// 需要显式标注：多个输入引用，无 &self
fn longer<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    if s1.len() >= s2.len() { s1 } else { s2 }
}

fn main() {
    let article = Article::new("Rust", "Rust is amazing...");

    // 规则 3：title() 返回 &self 生命周期内的引用
    println!("title: {}", article.title());

    // 规则 2：first_word 推断正确
    let sentence = "hello world";
    let word = first_word(sentence);
    println!("first word: {word}");

    // 需要显式：longer
    let s1 = String::from("hello world");
    let s2 = String::from("hi");
    println!("longer: {}", longer(&s1, &s2));
}
```

## 判断是否需要手写生命周期的流程

```
函数返回引用时，问自己：

1. 只有一个输入引用参数？
   → 不需要标注（规则 2）

2. 是方法（有 &self）？
   → 不需要标注（规则 3）

3. 多个输入引用，返回值和其中一个绑定？
   → 需要手写，指明和哪个绑定

4. 返回的引用和任何输入都无关（来自局部变量）？
   → 编译错误，不应该返回这样的引用
```
