## 核心规则

Rust 的模块系统把**模块声明**与**文件系统**一一对应：

| 声明方式 | Rust 去哪里找实现 |
|----------|------------------|
| `mod foo;`（在 `src/main.rs` 或 `src/lib.rs`） | `src/foo.rs` 或 `src/foo/mod.rs` |
| `mod bar;`（在 `src/foo/mod.rs` 中） | `src/foo/bar.rs` |

> 推荐优先使用 `src/foo.rs`（2018 edition 后的惯用风格），`mod.rs` 适合模块本身就需要对外重新 export 子项时使用。

## 本章示例：accommodation 模块

### 目录结构

```
src/
├── main.rs
└── accommodation/
    ├── mod.rs      ← 声明子模块 + trait 定义 + pub use
    ├── hotel.rs
    ├── airbnb.rs
    └── hostel.rs
```

### mod.rs 的三个职责

```rust
// 1. 声明子模块（让编译器去找对应文件）
pub mod hotel;
pub mod airbnb;
pub mod hostel;

// 2. 定义公共 trait（所有子文件均可 use super::Accommodation）
pub trait Accommodation { ... }
pub trait StayPolicy { ... }

// 3. Re-export：简化调用方路径
pub use hotel::Hotel;
pub use airbnb::Airbnb;
pub use hostel::Hostel;
```

### 子文件模式

每个结构体文件只关注自身实现，通过 `use super::...` 引用父模块的 trait：

```rust
// hotel.rs
use super::{Accommodation, StayPolicy};

pub struct Hotel { ... }
impl Accommodation for Hotel { ... }
impl StayPolicy for Hotel { ... }
```

### 调用方（main.rs）

```rust
mod accommodation;
use accommodation::{Accommodation, Airbnb, Hostel, Hotel, StayPolicy};
```

re-export 后，外部只需写 `accommodation::Hotel`，内部文件拆分对调用方透明。

---

## 要点总结

- `mod foo;` 声明 + `foo.rs` / `foo/mod.rs` 实现 = 文件式模块
- `mod.rs` 集中管理子模块声明与 re-export，降低调用方路径复杂度
- `use super::Trait` 是子文件引用父模块定义的标准写法
- `pub use` 实现**扁平化导出**，隐藏内部目录层级
