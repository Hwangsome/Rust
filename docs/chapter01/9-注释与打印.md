## 一分钟结论

- 注释是给人看的，不是给编译器执行的
- 学习 Rust 时，打印是最直接的观察手段
- `println!` 会自动换行，`print!` 不会
- 格式化输出能让示例更清晰

## 最小示例

```rust
fn main() {
    let language = "Rust";
    println!("I am learning {language}");
}
```

## 推荐做法

- 用注释解释“为什么”
- 用打印观察变量值、分支路径和循环过程
- 用命名参数或 `{variable}` 让输出更容易读

## 常见误区

### ❌ 把注释写成逐行翻译

```rust
let x = 1; // 定义 x
```

这种注释信息量太低，不值得保留。

### ❌ 打印太少，导致看不出程序在做什么

入门阶段宁可多打印一点关键状态，也不要完全靠脑补程序执行过程。
