## 一分钟结论

- module 的核心价值是**命名空间**
- 它让相关代码能被组织到一起
- 调用时通常要带路径，例如 `greetings::hello()`

## 证据来源

- 对应模块：[modules.rs](../../chapters/chapter04/src/modules.rs)
- 运行章节：`cargo run -p chapter04`

关键输出：

```text
hello from the greetings module
模块的作用是把相关代码放进同一个命名空间。
```

## 定义

module 是 crate 内部的命名空间单元。

## 作用

- 把相关代码收进同一命名空间
- 避免名字冲突
- 提供更清楚的路径结构

## 原理

示例：

```rust
mod greetings {
    pub fn hello() {
        println!("hello");
    }
}
```

调用时：

```rust
greetings::hello();
```

这说明：

- `hello` 不再是裸名字
- 它被组织进 `greetings` 这个模块路径下

## 最小示例

```rust
mod greetings {
    pub fn hello() {
        println!("hello from module");
    }
}

fn main() {
    greetings::hello();
}
```

## 注意点

### 1. module 不只是“为了分文件”

即使只有一个文件，module 也仍然有命名空间意义。

### 2. 有模块路径，就有访问边界

路径不是装饰，而是结构本身的一部分。

## 常见错误

### ❌ 错误 1：把 module 当目录标签，不看语义边界

### ❌ 错误 2：路径一长就想全部拍平

拍平路径虽然省输入，但通常会损失结构信息。

## 我的理解

- module 首先是“整理名字”，其次才是“整理文件”
- 一旦把命名空间这层想清楚，路径就不再显得啰嗦
