```text
first_word = borrow
title = Rust 2024
longer("abc", "defgh") = defgh
```

## 定义

生命周期省略是编译器在某些常见函数签名里自动补全生命周期参数的规则。

## 作用

- 减少样板代码
- 让简单借用函数更易读
- 保留生命周期检查而不强迫每个地方都显式标注

## 原理

编译器会按固定规则推断，例如“单个输入引用的函数，输出引用通常继承这个输入生命周期”。因此 `fn first_word(text: &str) -> &str` 可以直接成立。

## 最小示例

```rust
fn first_word(text: &str) -> &str {
    text.split_whitespace().next().unwrap_or("")
}
```

## 注意点

- 省略规则是规则推断，不是智能猜测
- 一旦超出规则范围，就必须自己显式写出来
- 方法签名里 `&self` 也参与省略规则

## 常见错误

- 以为“没写生命周期”就代表“不涉及生命周期”
- 省略失败时看不懂编译器为什么突然要求写 `'a`
- 想靠多写显式生命周期“压住”所有问题

## 我的理解

省略规则让我意识到：生命周期其实是常态，只是很多时候编译器已经帮我们把重复劳动做了。
