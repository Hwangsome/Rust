## 这一节解决什么问题

借用（引用）创建之后，如何**访问或修改它背后的值**？答案是**解引用**：`*reference`。

这个操作在 Rust 里有两个面：

1. **读取**：`*r` 从引用取出底层值（读）
2. **写入**：`*r = new_value` 通过可变引用修改底层值（写）

此外，Rust 还有"自动解引用"——在方法调用和 `println!` 等场景里，编译器会自动做 `*`，所以你常常"隐式"在用解引用而不自知。

理解"引用是指针，解引用是取出指针背后的值"，是理解 `Box<T>`、`Rc<T>`、`Arc<T>` 等智能指针的基础。

---

## 一分钟结论

- `*r`（前缀 `*`）是**解引用**，取出引用背后的值
- `*r = value` 通过 `&mut T` 修改原始数据
- 方法调用（`.method()`）和 `println!` 会**自动解引用**，不需要手写 `*`
- `r1 == r2`（两个引用比较）比的是**指向的值**，不是地址；`std::ptr::eq(r1, r2)` 比地址
- `*` 中缀是**乘法**，`*` 前缀是**解引用**——同符号不同语义
- 实现了 `Deref<Target = T>` 的类型，使用 `.method()` 时可以自动跨层解引用

## 与其他语言对比

| 语言 | 如何访问指针/引用背后的值 |
|-----|----------------------|
| C | `*ptr`（显式）；`ptr->field`（结构体指针） |
| C++ | `*ptr`（显式）；`ptr->field`；引用通常隐式 |
| Java | 所有对象都是引用，访问字段直接 `.field`，无显式解引用 |
| Python | 无指针概念，所有变量都是引用，直接用 |
| Go | `*ptr` 解引用指针；结构体指针 `ptr.field` 自动解引用 |
| Rust | `*r` 显式，但方法调用和大多数场景自动解引用 |

## 核心概念与心智模型

### 引用是指针，解引用是"跟着地址走到数据"

```
引用（&T）在内存中的样子：

  栈上：
  ┌──────────────────────────────────┐
  │  r: 0x7fff5c4a8000 (8字节地址)  │
  └──────────────────────────────────┘
           │ *r 就是"跟着这个地址走"
           ▼
  栈或堆上：
  ┌──────────────────────────────────┐
  │  value: 42 (i32, 4字节)         │
  └──────────────────────────────────┘

println!("{}", r)   → 自动解引用，打印 42
println!("{}", *r)  → 显式解引用，打印 42
```

### 自动解引用的触发场景

```rust
let s = String::from("hello");
let r = &s;

// 这四行效果完全一样：
println!("{}", r.len());       // 自动解引用：(*r).len()
println!("{}", (*r).len());    // 显式解引用
println!("{}", s.len());       // 直接调用（不通过引用）
println!("{}", &s.len());      // &s 也能自动 deref

// 方法调用时，Rust 会反复 deref 直到找到 target type
let boxed_string: Box<String> = Box::new(String::from("world"));
// Box<String> → String（通过 Box::Deref）→ str（通过 String::Deref）
println!("{}", boxed_string.len()); // 两层自动解引用！
```

## 详细原理

### 1. 基础读取 `*r`

```rust
let value = 42;
let r: &i32 = &value;

// 两种等价写法：
println!("via ref:   {r}");    // 自动解引用（打印 42）
println!("via deref: {}", *r); // 显式解引用（也是 42）

// 在算术中，必须显式解引用：
let doubled = *r * 2;  // *r + 1：解引用再做算术
let also_doubled = r * 2; // 有时编译器也能推断，但不总是
println!("doubled = {doubled}");
```

### 2. 通过 `&mut T` 写入（`*r = new_value`）

```rust
let mut x = 10;
let r: &mut i32 = &mut x;

// 写入：通过可变引用修改原始数据
*r = 20;         // x 现在是 20
*r += 5;         // x 现在是 25
println!("x = {x}"); // 25
// 关键：r 是"到 x 的可变借用"，*r = ... 是修改"r 指向的地方"，即 x
```

### 3. 引用比较：值相等 vs 地址相等

```rust
let a = 42;
let b = 42;
let ra = &a;
let rb = &b;

// `==` 比的是值（背后的数据）
println!("{}", ra == rb); // true（42 == 42）

// ptr::eq 比的是地址（是否指向同一块内存）
println!("{}", std::ptr::eq(ra, rb)); // false（a 和 b 在不同地址）

// 同一变量的两个引用：
let rc = &a;
println!("{}", std::ptr::eq(ra, rc)); // true（都指向 a 的地址）
```

### 4. `*` 前缀（解引用）vs `*` 中缀（乘法）

```rust
let x = 5;
let r = &x;

let product = 2 * x;   // 中缀 *：乘法（2 乘以 x）
let deref_val = *r;    // 前缀 *：解引用（取出 r 指向的值）
let mixed = *r * 2;    // 先解引用 *r 得到 5，再乘以 2

println!("{product}");    // 10
println!("{deref_val}");  // 5
println!("{mixed}");      // 10
```

### 5. 智能指针的解引用（`Deref` trait）

```rust
let boxed: Box<i32> = Box::new(42);

// 显式解引用：
println!("{}", *boxed);  // 42

// 方法调用自动解引用：
// Box<i32> 没有 to_string()，但编译器通过 Deref 找到 i32::to_string()
let s = boxed.to_string(); // 等价于 (*boxed).to_string()
println!("{s}");           // "42"

// Vec 通过多层 Deref 访问 [T] 的方法：
let v: Box<Vec<i32>> = Box::new(vec![1, 2, 3]);
// Box → Vec → [T]
println!("{}", v.len()); // 3（两层自动解引用）
```

## 完整运行示例

```rust
fn demonstrate_basic_deref() {
    println!("=== 基础解引用 ===");

    let value: i32 = 42;
    let reference: &i32 = &value;

    println!("  reference      = {reference}  (类型: &i32)");
    println!("  *reference     = {}", *reference);  // 显式解引用
    println!("  reference 类型是 &i32（8字节指针），不是 i32");

    // 算术中的解引用
    let plus_one = *reference + 1;
    println!("  *reference + 1 = {plus_one}");
}

fn demonstrate_mutable_deref() {
    println!("\n=== 通过 &mut T 修改值 ===");

    let mut counter = 0_i32;
    let r = &mut counter;

    *r += 10;
    *r *= 2;
    // r 的借用结束（NLL：最后一次使用在 *= 2 之后）
    println!("  counter = {counter}");  // 20

    // 另一个例子：修改 Vec 元素
    let mut v = vec![1, 2, 3];
    for item in v.iter_mut() {
        *item *= 3; // 解引用 &mut i32，然后修改
    }
    println!("  v 乘以 3 = {v:?}");
}

fn demonstrate_auto_deref() {
    println!("\n=== 自动解引用（方法调用）===");

    let s = String::from("hello, world");
    let r = &s;

    // 这几行效果完全一样：
    println!("  r.len()    = {}", r.len());      // 自动 deref
    println!("  (*r).len() = {}", (*r).len());   // 显式 deref
    println!("  s.len()    = {}", s.len());      // 直接

    // 多层自动 deref：
    let boxed = Box::new(String::from("rust"));
    // Box<String> → String（Deref）→ str（Deref）→ 找到 .len() 方法
    println!("  Box<String>.len() = {}", boxed.len());
}

fn demonstrate_comparison() {
    println!("\n=== 引用比较：值相等 vs 地址相等 ===");

    let x: i32 = 100;
    let y: i32 = 100;

    let rx = &x;
    let ry = &y;
    let rx2 = &x;  // 另一个指向 x 的引用

    println!("  rx == ry: {} (值比较: 100 == 100)", rx == ry);
    println!("  ptr::eq(rx, ry): {} (地址比较: 不同变量)", std::ptr::eq(rx, ry));
    println!("  ptr::eq(rx, rx2): {} (地址比较: 同一变量)", std::ptr::eq(rx, rx2));
}

fn main() {
    demonstrate_basic_deref();
    demonstrate_mutable_deref();
    demonstrate_auto_deref();
    demonstrate_comparison();
}
```

## 编译器错误分析

### ❌ 在没有 `&mut T` 的情况下通过引用写入

```rust
let x = 5;
let r = &x;  // 不可变引用
*r = 10;     // ❌
```

```text
error[E0594]: cannot assign to `*r`, which is behind a `&` reference
  |
3 | *r = 10;
  | ^^^^^^^^ `r` is a `&` reference, so the data it refers to cannot be written
  |
help: consider changing this to be a mutable reference
  |
2 | let r = &mut x;
  |          +++
```

**修复**：

```rust
let mut x = 5;
let r = &mut x;
*r = 10; // ✅
```

### ❌ 对数字常量解引用后试图用作 lvalue

```rust
let n = &42;
*n = 100; // ❌ 42 是字面量，不能写入
```

```text
error[E0594]: cannot assign to `*n`, which is behind a `&` reference
```

## 实际工程场景

### 1. 迭代器里的解引用

```rust
let v = vec![1, 2, 3, 4, 5];

// iter() 产出 &i32，需要解引用才能做算术
let sum: i32 = v.iter().map(|&x| x * 2).sum();
// 等价写法：
let sum: i32 = v.iter().map(|x| *x * 2).sum();

println!("{sum}"); // 30
```

### 2. 实现 `Deref` 让智能指针透明访问内部

```rust
struct MyBox<T>(T);

impl<T> std::ops::Deref for MyBox<T> {
    type Target = T;
    fn deref(&self) -> &T { &self.0 }
}

let mb = MyBox(String::from("hello"));
// 自动解引用链：&MyBox<String> → &String → &str
fn accept_str(s: &str) { println!("{s}"); }
accept_str(&mb); // ✅ 通过两层 Deref 自动到 &str
```

### 3. 在 struct 方法里通过 `&mut self` 修改字段

```rust
struct Account {
    balance: f64,
}

impl Account {
    fn deposit(&mut self, amount: f64) {
        self.balance += amount;
        // 等价于：(*self).balance += amount（自动解引用）
    }
}
```

## `Deref` Trait 速查

```rust
// 标准库中实现了 Deref 的关键类型：
Box<T>: Deref<Target = T>
String: Deref<Target = str>
Vec<T>: Deref<Target = [T]>
Rc<T>:  Deref<Target = T>
Arc<T>: Deref<Target = T>
MutexGuard<T>: Deref<Target = T>
RefMut<T>: Deref<Target = T>
```

这就是为什么 `&Box<String>` 可以在需要 `&str` 的地方使用——通过两层 Deref 链自动完成转换。

## 注意点与陷阱

### 陷阱 1：`for x in &v` 里的 `x` 是 `&T`，常需要解引用

```rust
let v = vec![1, 2, 3];
let mut sum = 0;
for x in &v {
    sum += x;   // x 是 &i32，i32 实现了 Add<&i32>，一般能直接用
    sum += *x;  // 显式解引用也可以
}
```

### 陷阱 2：比较引用默认比值，容易误以为在比地址

```rust
let a = String::from("hello");
let b = String::from("hello");
assert!(&a == &b);  // true：比的是字符串内容（值相等）
assert!(!std::ptr::eq(&a, &b)); // 地址不同
```

## 我的理解与记忆方法

**核心直觉**：

> 引用（`&T`）是"通往某个值的门牌号"；解引用（`*r`）是"按照门牌号找到那个值"。

**什么时候需要显式 `*r`？**

```
方法调用（.method()）    → 不需要，自动解引用
println!("{}", r)        → 不需要，自动解引用
算术运算（*r + 1）       → 需要
赋值（*r = value）       → 需要
模式匹配（match *r）     → 有时需要，视情况
```
