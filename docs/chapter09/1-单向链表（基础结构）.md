## 定义

单向链表由一串节点组成，每个节点保存一个值和“指向下一个节点”的链接。

## 作用

- 训练对递归类型和所有权模型的理解
- 理解为什么链式结构天然适合 `Option<Box<Node>>`
- 为后面的插入、删除、双向结构打基础

## 原理

节点里直接包含下一个节点会导致类型无限展开。用 `Box<Node>` 后，当前节点只存一个固定大小的堆指针；再加上 `Option`，就能同时表示“有后继节点”和“到尾了”。

## 最小示例

```rust
type Link = Option<Box<Node>>;

struct Node {
    element: i32,
    next: Link,
}
```

## 注意点

- 这一步先关注结构，不急着做操作方法
- 链表头通常单独包在 `LinkedList { head: Link }` 里
- `None` 既可以表示空链表，也可以表示尾节点的 next

## 常见错误

- 直接把 `next: Node` 写进节点里
- 还没想清结构就急着写增删改查
- 只盯着值，不盯着“链接怎么表达”
