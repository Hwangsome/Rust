## 这一节解决什么问题

初学 Rust 时最常见的困惑之一：

```rust
let x = 5;
x = 6;  // ❌ 编译错误！
```

其他语言（Java、Python、C++）里 `x = 6` 是正常操作。为什么 Rust 默认不让改？

Rust 把"不可变"作为默认，把"可变"设为显式可选——这个设计让"意外的状态改变"变成了编译错误，而不是运行时 bug。

---

## 一分钟结论

- `let x = 5`：不可变绑定，之后不能修改 x 的值
- `let mut x = 5`：可变绑定，可以修改
- `let x = ...` 再次声明：**shadowing（遮蔽）**，不是修改，是创建新绑定
- 遮蔽允许类型改变；`mut` 不允许
- 类型通常由编译器推断，显式标注用 `: Type` 语法

---

## 与其他语言对比

| 特性 | Java | Python | JavaScript | Rust |
|-----|------|--------|-----------|------|
| 默认可变性 | 可变（`int x = 5; x = 6;` OK）| 可变 | `let`: 可变 | **不可变** |
| 常量声明 | `final` | 无官方方式 | `const` | `const` / `let`（默认）|
| 重新声明同名变量 | 同一作用域不行 | 可以 | 看 `let`/`var` | **可以（shadowing）**|

## 详细原理

### 1. 不可变绑定（默认）

```rust
let language = "Rust";

// language = "Go"; // ❌ E0384：不能对不可变绑定重新赋值

println!("language = {language}");
// language 的值永远是 "Rust"（在这个绑定的生命周期内）
```

**为什么默认不可变？**

- 减少意外的状态改变（并发安全的基础）
- 让代码更容易推理（看到变量名就知道值不会变）
- 编译器优化机会更多

### 2. 可变绑定：`mut`

```rust
let mut score = 0;
println!("初始分数: {score}");

score += 10;
score += 5;
println!("当前分数: {score}");  // 15

// mut 只允许改值，不允许改类型：
// score = "high";  // ❌ 类型不匹配
```

### 3. Shadowing（遮蔽）

```rust
let spaces = "   ";          // 类型：&str
let spaces = spaces.len();   // 类型：usize（类型变了！）
println!("spaces 的长度: {spaces}");  // 3

// 遮蔽 vs mut 的本质区别：
// - mut：改变同一个绑定的值（类型必须相同）
// - shadowing：创建新的绑定（可以是完全不同的类型）

// 遮蔽的另一个用途：变换而不用新名字
let n = "42";           // &str
let n: i32 = n.parse().unwrap();  // 解析成 i32，用同一个名字
let n = n * 2;          // i32 * 2 = 84
println!("n = {n}");    // 84
```

### 4. 类型推断 vs 显式标注

```rust
// 编译器推断（大多数情况）
let x = 42;          // i32（整数默认）
let y = 3.14;        // f64（浮点默认）
let z = "hello";     // &str

// 显式标注（需要精确控制类型时）
let a: u8 = 255;
let b: f32 = 3.14;
let c: Vec<String> = Vec::new();  // 类型参数 T 需要显式

// 有时编译器需要帮助
let parsed: i32 = "42".parse().unwrap();  // 不写 :i32 编译器不知道 parse 成什么
```

## 完整运行示例

```rust
fn main() {
    println!("=== 不可变绑定 ===");
    let language = "Rust";
    let year_released = 2015_i32;
    println!("{language} 于 {year_released} 年发布");
    println!();

    println!("=== 可变绑定 ===");
    let mut health = 100;
    let mut name = String::from("Hero");
    println!("初始: {name}, HP={health}");

    health -= 20;
    name.push_str(" the Brave");
    println!("战斗后: {name}, HP={health}");
    println!();

    println!("=== Shadowing（遮蔽）===");
    let original = "original value";
    println!("原始: {original}");

    let original = 42;  // 创建新绑定，类型从 &str 变成 i32
    println!("遮蔽后: {original}");  // 42

    let original = format!("数值是 {original}");  // 再次遮蔽，变成 String
    println!("再次遮蔽: {original}");
    println!();

    println!("=== mut vs shadowing 的区别 ===");
    // mut：值可变，类型不变
    let mut counter = 0_i32;
    counter += 1;
    counter += 1;
    println!("counter (mut): {counter}");

    // shadowing：每次都是新绑定，类型可变
    let x: i32 = 5;
    let x = x + 1;        // 新绑定（i32）
    let x = x * 2;        // 新绑定（i32）
    let x = format!("{x}"); // 新绑定（String）
    println!("x (shadowed to String): {x}");
    println!();

    println!("=== 作用域里的 shadowing ===");
    let value = 42;
    println!("外层: {value}");
    {
        let value = value + 100;  // 内层 shadowing
        println!("内层: {value}");  // 142
    }
    println!("回到外层: {value}");  // 仍然是 42（内层的遮蔽不影响外层）
    println!();

    println!("=== 类型推断和显式标注 ===");
    let inferred = 100;          // 推断为 i32
    let explicit: u8 = 200;      // 显式 u8
    let parsed: i32 = "99".parse().unwrap();  // 必须显式

    println!("inferred: {inferred} (i32)");
    println!("explicit: {explicit} (u8)");
    println!("parsed:   {parsed} (i32)");
}
```

## 编译器错误分析

### ❌ E0384：重新赋值给不可变变量

```rust
let x = 5;
x = 6; // ❌
```

```text
error[E0384]: cannot assign twice to immutable variable `x`
  |
1 | let x = 5;
  |     -
  |     |
  |     first assignment to `x`
  |     help: consider making this binding mutable: `mut x`
2 | x = 6;
  |     ^ cannot assign twice to immutable variable
```

### ❌ E0308：mut 不能改类型

```rust
let mut x = 5;
x = "hello"; // ❌ 类型从 i32 变成 &str
```

```text
error[E0308]: mismatched types
  |
  | x = "hello";
  |     ^^^^^^^ expected integer, found `&str`
```

**区别**：shadowing 可以改类型，`mut` 不可以。

---

## 我的理解与记忆方法

**三种"改变"的区别**：

```
x = 6     → 报错（x 是不可变绑定）
mut x; x = 6 → OK（同类型重新赋值）
let x = 6  → OK（shadowing：创建全新绑定，类型可以不同）
```

**何时用 `mut`，何时用 shadowing**：

```
需要多次修改同一个值（循环计数器、累加器）→ mut
需要一步步变换，但每一步概念上是"新的值"→ shadowing
```
