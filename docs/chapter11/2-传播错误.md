## 这一节解决什么问题

当你调用多个可能失败的函数时，每个都要 `match`，代码变得很冗长：

```rust
fn load_config(path: &str) -> Result<Config, Error> {
    let content = match fs::read_to_string(path) {
        Ok(c) => c,
        Err(e) => return Err(e.into()),
    };
    let config = match parse_config(&content) {
        Ok(c) => c,
        Err(e) => return Err(e.into()),
    };
    Ok(config)
}
```

`?` 操作符把这些样板代码压缩成一个符号：

```rust
fn load_config(path: &str) -> Result<Config, Error> {
    let content = fs::read_to_string(path)?;
    let config = parse_config(&content)?;
    Ok(config)
}
```

---

## 一分钟结论

- `expr?` 等价于：成功时解包 `Ok(v)` 得到 v；失败时调用 `From::from(e)` 转换错误类型后 `return Err(e)`
- `?` 只能用在返回 `Result<_, _>` 或 `Option<_>` 的函数中
- 通过 `From` trait 自动转换错误类型（不同 crate 的错误可以无缝传播）
- `?` 也可以用在 `Option`：`None` 时提前返回 `None`
- 链式调用：`f1()?.f2()?.f3()`

---

## 详细原理

### `?` 的完整展开

```rust
// 你写的
let value = some_result?;

// 等价于：
let value = match some_result {
    Ok(v)  => v,
    Err(e) => return Err(From::from(e)),
    //                   ^^^^^^^^^^^ 自动类型转换！
};
```

关键：`From::from(e)` 允许从 `io::Error`、`ParseIntError` 等任意错误类型转换到当前函数的返回类型 `E`，只要有对应的 `impl From<SrcError> for E`。

### `?` 在 `Option` 上

```rust
fn parse_and_double(s: &str) -> Option<i32> {
    let n: i32 = s.parse().ok()?;  // 解析失败 → 返回 None
    Some(n * 2)
}

println!("{:?}", parse_and_double("5"));    // Some(10)
println!("{:?}", parse_and_double("abc"));  // None
```

## 完整运行示例

```rust
use std::{fs, io, num::ParseIntError};

#[derive(Debug)]
enum AppError {
    Io(io::Error),
    Parse(ParseIntError),
    Logic(String),
}

impl From<io::Error> for AppError {
    fn from(e: io::Error) -> Self { AppError::Io(e) }
}

impl From<ParseIntError> for AppError {
    fn from(e: ParseIntError) -> Self { AppError::Parse(e) }
}

impl std::fmt::Display for AppError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            AppError::Io(e) => write!(f, "IO 错误: {e}"),
            AppError::Parse(e) => write!(f, "解析错误: {e}"),
            AppError::Logic(s) => write!(f, "逻辑错误: {s}"),
        }
    }
}

fn read_number(path: &str) -> Result<i32, AppError> {
    let content = fs::read_to_string(path)?;  // io::Error → AppError::Io
    let n = content.trim().parse::<i32>()?;   // ParseIntError → AppError::Parse
    if n < 0 {
        return Err(AppError::Logic(format!("{n} 不能为负数")));
    }
    Ok(n)
}

fn main() {
    println!("=== ? 操作符基础 ===");
    // 不存在的文件
    match read_number("nonexistent.txt") {
        Ok(n) => println!("  读取到: {n}"),
        Err(e) => println!("  错误: {e}"),
    }
    println!();

    println!("=== ? 在 Option 上 ===");
    fn first_positive(nums: &[i32]) -> Option<i32> {
        nums.iter().find(|&&x| x > 0).copied()
    }

    fn double_first_positive(nums: &[i32]) -> Option<i32> {
        let n = first_positive(nums)?;  // 没有正数时返回 None
        Some(n * 2)
    }

    println!("  [1,2,3] → {:?}", double_first_positive(&[1, 2, 3]));
    println!("  [-1,-2] → {:?}", double_first_positive(&[-1, -2]));
    println!();

    println!("=== ? 链式调用 ===");
    fn pipeline(s: &str) -> Result<String, AppError> {
        let n: i32 = s.parse().map_err(AppError::Parse)?;
        let doubled = n.checked_mul(2)
            .ok_or_else(|| AppError::Logic("溢出".into()))?;
        Ok(format!("结果: {doubled}"))
    }

    for input in ["21", "abc", "2000000000"] {
        match pipeline(input) {
            Ok(s) => println!("  ✅ {input} → {s}"),
            Err(e) => println!("  ❌ {input} → {e}"),
        }
    }
}
```
