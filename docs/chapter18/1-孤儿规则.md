# 1. 孤儿规则（Orphan Rule）

> - **所属章节**：第 18 章 · Trait Object Limitations
> - **Cargo package**：`chapter18`
> - **代码位置**：`chapters/chapter18/src/topic_01_orphan_rule.rs`

---

## 规则

`impl Trait for Type` 合法当且仅当：**Trait 或 Type 至少有一个定义在当前 crate 中**。

这防止了两个不同 crate 都为同一外部类型实现同一外部 trait 导致的冲突。

---

## 合法 vs 非法

```rust
// ✅ 自己的 trait + 外部类型
trait Printable { fn print(&self); }
impl Printable for i32 { fn print(&self) { println!("{self}"); } }

// ✅ 外部 trait + 自己的类型
struct MyStruct;
impl std::fmt::Display for MyStruct {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        write!(f, "MyStruct")
    }
}

// ❌ 外部 trait + 外部类型
// impl std::fmt::Display for Vec<i32> { ... } // E0117!
```

---

## Newtype 绕过孤儿规则

```rust
struct Wrapper(Vec<i32>);

impl std::fmt::Display for Wrapper {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        write!(f, "[{}]", self.0.iter().map(|n| n.to_string()).collect::<Vec<_>>().join(", "))
    }
}

println!("{}", Wrapper(vec![1, 2, 3])); // [1, 2, 3]
```
