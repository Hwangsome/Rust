## 一分钟结论

- Rust 里的变量默认是**不可变**的
- 想修改绑定的值，必须显式写 `mut`
- **遮蔽**不是修改原变量，而是创建一个同名的新绑定
- Rust 这样设计，是为了让“状态变化”更显眼，减少无意修改

## 证据来源

### 本次实验

不可变变量重新赋值：

```rust
fn main() {
    let x = 5;
    x = 6;
}
```

编译器输出节选：

```text
error[E0384]: cannot assign twice to immutable variable `x`
help: consider making this binding mutable
```

遮蔽示例：

```rust
fn main() {
    let spaces = "   ";
    let spaces = spaces.len();
    println!("{}", spaces);
}
```

运行结果：

```text
3
```

## 定义

在 Rust 里，变量更准确地说是 **binding（绑定）**：

- 用 `let` 把一个名字绑定到一个值
- 这个绑定默认不可变
- 如果需要修改，必须显式写 `mut`
- 如果再次 `let` 同名变量，这叫 **shadowing（遮蔽）**

## 作用

Rust 把变量默认设计成不可变，核心目的不是“写起来麻烦”，而是：

- 让状态变化更显式
- 让代码更容易推理
- 让编译器更容易帮助你发现问题
- 让“我到底是在复用旧值，还是创建新值”这件事更清楚

这也是 Rust 和很多默认可变语言在使用感上的一个重要差异。

## 原理

可以把这一节拆成三种不同动作：

### 1. 默认不可变

```rust
let language = "Rust";
```

这里的意思是：`language` 这个绑定一旦建立，就不能再被重新赋值。

### 2. 用 `mut` 允许同一个绑定变化

```rust
let mut score = 60;
score = 80;
```

这表示：还是同一个绑定，但值允许改变。

### 3. 用遮蔽创建一个新绑定

```rust
let spaces = "   ";
let spaces = spaces.len();
```

这里不是“把字符串原地改成数字”，而是重新创建了一个同名的新绑定，所以类型也能变化。

## 最小示例

```rust
fn main() {
    let language = "Rust";
    println!("language = {language}");

    let mut score = 60;
    println!("before = {score}");
    score = 80;
    println!("after = {score}");

    let spaces = "   ";
    let spaces = spaces.len();
    println!("spaces length = {spaces}");
}
```

## 常见错误

### ❌ 给不可变变量重新赋值

```rust
fn main() {
    let x = 5;
    x = 6;
}
```

这会触发编译错误，因为默认绑定不可变。

### ❌ 以为遮蔽和 `mut` 是一回事

不是。

- `mut`：修改同一个绑定的值
- 遮蔽：创建一个同名的新绑定

## 我的理解

这一节最值得记住的不是语法，而是 Rust 对“状态变化”的态度：

- 默认先假设不变
- 真要变，再显式写出来
- 如果想表达“这是一个新的阶段、新的值”，就用遮蔽
