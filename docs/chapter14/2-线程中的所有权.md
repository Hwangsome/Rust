## 为什么需要 `move`

```rust
let data = vec![1, 2, 3];

// ❌ 不加 move：编译器不确定 data 是否活得够长
let h = thread::spawn(|| {
    println!("{data:?}"); // 借用 data，但线程可能活过 data 的作用域
});

// ✅ 加 move：data 的所有权被搬进线程，生命周期跟着线程
let h = thread::spawn(move || {
    println!("{data:?}");
});
h.join().unwrap();
// println!("{data:?}"); // ❌ data 已被 move 进线程
```

---

## `Send` 和 `Sync`

```
Send：类型的所有权可以安全转移到另一个线程
  → 大多数类型自动是 Send
  → Rc<T> 不是 Send（引用计数非原子）→ 多线程用 Arc<T>

Sync：类型的 &T 可以安全被多线程同时持有
  → T: Sync ⟺ &T: Send
  → Cell<T> / RefCell<T> 不是 Sync → 多线程用 Mutex<T>
```

## 完整运行示例

```rust
use std::thread;

pub fn run() {
    println!("=== move 闭包 ===");
    let message = String::from("hello from main");

    let h = thread::spawn(move || {
        // message 的所有权已经在这里了
        println!("  线程收到: {message}");
    });
    h.join().unwrap();

    println!();
    println!("=== 什么类型可以跨线程 ===");
    println!("  Send（可以转移）：i32, String, Vec<T>, Arc<T>, Mutex<T>...");
    println!("  !Send（不能）：Rc<T>, *mut T（裸指针）");
    println!("  Sync（&T 可以共享）：同上，RefCell<T> 不是 Sync");

    // Arc 是线程安全的 Rc
    let shared = std::sync::Arc::new(42i32);
    let shared_clone = std::sync::Arc::clone(&shared);
    let h = thread::spawn(move || {
        println!("  Arc 共享值: {shared_clone}");
    });
    h.join().unwrap();
    println!("  主线程仍可访问: {shared}");
}
```
