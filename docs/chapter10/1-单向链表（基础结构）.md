## 这一节解决什么问题

手写数据结构是检验 Rust 所有权理解的最好方式。链表看起来简单，但在 Rust 里需要精心设计所有权——每个节点"谁拥有它？"

---

## 为什么链表需要 `Box`

```
❌ 不行：
struct Node {
    value: i32,
    next: Node,  // 递归！大小无穷大
}

✅ 可以：
struct Node {
    value: i32,
    next: Option<Box<Node>>,  // Box 把下一节点放堆上，大小固定
}
         ↑
    None 表示链表末尾，Box 是堆上的指针（固定8字节）
```

---

## 内存布局

```
LinkedList {
  head: Some(Box( Node {
    value: 1,
    next: Some(Box( Node {
      value: 2,
      next: Some(Box( Node {
        value: 3,
        next: None
      }))
    }))
  }))
}

栈：
  head_ptr [8字节指针] → 堆

堆：
  [value=1, next_ptr] → [value=2, next_ptr] → [value=3, next=None]
```

## 详细原理

```rust
type Link = Option<Box<Node>>;

struct Node {
    element: i32,
    next: Link,
}

struct LinkedList {
    head: Link,
    size: usize,
}

impl LinkedList {
    fn new() -> Self { LinkedList { head: None, size: 0 } }

    fn is_empty(&self) -> bool { self.head.is_none() }

    fn len(&self) -> usize { self.size }

    // 遍历产出值
    fn values(&self) -> Vec<i32> {
        let mut result = Vec::new();
        let mut current = &self.head;
        while let Some(node) = current {
            result.push(node.element);
            current = &node.next;
        }
        result
    }

    // peek：返回头节点值的引用（不移动）
    fn peek(&self) -> Option<&i32> {
        self.head.as_ref().map(|node| &node.element)
    }
}
```

## 完整运行示例

```rust
pub fn run() {
    // 手动构建链表（演示结构）
    let list = LinkedList {
        head: Some(Box::new(Node {
            element: 1,
            next: Some(Box::new(Node {
                element: 2,
                next: Some(Box::new(Node {
                    element: 3,
                    next: None,
                })),
            })),
        })),
        size: 3,
    };

    println!("链表内容: {:?}", list.values()); // [1, 2, 3]
    println!("头节点: {:?}", list.peek());     // Some(1)
    println!("长度: {}", list.len());          // 3
    println!("是否为空: {}", list.is_empty()); // false
}
```
