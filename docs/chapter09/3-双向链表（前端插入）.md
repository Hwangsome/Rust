## 为什么双向链表需要 `Rc<RefCell<Node>>`

双向链表的节点同时被：
1. **链表本身**（head/tail）引用
2. **前后节点**（prev/next）引用

这违反了 Rust 的"单一所有者"原则，必须用引用计数解决：

```
为什么需要 Rc？
  同一个节点被 head、前节点的 next、后节点的 prev 三个地方引用
  → 需要共享所有权 → Rc<T>

为什么需要 RefCell？
  需要修改节点的 prev/next 字段（通过 &self 的方法）
  → 需要内部可变性 → RefCell<T>

组合：Rc<RefCell<Node>>
```

---

## 核心数据结构

```rust
use std::{cell::RefCell, rc::Rc};

type NodePtr = Option<Rc<RefCell<Node>>>;

struct Node {
    value: i32,
    next: NodePtr,
    prev: NodePtr,
}

struct DoublyLinkedList {
    head: NodePtr,
    tail: NodePtr,
    size: usize,
}
```

---

## 前端插入

```rust
impl DoublyLinkedList {
    fn push_front(&mut self, value: i32) {
        let new_node = Rc::new(RefCell::new(Node {
            value,
            next: self.head.take(), // 新节点的 next = 旧 head
            prev: None,
        }));

        // 如果原来有 head，更新它的 prev 指向新节点
        if let Some(old_head) = &self.head {
            old_head.borrow_mut().prev = Some(Rc::clone(&new_node));
        } else {
            // 链表为空，new_node 也是 tail
            self.tail = Some(Rc::clone(&new_node));
        }

        self.head = Some(new_node);
        self.size += 1;
    }
}
```
