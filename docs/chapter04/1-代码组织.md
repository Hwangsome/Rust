## 这一节解决什么问题

Rust 代码组织有三个层次——package、crate、module——初学者经常把它们混淆。理解这三者，是看懂任何 Rust 项目结构的前提。

---

## 一分钟结论

**三层关系**（从大到小）：

```
Workspace（可选）
  └── Package（Cargo.toml 描述）
        ├── Binary Crate（src/main.rs，可执行）
        ├── Library Crate（src/lib.rs，可复用）
        └── Module（mod 关键字，代码命名空间）
```

- **Package**：由一个 `Cargo.toml` 描述的项目单元
- **Crate**：编译的最小单位；binary（有 main）或 library（有 lib.rs）
- **Module**：crate 内部的代码命名空间，用 `mod` 声明
- **Workspace**：多个 package 共享一个 `Cargo.lock`

## 详细说明

### 1. Package

```
一个 Package 包含：
  - 一个 Cargo.toml
  - 最多一个 library crate（src/lib.rs）
  - 任意多个 binary crate（src/main.rs 或 src/bin/*.rs）
  - 可选：examples/、tests/、benches/

典型结构：
my-project/
├── Cargo.toml       ← package 描述
└── src/
    ├── main.rs      ← binary crate 入口
    └── lib.rs       ← library crate 入口（可选）
```

### 2. Crate

```
Binary crate：
  - 有 fn main()
  - 编译后是可执行文件
  - 不能被其他 crate 直接 use

Library crate：
  - 没有 main()
  - 编译后是 .rlib 文件
  - 可以被其他 crate use
  - 用 src/lib.rs 作为根

同一个 Package 有 lib + binary 时：
  - binary 可以 use 本 package 的 library
  - 名字和 package 名一样
```

### 3. Module

```rust
// 在 src/lib.rs 或 src/main.rs 里声明模块
mod math {
    pub fn add(a: i32, b: i32) -> i32 { a + b }

    mod internal {  // 私有子模块
        pub fn helper() { }
    }
}

// 用 :: 路径访问
let result = math::add(1, 2);

// use 导入
use math::add;
let result = add(3, 4);
```

### 4. Workspace（多包项目）

```toml
# workspace 根目录的 Cargo.toml
[workspace]
members = [
    "crates/core",
    "crates/cli",
    "crates/web",
]
```

```
my-project/
├── Cargo.toml        ← workspace 根
├── Cargo.lock        ← 共享的依赖锁文件
└── crates/
    ├── core/         ← Package 1（library）
    ├── cli/          ← Package 2（binary）
    └── web/          ← Package 3（binary）
```

## 完整运行示例

```rust
pub fn run() {
    println!("=== Package / Crate / Module 关系 ===");
    println!();
    println!("本仓库的结构:");
    println!("  Rust/                     ← Workspace 根");
    println!("  ├── Cargo.toml            ← Workspace 配置");
    println!("  └── chapters/");
    println!("      ├── chapter01/        ← Package 1");
    println!("      │   ├── Cargo.toml    ← Package 定义");
    println!("      │   └── src/");
    println!("      │       ├── main.rs   ← Binary Crate 根");
    println!("      │       └── topic_01_first_program.rs  ← Module");
    println!("      └── chapter04/        ← Package 4（当前）");
    println!();
    println!("层次关系:");
    println!("  Workspace > Package > Crate > Module");
    println!("  一个 Workspace 含多个 Package");
    println!("  一个 Package 含最多1个 lib crate + 多个 bin crate");
    println!("  一个 Crate 含多个 Module");
}
```
