## 这一节解决什么问题

你已经知道了借用规则。现在来看函数签名里借用的**工程应用**：

```rust
// 这两种写法有什么区别？
fn greet_1(name: &String) { println!("Hello, {name}!"); }
fn greet_2(name: &str) { println!("Hello, {name}!"); }

// 哪种更好？为什么？
```

以及：

```rust
fn sum_1(nums: &Vec<i32>) -> i32 { nums.iter().sum() }
fn sum_2(nums: &[i32]) -> i32 { nums.iter().sum() }

// 两者都能工作，区别在哪里？
```

理解这些，能让你写出更地道、更通用的 Rust API——不会强迫调用方做不必要的转换。

---

## 一分钟结论

- 函数只读访问字符串时，用 `&str` 比 `&String` 更通用（接受字面量和 String）
- 函数只读访问集合时，用 `&[T]` 比 `&Vec<T>` 更通用（接受数组、切片、Vec）
- `&String` 能自动 deref 成 `&str`，`&Vec<T>` 能自动 deref 成 `&[T]`——这叫 **Deref Coercion**
- `&mut String` 和 `&mut Vec<T>` 在需要修改（push/insert 等改变长度）时才用
- `&mut [T]` 适合"原地改元素但不改长度"的场景

## 与其他语言对比

| 语言 | 通用字符串参数怎么写 |
|-----|-----------------|
| Java | `String`（只有一种，但是对象引用，不分只读）|
| C++ | `const std::string&` 或 `std::string_view`（C++17 起推荐 string_view）|
| Go | `string`（值类型，但内部共享数据，接近 Rust 的 &str）|
| Python | `str`（不可变，类似 &str）|
| Rust | `&str`（最通用，接受字面量 + String + Box<str> 等）|

C++ 的 `std::string_view` 和 Rust 的 `&str` 非常相似——都是"胖指针：地址+长度，不拥有数据"。

## 核心概念与心智模型

### Deref Coercion（解引用强转）

Rust 在特定位置会自动把"更具体"的类型强转成"更抽象"的借用类型：

```
&String → &str    （通过 String: Deref<Target=str>）
&Vec<T> → &[T]    （通过 Vec<T>: Deref<Target=[T]>）
&Box<T> → &T      （通过 Box<T>: Deref<Target=T>）
```

这意味着：

```rust
fn accept_str(s: &str) { println!("{s}"); }

let owned = String::from("hello");
accept_str("literal");   // &str 字面量 → 直接
accept_str(&owned);       // &String → 自动变 &str
accept_str(&owned[1..]);  // &str 切片 → 直接
```

```
为什么 &String 能自动变 &str？
因为 String 实现了 Deref<Target = str>，
&String → deref() → &str（在需要 &str 的地方自动触发）
```

### `&[T]` vs `&Vec<T>` 的类型包容关系

```
能传给 &Vec<i32> 的：  Vec<i32>
能传给 &[i32] 的：    Vec<i32>（通过 Deref）
                       [i32; N]（数组，通过 &arr[..]）
                       &[i32]（切片）
                       Box<[i32]>（通过 Deref）

所以 &[T] 接受的类型更多，API 更通用
```

## 详细原理

### 1. `&str` vs `&String`

```rust
fn measure_len(s: &str) -> usize {
    s.len()
}

let owned = String::from("hello, rust");
let literal: &str = "hello, world";

// 下面三种调用都合法：
println!("{}", measure_len("literal str")); // ← &'static str → &str
println!("{}", measure_len(&owned));        // ← &String → &str（Deref）
println!("{}", measure_len(&owned[..5]));   // ← &str → &str

// 如果改成 &String，只有 &owned 这种用法合法
fn measure_len_string(s: &String) -> usize { s.len() }
// measure_len_string("literal"); // ❌ &str 不能传给 &String
```

### 2. `&[T]` vs `&Vec<T>`

```rust
fn sum(nums: &[i32]) -> i32 {
    nums.iter().sum()
}

let v = vec![1, 2, 3, 4, 5];
let arr: [i32; 3] = [10, 20, 30];

// 全部合法：
println!("{}", sum(&v));              // &Vec<i32> → &[i32]（Deref）
println!("{}", sum(&arr));            // &[i32; 3] → &[i32]（Coerce）
println!("{}", sum(&v[2..]));         // &[i32] 切片 → 直接
println!("{}", sum(&[100, 200, 300])); // 数组字面量 → &[i32]
```

### 3. `&mut String` vs `&mut Vec<T>`：什么时候真的需要

```rust
// 只需要改元素，不需要改长度/容量
fn uppercase_all(words: &mut [String]) {
    for w in words.iter_mut() {
        *w = w.to_uppercase();
    }
}

// 需要改长度（push/pop/insert 等）
fn append_suffix(v: &mut Vec<String>, suffix: &str) {
    for w in v.iter_mut() {
        w.push_str(suffix);
    }
    v.push(format!("(added by append_suffix: {suffix})"));
}

// 需要改 String 内容（push_str 等）
fn add_exclamation(s: &mut String) {
    s.push('!');
}
```

### 4. 混合参数：只读 + 只读 / 只读 + 可改

```rust
// 常见模式：一个参数提供上下文，另一个参数被修改
fn prepend_label(label: &str, target: &mut String) {
    let original = target.clone();
    *target = format!("{label}: {original}");
}

let mut message = String::from("hello world");
prepend_label("INFO", &mut message);
println!("{message}"); // "INFO: hello world"
```

## 完整运行示例

```rust
// ===== 字符串接口：&str vs &String =====

fn print_length(s: &str) {
    println!("  长度: {} (via &str)", s.len());
}

fn modify_string(s: &mut String, suffix: &str) {
    s.push_str(suffix);
}

// ===== 切片接口：&[T] vs &Vec<T> =====

fn sum_values(nums: &[i32]) -> i32 {
    nums.iter().sum()
}

fn double_in_place(nums: &mut [i32]) {
    for n in nums.iter_mut() {
        *n *= 2;
    }
}

fn append_and_sort(nums: &mut Vec<i32>, extra: i32) {
    nums.push(extra);
    nums.sort();
}

fn main() {
    println!("=== &str：最通用的字符串只读接口 ===");
    let owned = String::from("hello, rust");
    print_length("字面量");          // ✅ &str
    print_length(&owned);             // ✅ &String → &str (Deref)
    print_length(&owned[..5]);        // ✅ &str 切片

    println!();

    println!("=== &mut String：需要改内容时 ===");
    let mut s = String::from("hello");
    modify_string(&mut s, " world!");
    println!("  修改后: {s}");

    println!();

    println!("=== &[T]：最通用的切片只读接口 ===");
    let vec_data = vec![1, 2, 3, 4, 5];
    let arr_data: [i32; 4] = [10, 20, 30, 40];

    println!("  sum(vec)    = {}", sum_values(&vec_data));
    println!("  sum(array)  = {}", sum_values(&arr_data));
    println!("  sum(slice)  = {}", sum_values(&vec_data[1..4]));
    println!("  sum([1,2,3])= {}", sum_values(&[1, 2, 3]));

    println!();

    println!("=== &mut [T]：原地改元素（不改长度）===");
    let mut nums = vec![1, 2, 3, 4, 5];
    double_in_place(&mut nums);
    println!("  double_in_place: {nums:?}");

    // 也可以对数组用：
    let mut arr = [1, 2, 3];
    double_in_place(&mut arr);
    println!("  double array: {arr:?}");

    println!();

    println!("=== &mut Vec<T>：需要改长度时 ===");
    let mut v = vec![3, 1, 4];
    append_and_sort(&mut v, 2);
    println!("  append_and_sort: {v:?}");
}
```

## 编译器错误分析

### ❌ 传 `&String` 给接受 `&str` 的函数——这其实没错，自动 Deref

```rust
fn f(s: &str) { println!("{s}"); }
let s = String::from("hi");
f(&s); // ✅ 自动 Deref
```

### ❌ 传 `&str` 给接受 `&String` 的函数

```rust
fn g(s: &String) { println!("{s}"); }
g("hello"); // ❌ &str 不能传给 &String
```

```text
error[E0308]: mismatched types
  |
  | g("hello");
  |   ^^^^^^^
  |   expected `&String`, found `&str`
  |
  = note: expected reference `&String`
             found reference `&str`
```

**原因**：Deref coercion 是单向的（从更具体到更抽象），`&str` 不能自动变 `&String`。

**修复**：改函数签名为 `fn g(s: &str)`，或者调用方写 `g(&String::from("hello"))`（但这很蠢，不推荐）。

### ❌ `&mut [T]` 和 `&mut Vec<T>` 混淆

```rust
fn append(v: &mut [i32], x: i32) {
    // v.push(x); // ❌ &mut [i32] 是固定长度视图，没有 push
}

fn append_correct(v: &mut Vec<i32>, x: i32) {
    v.push(x); // ✅ Vec 有 push
}
```

## 实际工程场景

### 1. 实现通用搜索函数

```rust
fn find<T: PartialEq>(haystack: &[T], needle: &T) -> Option<usize> {
    haystack.iter().position(|item| item == needle)
}

let words = vec!["rust", "is", "great"];
let numbers = vec![1, 2, 3, 4, 5];

println!("{:?}", find(&words, &"is"));    // Some(1)
println!("{:?}", find(&numbers, &3));    // Some(2)
println!("{:?}", find(&words, &"go"));   // None

// 同一个函数，数组也能用
let arr = [10, 20, 30];
println!("{:?}", find(&arr, &20)); // Some(1)
```

### 2. 标准库 API 的设计风格（模仿）

```rust
// 标准库：
// fn split_at(&self, mid: usize) -> (&str, &str)  ← 返回两个切片（零拷贝）
// fn trim(&self) -> &str                           ← 返回原字符串的子切片

// 模仿：实现一个按行分割的函数
fn first_line(text: &str) -> &str {
    text.lines().next().unwrap_or("")
}

let paragraph = "第一行\n第二行\n第三行";
println!("{}", first_line(paragraph)); // "第一行"
// 注意：返回的 &str 指向 paragraph 原始字节，零拷贝
```

### 3. 配置修改（`&mut` 的工程场景）

```rust
#[derive(Debug, Default)]
struct Config {
    debug: bool,
    log_level: String,
    timeout_ms: u64,
}

fn apply_debug_settings(cfg: &mut Config) {
    cfg.debug = true;
    cfg.log_level = "DEBUG".into();
    cfg.timeout_ms = 30_000;
}

let mut cfg = Config::default();
apply_debug_settings(&mut cfg);
println!("{cfg:?}");
```

## 性能影响

| 函数签名 | 适用场景 | 调用方成本 |
|---------|---------|---------|
| `fn f(s: &str)` | 只读访问字符串 | 8 字节（地址），或 16 字节（len+ptr）|
| `fn f(s: &String)` | 需要 String 的元数据（cap 等）| 8 字节，但限制了输入类型 |
| `fn f(s: &[T])` | 只读访问序列 | 16 字节（ptr+len）|
| `fn f(s: &Vec<T>)` | 需要 Vec 的容量信息 | 8 字节，但限制了输入类型 |
| `fn f(s: &mut [T])` | 原地修改，长度不变 | 16 字节（ptr+len）|
| `fn f(s: &mut Vec<T>)` | 可以 push/pop | 8 字节，但限制了输入类型 |

**建议**：API 里的"接收者"总是选择最宽泛的类型：字符串用 `&str`，序列用 `&[T]`，需要修改用 `&mut [T]`，只有真的需要 push/pop 才用 `&mut Vec<T>`。

## 注意点与陷阱

### 陷阱 1：返回借用时，生命周期必须关联到输入

```rust
// 这个函数返回了一个对输入数据的借用
fn first_word(s: &str) -> &str {
    s.split_whitespace().next().unwrap_or("")
}
// 编译器自动推断：返回的 &str 和 s 的生命周期绑定

// 下面这个是错的（返回了局部变量的借用）：
fn bad() -> &str {
    let s = String::from("hello");
    &s  // ❌ s 在函数结束时 drop，返回悬垂引用
}   // s 在这里被 drop
```

### 陷阱 2：`&mut [T]` 不能 push，只能改元素

```rust
fn try_push(v: &mut [i32]) {
    // v.push(42); // ❌ 切片引用 [i32] 没有 push 方法
    v[0] = 42; // ✅ 可以修改元素
}
```

## 我的理解与记忆方法

**API 设计的"最宽泛原则"**：

> 函数参数应该接受"能完成任务的最宽泛类型"，不要强迫调用方转换。

**速记**：

```
需要读字符串 → &str（不是 &String）
需要读序列   → &[T]（不是 &Vec<T>）
需要改元素   → &mut [T]
需要改长度   → &mut Vec<T>
需要拥有     → T（String 或 Vec<T>）
```
