## 扩展演示输出（当前代码已升级）

`topic_01_closures.rs` 现在系统演示闭包的 3 种捕获方式 + 对应 3 个 trait（`Fn ⊂ FnMut ⊂ FnOnce`）：
- `Fn`：不可变捕获 `min_length` 做校验
- `FnMut`：可变捕获 `count` 做计数
- `move` + `make_adder` 返回一个 `impl Fn`，把 `delta` 的所有权搬进闭包
- `FnOnce`：`drop(name)` 把捕获的 String 消费掉——闭包只能调一次

```text
-- (3) move：强制按值捕获 --
adder5(3)  = 8
adder10(3) = 13

-- (4) FnOnce：闭包只能调一次 --
FnOnce 闭包只能调 1 次（drop 掉了内部捕获的 String）
```

## 定义

闭包是匿名的可调用值。它看起来像函数，但还能捕获定义处周围的环境。

## 作用

- 把短小行为当成参数传入函数
- 延迟决定具体逻辑
- 在不额外定义命名函数的情况下携带上下文

## 原理

闭包会根据用法自动推断参数和返回值，并根据是否捕获环境、是否修改环境，分别实现 `Fn`、`FnMut`、`FnOnce` 等 trait。

## 最小示例

```rust
let min_length = 4;
let is_valid_user = |name: &str| name.len() >= min_length;
```

## 注意点

- 闭包和函数最大的区别是“能否捕获环境”
- 捕获方式会影响它实现的是哪一个 `Fn` trait
- 零捕获闭包有时可以退化成函数指针

## 常见错误

- 把闭包当成普通函数理解，忽略外部变量捕获
- 不看签名约束，就把任意闭包传给函数
- 遇到 move 闭包时不知道为什么外部值被拿走

## 我的理解

闭包让“行为”真正成为一等值。后面很多迭代器组合子之所以顺手，本质上就是因为闭包能很自然地嵌进去。
