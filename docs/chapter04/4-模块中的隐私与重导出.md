## 一分钟结论

- `pub use` 可以整理对外暴露路径
- 但它不能突破现有隐私边界
- `pub struct` 不等于字段自动公开
- 常见设计方式是：字段私有，提供 `new()` 和方法

## 定义

这节关注两个相关话题：

- **隐私**：哪些字段 / 项不该直接暴露
- **重导出**：怎样整理对外 API 路径

## 作用

- 隐藏内部表示
- 保持创建方式受控
- 对外提供更稳定、更干净的 API 路径

## 原理

当前示例里：

```rust
pub use self::catalog::{Category, Product};
```

这表示：

- 外部可以从更浅路径拿到这些名字
- 不必关心它们内部原本在 `catalog` 模块下

同时：

```rust
pub struct Product {
    id: u32,
    name: String,
    category: Category,
}
```

虽然 `Product` 是公开类型，但字段仍然私有。

## 最小示例

```rust
pub struct Product {
    id: u32,
    name: String,
}

impl Product {
    pub fn new(id: u32, name: String) -> Self {
        Self { id, name }
    }
}
```

## 注意点

### 1. `pub struct` 只公开类型名，不自动公开字段

这是很多新手第一次读 Rust 时会误判的点。

### 2. `pub use` 是整理路径，不是绕过私有规则

如果底层本来不可见，它不会被魔法放出来。

### 3. 构造函数是保持约束的好方法

比起暴露全部字段，`new()` 往往更能保持不变量。

## 常见错误

### ❌ 错误 1：以为 `pub struct` 就能外部字面量初始化所有字段

### ❌ 错误 2：用 `pub use` 试图绕过隐私边界

### ❌ 错误 3：为了省事直接公开所有字段

短期方便，长期会丧失类型约束。

- 这一节是在教“怎么设计对外 API”
- 真正重要的不是能不能访问，而是该不该让别人直接访问
