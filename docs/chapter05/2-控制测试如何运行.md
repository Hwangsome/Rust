## 这一节解决什么问题

随着测试变多，你需要控制运行哪些测试、如何查看输出、如何处理慢测试或条件性测试。

---

## 一分钟结论

- `cargo test` 并发运行所有测试
- `cargo test 函数名` 按名字过滤（支持子串匹配）
- `#[ignore]` 标记慢测试/条件性测试，用 `--ignored` 单独运行
- `-- --show-output` 看到测试的 println! 输出（默认捕获）
- `-- --test-threads=1` 单线程运行（调试顺序依赖问题）
- `-- --list` 列出所有测试名但不运行

---

## 常用命令速查

```bash
# 运行所有测试
cargo test -p chapter05

# 按名字过滤（子串匹配）
cargo test -p chapter05 add        # 所有名字含 add 的测试
cargo test -p chapter05 tests::    # 所有在 tests 模块里的

# 只运行 ignored 测试
cargo test -p chapter05 -- --ignored

# 同时运行正常 + ignored
cargo test -p chapter05 -- --include-ignored

# 显示测试的 println! 输出
cargo test -p chapter05 -- --show-output

# 单线程串行运行（调试用）
cargo test -p chapter05 -- --test-threads=1

# 列出所有测试名（不执行）
cargo test -p chapter05 -- --list

# 失败时不停止
cargo test -p chapter05 -- --no-fail-fast
```

## 详细原理

### `#[ignore]`：标记暂时不运行的测试

```rust
#[test]
#[ignore = "需要外部服务，CI 环境里跳过"]
fn test_database_connection() {
    // 依赖真实数据库，不在 unit test 里跑
    todo!()
}

#[test]
#[ignore]  // 也可以不写原因
fn slow_test() {
    std::thread::sleep(std::time::Duration::from_secs(30));
}
```

### 输出捕获

```rust
#[test]
fn test_with_print() {
    println!("这条信息默认被捕获，测试通过时不显示");
    assert_eq!(2 + 2, 4);
}
// 用 -- --show-output 才能看到 println!
```

## 完整运行示例

```rust
pub fn is_prime(n: u64) -> bool {
    if n < 2 { return false; }
    if n == 2 { return true; }
    if n % 2 == 0 { return false; }
    let sqrt = (n as f64).sqrt() as u64;
    (3..=sqrt).step_by(2).all(|i| n % i != 0)
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_small_primes() {
        assert!(is_prime(2));
        assert!(is_prime(3));
        assert!(is_prime(5));
        assert!(is_prime(7));
    }

    #[test]
    fn test_non_primes() {
        assert!(!is_prime(0));
        assert!(!is_prime(1));
        assert!(!is_prime(4));
        assert!(!is_prime(100));
    }

    // 过滤示例
    #[test]
    fn prime_number_check() {
        // cargo test prime 可以匹配到这两个
        assert!(is_prime(97));
    }

    // ignore 示例
    #[test]
    #[ignore = "计算大质数耗时较长"]
    fn test_large_prime() {
        assert!(is_prime(1_000_000_007));  // 慢
    }

    // should_panic 示例
    #[test]
    #[should_panic]
    fn test_overflow_should_panic() {
        // 在 debug 模式下，整数溢出会 panic
        let _: u8 = 255u8.wrapping_add(1); // 实际不 panic（wrapping）
        // 这个测试只是示例
        panic!("手动 panic");
    }
}
```
