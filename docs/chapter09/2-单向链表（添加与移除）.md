## 定义

这一节在单向链表上增加最小操作：头插、头删和遍历。

## 作用

- 练习 `Option::take()` 的所有权技巧
- 理解链表操作为什么经常围绕 head 展开
- 让前一节的静态结构变成动态结构

## 原理

`take()` 会把 `self.head` 里的值拿走，并把原位置留成 `None`。这样我们就能在不违反借用规则的前提下，临时拥有旧头节点，再安全地重新拼接链表。

## 最小示例

```rust
fn push_front(&mut self, element: i32) {
    let previous_head = self.head.take();
    self.head = Some(Box::new(Node {
        element,
        next: previous_head,
    }));
}
```

## 注意点

- 教学版链表通常先实现头插 / 头删，因为它们最直接
- 遍历时常用 `while let Some(node) = current`
- 很多链表技巧的核心都是“谁暂时拥有旧节点”

## 常见错误

- 直接从 `self.head` 里 move 值，导致借用冲突
- 修改链表时没先想清楚旧链接去哪了
- 遍历时不小心把节点 move 掉

## 我的理解

这一节让我感觉最明显的是：链表不是算法难，而是所有权路径必须设计清楚。
