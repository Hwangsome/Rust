## 一分钟结论

- `if` 的条件必须是 `bool`
- `loop` 适合手动控制退出
- `while` 适合条件型循环
- `for` + `range` 是最常见、最清晰的遍历方式

## 作用

控制流决定程序走哪条路径。  
如果没有这部分，程序就只能从上到下线性执行。

## 最小示例

```rust
fn main() {
    for number in 1..=3 {
        println!("{number}");
    }
}
```

输出：

```text
1
2
3
```

## 关键点

### `if` 条件必须明确为布尔值

Rust 不会像某些语言那样把整数自动当真假值。

### `loop` 可以配合 `break`

```rust
let mut count = 0;

loop {
    count += 1;
    if count == 3 {
        break;
    }
}
```

### `for` + `range` 最适合入门

- `1..5` 表示不包含 `5`
- `1..=5` 表示包含 `5`

## 常见错误

### ❌ 把整数直接写进 `if`

```rust
fn main() {
    let x = 1;

    if x {
        println!("hello");
    }
}
```

Rust 不会把整数自动转换成布尔值，所以这里会报类型错误。

## 我的理解

这一节最值得记住的是：Rust 希望控制流的意图足够明确。

- `if` 要的就是 `bool`
- `loop` 要你自己决定退出条件
- `while` 和 `for` 的语义边界更清楚

这种“显式”设计，和变量默认不可变的思路是一致的。
