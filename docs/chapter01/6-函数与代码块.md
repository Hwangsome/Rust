## 一分钟结论

- Rust 用 `fn` 定义函数
- 函数参数必须写类型
- 代码块不仅能组织代码，还能产生值
- 返回值既可以显式 `return`，也可以用尾表达式

## 定义

函数是可复用的逻辑单元。  
代码块是由 `{}` 包起来的一段表达式和语句，它本身也可能产生值。

## 作用

学习函数和代码块，主要是为了回答两个问题：

- 怎样把重复逻辑抽出来
- Rust 为什么经常“不写 return 也能返回值”

## 最小示例

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn main() {
    let result = add(2, 3);
    println!("{result}");
}
```

输出：

```text
5
```

## 关键点

### 参数必须标注类型

Rust 不允许在函数签名里省略参数类型，因为函数边界是最重要的类型信息来源之一。

### 代码块可以返回值

```rust
let number = {
    let base = 5;
    base + 1
};
```

这里 `base + 1` 没有分号，所以它是这个代码块的返回值。

## 常见错误

### ❌ 误把带分号的语句当返回值

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b;
}
```

这里最后一行因为带分号，变成了一条语句，不再是返回值表达式。

## 我的理解

函数帮你解决“逻辑复用”，代码块帮你解决“局部组织和局部求值”。

Rust 初学阶段最容易忽略的是：

- 代码块不仅仅是包一段代码
- 它本身也可以产生值

一旦理解这一点，很多 Rust 写法就会自然得多。
