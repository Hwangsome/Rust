## 定义

具体生命周期指的是代码里某个引用实际有效的那一段时间。它不一定等同于词法作用域的整段范围。

## 作用

- 帮编译器判断引用是否还能安全使用
- 决定某些借用何时结束
- 解释为什么同一作用域里有时还能继续修改原值

## 原理

Rust 现在会使用非词法生命周期（NLL）分析。也就是说，一个引用在“最后一次被使用后”就可以视为结束，而不是必须等到整个花括号结束。

## 最小示例

```rust
let mut values = vec![10, 20, 30];
let first = &values[0];
println!("{}", first);
values.push(40);
```

## 注意点

- 生命周期分析关注“引用还会不会再被用到”
- 不是所有同作用域代码都天然冲突
- 看不懂时就问：这个借用最后一次真正使用在哪

## 常见错误

- 以为引用一定活到作用域末尾
- 只凭视觉判断借用冲突，不看最后使用点
- 把 NLL 理解成“借用规则放松了”

## 我的理解

这一节最重要的收获是：生命周期不是玄学，它其实和“这段引用后面还会不会再被读到”强相关。
