## 这一节解决什么问题

你看到 Rust 代码里很多 `#[derive(Debug)]`、`#[allow(dead_code)]` 这样的语法，却不知道它们是什么、怎么用。

这些叫做 **attribute（属性/编译器指令）**，是 Rust 向编译器传递"元信息"的方式：

```rust
#[derive(Debug, Clone)]          // 自动实现 Debug 和 Clone
#[allow(dead_code)]               // 允许有未使用的代码（不报警告）
#[cfg(target_os = "linux")]       // 只在 Linux 上编译
#[test]                           // 标记这是测试函数
```

---

## 一分钟结论

- 两种语法：`#[...]` 作用于**紧接的单个 item**；`#![...]` 作用于**整个文件/模块**
- 最常用：`#[derive(...)]`（自动实现 trait）
- 控制警告：`#[allow(...)]`、`#[warn(...)]`、`#[deny(...)]`
- 条件编译：`#[cfg(...)]`（如 `#[cfg(test)]`、`#[cfg(debug_assertions)]`）
- 测试：`#[test]`、`#[should_panic]`、`#[ignore]`
- 优化提示：`#[inline]`、`#[cold]`

## 详细原理

### 1. `#[derive]`：自动生成 trait 实现

```rust
#[derive(Debug, Clone, PartialEq)]
struct Point {
    x: f64,
    y: f64,
}

let p1 = Point { x: 1.0, y: 2.0 };
let p2 = p1.clone();  // 来自 Clone
println!("{p1:?}");   // 来自 Debug：Point { x: 1.0, y: 2.0 }
println!("{}", p1 == p2);  // 来自 PartialEq：true
```

### 2. 控制 lint（警告/错误）

```rust
// 全局：允许整个文件有未使用变量
#![allow(unused_variables)]

// 局部：只对某个函数允许
#[allow(dead_code)]
fn never_called() { println!("从不被调用"); }

// 也可以提升为错误
#[deny(unsafe_code)]  // 禁止 unsafe 代码（违反会报错）
fn safe_function() { }
```

### 3. 条件编译 `#[cfg]`

```rust
// 只在测试时编译
#[cfg(test)]
mod tests {
    #[test]
    fn it_works() {
        assert_eq!(2 + 2, 4);
    }
}

// 只在特定平台编译
#[cfg(target_os = "windows")]
fn windows_only() { println!("Windows!"); }

#[cfg(debug_assertions)]  // debug 模式下
fn debug_info() { println!("调试信息"); }
```

### 4. 其他常用 attribute

```rust
// 函数可能内联
#[inline]
fn add(a: i32, b: i32) -> i32 { a + b }

// 标记不稳定的 API（库开发时用）
#[deprecated(since = "1.2.0", note = "用 new_func 代替")]
fn old_func() { }

// 必须处理返回值（忘了用会报警告）
#[must_use = "这个结果必须被使用"]
fn important_operation() -> i32 { 42 }

// 允许 unsafe（通常在 FFI 中）
unsafe fn dangerous() { }
```

## 完整运行示例

```rust
// 文件级 attribute：允许未使用变量
#![allow(unused_variables)]

#[derive(Debug, Clone, PartialEq)]
struct Config {
    host: String,
    port: u16,
    debug: bool,
}

#[derive(Debug)]
#[allow(dead_code)]  // 允许未使用字段
struct LegacyData {
    old_field: i32,
    another_old: String,
}

#[must_use = "请检查操作是否成功"]
fn validate_port(port: u16) -> Result<(), String> {
    if port == 0 {
        Err("端口不能为 0".into())
    } else {
        Ok(())
    }
}

fn main() {
    println!("=== derive 自动实现 ===");
    let cfg = Config {
        host: "localhost".into(),
        port: 8080,
        debug: true,
    };
    let cfg2 = cfg.clone();
    println!("{cfg:?}");
    println!("相等: {}", cfg == cfg2);
    println!();

    println!("=== must_use 警告 ===");
    // validate_port(0); // ← 会触发 must_use 警告
    match validate_port(0) {
        Ok(()) => println!("端口有效"),
        Err(e) => println!("无效端口: {e}"),
    }
    println!();

    println!("=== cfg：条件编译 ===");
    #[cfg(debug_assertions)]
    println!("当前是 debug 模式");

    #[cfg(not(debug_assertions))]
    println!("当前是 release 模式");
}

// cfg(test) 代码块只在 `cargo test` 时编译
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_validate_port() {
        assert!(validate_port(8080).is_ok());
        assert!(validate_port(0).is_err());
    }

    #[test]
    #[should_panic]
    fn test_panic() {
        panic!("这个测试期望 panic");
    }
}
```

## 重要 attribute 速查表

| Attribute | 用途 |
|-----------|-----|
| `#[derive(T)]` | 自动实现 T trait |
| `#[allow(lint)]` | 允许某类 lint |
| `#[warn(lint)]` | 警告某类 lint |
| `#[deny(lint)]` | 将 lint 设为错误 |
| `#[cfg(cond)]` | 条件编译 |
| `#[test]` | 标记测试函数 |
| `#[should_panic]` | 期望测试 panic |
| `#[ignore]` | 忽略测试 |
| `#[inline]` | 内联建议 |
| `#[deprecated]` | 标记弃用 |
| `#[must_use]` | 强制使用返回值 |
| `#[repr(C)]` | C 兼容内存布局 |
