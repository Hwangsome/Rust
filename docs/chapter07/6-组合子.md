## 定义

组合子是迭代器上的一类方法，例如 `map`、`filter`、`collect`。它们把一个处理步骤接到另一个步骤后面，形成可读的变换链。

## 作用

- 把数据处理拆成多个小步骤
- 避免手写大量临时变量和显式循环
- 利用惰性求值按需执行

## 原理

像 `filter`、`map` 这样的适配器通常不会立即执行，它们只是构造出新的迭代器。真正的消费通常发生在 `collect`、`sum`、`for_each`、`for` 循环这类终结步骤。

## 最小示例

```rust
let even_squares: Vec<i32> = numbers
    .into_iter()
    .filter(|number| number % 2 == 0)
    .map(|number| number * number)
    .collect();
```

## 注意点

- 组合子不是“函数式装饰”，它们依赖 Iterator 协议工作
- 读不懂时就看“每一步输入什么、输出什么”
- 链太长时可以拆中间变量，提升可读性

## 常见错误

- 以为 `map`、`filter` 会立刻执行
- 闭包里副作用过多，导致链式风格难读
- 为了链而链，忽略中间变量反而更清晰

## 我的理解

组合子让“遍历”和“变换”贴在一起表达。它的价值不是更炫，而是让数据流更直。
