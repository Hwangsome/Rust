## 这一节解决什么问题

闭包很强大，但有时候你不需要那么强大——你只需要"指向某个函数的指针"，不需要捕获环境。

函数指针（`fn(T) -> U`）是一种**更简单、更轻量**的类型：

- 大小固定（8 字节，一个地址）
- 不捕获任何环境
- 可以在 C FFI 中使用（闭包不能）
- 编译期已知具体函数地址

---

## 一分钟结论

- `fn(i32) -> i32` 是**函数指针类型**（不是 `Fn` trait）
- 函数名本身就是一个函数指针（可以赋给 `fn` 类型变量）
- **没有捕获的闭包**可以自动转换为函数指针
- **有捕获的闭包**不能转换为函数指针（它额外存储了捕获变量）
- `fn` 实现了 `Fn`、`FnMut`、`FnOnce` 三个 trait（函数指针比闭包更"通用"）
- 需要 FFI（与 C 交互）时必须用 `fn` 指针，不能用闭包

---

## 与其他语言对比

| 语言 | "可调用的指针" |
|-----|------------|
| C | `int (*fp)(int, int)` 函数指针 |
| C++ | 函数指针 + `std::function<>` + lambda |
| Java | 方法引用（`String::length`）→ 函数式接口 |
| Python | 函数就是对象，直接传递 |
| Go | `func(int) int` 函数值 |
| Rust | `fn(i32) -> i32`（轻量）或 `impl Fn(i32) -> i32`（强大）|

## 详细原理

### 1. 函数指针的基础用法

```rust
fn double(x: i32) -> i32 { x * 2 }
fn square(x: i32) -> i32 { x * x }
fn add_one(x: i32) -> i32 { x + 1 }

// fn 类型的变量
let op: fn(i32) -> i32 = double;
println!("{}", op(5));  // 10

// 函数指针可以在数组/Vec 里
let operations: [fn(i32) -> i32; 3] = [double, square, add_one];
for op in &operations {
    println!("{}", op(4));  // 8, 16, 5
}

// 作为参数传递
fn apply(x: i32, f: fn(i32) -> i32) -> i32 {
    f(x)
}
println!("{}", apply(3, double));  // 6
println!("{}", apply(3, square));  // 9
```

### 2. 零捕获闭包 → 函数指针（自动转换）

```rust
fn apply_fn(x: i32, f: fn(i32) -> i32) -> i32 { f(x) }

// 零捕获闭包可以自动转换为函数指针
let result = apply_fn(5, |x| x * 3);  // ✅
println!("{result}");  // 15

// 有捕获的闭包不能
let factor = 4;
// apply_fn(5, |x| x * factor); // ❌ 捕获了 factor
```

### 3. `fn` vs `Fn` vs `impl Fn`

```rust
// fn(i32) -> i32：具体类型，8字节，不能接受有捕获的闭包
fn uses_fn_pointer(f: fn(i32) -> i32) -> i32 { f(5) }

// impl Fn(i32) -> i32：接受函数指针和所有闭包（推荐）
fn uses_impl_fn(f: impl Fn(i32) -> i32) -> i32 { f(5) }

// Box<dyn Fn(i32) -> i32>：接受函数指针和所有闭包，可以存储
fn uses_boxed_fn(f: Box<dyn Fn(i32) -> i32>) -> i32 { f(5) }

let captured = 100;
let closure = |x| x + captured;

// uses_fn_pointer(closure);  // ❌ 不行
uses_impl_fn(closure);         // ✅
uses_impl_fn(add_one);         // ✅ fn 指针也满足 Fn
uses_boxed_fn(Box::new(closure)); // ✅
```

### 4. 函数指针与枚举的配合（查找表）

```rust
#[derive(Debug)]
enum Operation { Double, Square, Negate }

fn get_op(op: Operation) -> fn(i32) -> i32 {
    match op {
        Operation::Double => |x| x * 2,  // 零捕获 → fn 指针
        Operation::Square => |x| x * x,
        Operation::Negate => |x| -x,
    }
}

let ops = [Operation::Double, Operation::Square, Operation::Negate];
for op in ops {
    let f = get_op(op);
    println!("f(5) = {}", f(5));  // 10, 25, -5
}
```

## 完整运行示例

```rust
use std::mem::size_of;

fn add(a: i32, b: i32) -> i32 { a + b }
fn sub(a: i32, b: i32) -> i32 { a - b }
fn mul(a: i32, b: i32) -> i32 { a * b }

fn calculate(a: i32, b: i32, op: fn(i32, i32) -> i32) -> i32 {
    op(a, b)
}

fn make_op(name: &str) -> fn(i32, i32) -> i32 {
    match name {
        "add" => add,
        "sub" => sub,
        "mul" => mul,
        _     => |_, _| 0,
    }
}

fn main() {
    println!("=== 函数指针大小 ===");
    println!("fn(i32) -> i32 = {} bytes", size_of::<fn(i32) -> i32>());  // 8
    println!("fn item (add)  = {} bytes", size_of_val(&add));              // 0（函数项不是胖指针）

    println!();

    println!("=== 基础用法 ===");
    let op: fn(i32, i32) -> i32 = add;
    println!("op(3, 4) = {}", op(3, 4));

    let ops: Vec<fn(i32, i32) -> i32> = vec![add, sub, mul];
    for f in &ops {
        println!("f(10, 3) = {}", f(10, 3));  // 13, 7, 30
    }

    println!();

    println!("=== 高阶函数 ===");
    println!("calculate(10, 4, add) = {}", calculate(10, 4, add));
    println!("calculate(10, 4, sub) = {}", calculate(10, 4, sub));

    println!();

    println!("=== 动态选择操作 ===");
    for name in ["add", "sub", "mul", "unknown"] {
        let f = make_op(name);
        println!("  {name}(7, 3) = {}", f(7, 3));
    }

    println!();

    println!("=== 零捕获闭包 → fn 指针 ===");
    let double: fn(i32) -> i32 = |x| x * 2;  // 零捕获，可以转换
    println!("double(5) = {}", double(5));

    // 有捕获的闭包不能转换
    let factor = 3;
    let triple = |x: i32| x * factor;  // 捕获了 factor
    // let typed: fn(i32) -> i32 = triple; // ❌

    // 但可以用 impl Fn 接收
    fn apply_impl(x: i32, f: impl Fn(i32) -> i32) -> i32 { f(x) }
    println!("triple(5) = {}", apply_impl(5, triple));  // ✅
}
```

## 实际工程场景

### 1. FFI（与 C 交互）

```rust
// C 回调函数必须是函数指针，不能是闭包
extern "C" {
    fn qsort(
        base: *mut std::ffi::c_void,
        nmemb: usize,
        size: usize,
        compare: Option<unsafe extern "C" fn(*const std::ffi::c_void, *const std::ffi::c_void) -> i32>,
    );
}

// 实现比较函数
unsafe extern "C" fn compare_ints(
    a: *const std::ffi::c_void,
    b: *const std::ffi::c_void,
) -> i32 {
    let a = *(a as *const i32);
    let b = *(b as *const i32);
    a - b
}
```

### 2. 分发表（dispatch table）

```rust
type Handler = fn(&str) -> String;

struct Router {
    routes: std::collections::HashMap<String, Handler>,
}

impl Router {
    fn new() -> Self { Router { routes: Default::default() } }
    fn add(&mut self, path: &str, handler: Handler) {
        self.routes.insert(path.to_string(), handler);
    }
    fn handle(&self, path: &str) -> Option<String> {
        self.routes.get(path).map(|f| f(path))
    }
}
```

## 我的理解与记忆方法

**何时选函数指针 vs 闭包**：

```
需要存在 C FFI 里         → fn 指针（唯一选择）
需要存在固定大小的数组里  → fn 指针（零捕获）
需要捕获环境              → 闭包（impl Fn 或 Box<dyn Fn>）
不确定                    → 用 impl Fn（接受两者）
```
