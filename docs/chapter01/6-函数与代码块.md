## 一分钟结论

- Rust 用 `fn` 定义函数
- 函数参数必须写类型
- 函数体本身就是一个代码块
- 代码块不仅能组织代码，还能产生值
- 返回值既可以显式 `return`，也可以用尾表达式
- Rust 想“返回多个值”时，通常返回一个 tuple

## 定义

函数是可复用的逻辑单元。  
代码块是由 `{}` 包起来的一段表达式和语句，它本身也可能产生值。

这一节里还要顺带区分两个概念：

- 表达式：会产生值
- 语句：负责执行动作，本身不作为值继续参与计算

Rust 里一个很关键的规则是：**“没有分号的最后一个表达式”可以成为代码块或函数的返回值。**

## 作用

学习函数和代码块，主要是为了回答两个问题：

- 怎样把重复逻辑抽出来
- Rust 为什么经常“不写 return 也能返回值”

再往前走一步，还要理解：

- 为什么函数可以像“计算器”一样返回结果
- 为什么 `{}` 不只是包一段代码，而是可以直接参与赋值
- 为什么多返回值通常会写成 tuple

## 最小示例

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn main() {
    let result = add(2, 3);
    println!("{result}");
}
```

输出：

```text
5
```

## 原理

### 函数体本身就是代码块

下面这段代码里，`{ a + b }` 就是函数体。  
它本质上仍然是一个代码块，所以“最后一个表达式返回值”的规则同样适用。

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

### `return` 和尾表达式都能返回值

```rust
fn add_with_return(a: i32, b: i32) -> i32 {
    return a + b;
}

fn add_without_return(a: i32, b: i32) -> i32 {
    a + b
}
```

两种都合法。  
入门阶段可以先这样记：

- 正常收尾时，常用尾表达式
- 需要提前结束函数时，常用 `return`

### Rust 返回多个值，通常是返回一个 tuple

```rust
fn sum_and_product(a: i32, b: i32) -> (i32, i32) {
    (a + b, a * b)
}
```

调用时可以直接解构：

```rust
let (sum, product) = sum_and_product(3, 4);
```

这不是“特殊的多返回值语法”，而是“返回了一个 tuple 值”。

## 关键点

### 参数必须标注类型

Rust 不允许在函数签名里省略参数类型，因为函数边界是最重要的类型信息来源之一。

### 代码块可以返回值

```rust
let number = {
    let base = 5;
    base + 1
};
```

这里 `base + 1` 没有分号，所以它是这个代码块的返回值。

### 分号会让表达式变成语句

```rust
let unit_value = {
    let base = 5;
    base + 1;
};
```

这里最后一行多了分号，所以这个代码块不再返回 `6`，而是返回 `()`。

### 代码块还能制造更小的作用域

```rust
let outer = 10;

let scoped_result = {
    let outer = outer + 5;
    outer * 2
};
```

内层的 `outer` 只是临时绑定，不会改掉外层原来的 `outer`。

## 常见错误

### ❌ 误把带分号的语句当返回值

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b;
}
```

这里最后一行因为带分号，变成了一条语句，不再是返回值表达式。

### ❌ 想当然地认为 Rust 有“多个独立返回值”

```rust
fn wrong(a: i32, b: i32) -> i32, i32 {
    a + b, a * b
}
```

这种写法不合法。  
正确做法通常是返回 tuple：

```rust
fn correct(a: i32, b: i32) -> (i32, i32) {
    (a + b, a * b)
}
```

### ❌ 以为代码块里的临时变量出了代码块还能用

```rust
let number = {
    let base = 5;
    base + 1
};

println!("{base}");
```

这里的 `base` 只活在代码块内部，出了 `{}` 就不可用了。

## 我的理解

函数帮你解决“逻辑复用”，代码块帮你解决“局部组织和局部求值”。

Rust 初学阶段最容易忽略的是：

- 函数体其实也是代码块
- 代码块不仅仅是包一段代码
- 它本身也可以产生值
- 多返回值通常只是“返回一个 tuple”

一旦理解这一点，很多 Rust 写法就会自然得多。
