## 关键技巧：`Option::take()`

链表的头插/头删需要"临时取出旧头"再"放入新节点"。这里的关键是 `Option::take()`：

```rust
// take() = 把 Option 里的值取出来，原地替换为 None
// 等价于 std::mem::replace(&mut self.head, None)

let old_head = self.head.take(); // head 变成 None，old_head 拿到旧值
// 现在可以安全地修改 head，不存在"两个引用指向同一节点"的问题
```

---

## 详细原理

```rust
impl LinkedList {
    fn push_front(&mut self, value: i32) {
        let old_head = self.head.take(); // 取出旧头（head 变 None）
        let new_node = Box::new(Node {
            element: value,
            next: old_head, // 新节点的 next 指向旧头
        });
        self.head = Some(new_node); // 新头指向新节点
        self.size += 1;
    }

    fn pop_front(&mut self) -> Option<i32> {
        self.head.take().map(|node| { // 取出头节点（Box<Node>）
            self.head = node.next;     // 头变成原来的第二个节点
            self.size -= 1;
            node.element               // 返回原来头节点的值
        })
    }
}
```

---

## 完整运行示例

```rust
pub fn run() {
    let mut list = LinkedList::new();

    // 头插
    list.push_front(3);
    list.push_front(2);
    list.push_front(1);
    println!("push 1,2,3: {:?}", list.values()); // [1, 2, 3]

    // 头删
    while let Some(val) = list.pop_front() {
        println!("pop: {val}");
    }
    println!("清空后长度: {}", list.len()); // 0
}
```

---

## 注意：链表在 Rust 里很"麻烦"

Rust 的借用检查器对链表特别严格，因为链表的节点形成了复杂的所有权关系。实际工程中：

- 单向链表 → 通常用 `Vec` 或 `VecDeque`（性能更好，实现更简单）
- 双向链表 → 用 `std::collections::LinkedList`（标准库已有）

手写链表的价值在于理解所有权，而不是生产使用。
