## 这一节解决什么问题

随着代码增长，所有东西都放在一个文件里会变得混乱。模块系统让你把代码分到不同的命名空间和文件里，同时控制哪些部分对外可见。

---

## 一分钟结论

- `mod name { ... }` 声明内联模块
- `mod name;` 在单独文件 `name.rs` 或 `name/mod.rs` 里定义模块
- `use` 把路径引入当前作用域
- 绝对路径：从 `crate::` 开始
- 相对路径：从当前模块开始（`self::` 或直接名字）
- `super::` 访问父模块
- `use xxx as yyy` 起别名；`use xxx::{A, B, C}` 一次导入多个

## 详细原理

### 1. 内联模块

```rust
mod geometry {
    // 默认私有！外部不可见
    fn internal_helper(x: f64) -> f64 { x.abs() }

    // pub 才对外可见
    pub fn distance(x1: f64, y1: f64, x2: f64, y2: f64) -> f64 {
        let dx = x2 - x1;
        let dy = y2 - y1;
        (dx * dx + dy * dy).sqrt()
    }

    pub mod shapes {
        pub struct Circle { pub radius: f64 }
        pub struct Square { pub side: f64 }
    }
}

// 使用绝对路径
let d = geometry::distance(0.0, 0.0, 3.0, 4.0);

// use 导入
use geometry::shapes::Circle;
let c = Circle { radius: 5.0 };
```

### 2. 文件模块

```
src/
├── main.rs
├── network.rs          ← mod network; 对应这个文件
└── network/            ← 或者这个目录
    └── mod.rs
```

```rust
// src/main.rs
mod network;            // 从 src/network.rs 或 src/network/mod.rs 加载

use network::connect;
```

### 3. 路径类型

```rust
mod a {
    pub mod b {
        pub fn hello() { println!("hello from a::b"); }
    }

    pub fn use_b() {
        // 绝对路径（从 crate 根）
        crate::a::b::hello();

        // 相对路径（从当前模块）
        self::b::hello();  // self 可省略
        b::hello();

        // super（父模块）
        // super::something();
    }
}
```

### 4. `use` 的各种形式

```rust
// 基础
use std::collections::HashMap;

// 别名
use std::collections::HashMap as Map;

// 批量导入
use std::io::{self, Read, Write};
// 等同于：
// use std::io;
// use std::io::Read;
// use std::io::Write;

// 通配符（谨慎使用，可能导致命名冲突）
use std::prelude::*;

// 重新导出（pub use）
pub use crate::internal_module::PublicType;
// 现在外部代码可以 use your_crate::PublicType
```

## 完整运行示例

```rust
mod greetings {
    pub fn hello(name: &str) { println!("Hello, {name}!"); }
    pub fn goodbye(name: &str) { println!("Goodbye, {name}!"); }

    pub mod formal {
        pub fn greet(name: &str) { println!("Good day, {name}."); }

        // 用 super 访问父模块
        pub fn informal_then_formal(name: &str) {
            super::hello(name);  // 父模块的 hello
            self::greet(name);   // 自身的 greet
        }
    }
}

use greetings::{hello, goodbye};
use greetings::formal::greet;

pub fn run() {
    hello("Alice");
    goodbye("Bob");
    greet("Carol");
    greetings::formal::informal_then_formal("Dave");
}
```

## 模块文件结构示例（真实项目）

```
src/
├── lib.rs              // crate 根
├── auth/
│   ├── mod.rs          // mod auth; 对应这里
│   ├── password.rs     // mod password; 在 auth/mod.rs 里声明
│   └── token.rs        // mod token;
└── db/
    ├── mod.rs          // mod db;
    └── queries.rs      // mod queries;
```

```rust
// src/lib.rs
pub mod auth;
pub mod db;

// src/auth/mod.rs
mod password;
mod token;
pub use password::hash;
pub use token::Token;
```
