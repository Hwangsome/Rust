## 这一节解决什么问题

所有权在变量赋值时的行为比较直觉——到了函数调用边界就开始让新手困惑：

```rust
let v = vec![1, 2, 3];
use_vector(v);        // v 传进去了
println!("{v:?}");    // ❌ E0382！为什么 v 没了？
```

函数调用**和赋值一样**会触发 move——参数按值传入函数就是 move 进函数，调用方的原变量失效。

理解这个边界是设计 Rust API 的基础：**函数签名决定了调用方能不能继续使用被传入的值**。

---

## 一分钟结论

- 函数**按值**接收参数（`fn f(x: T)`）= x 被 move 进函数，调用方失去所有权
- 函数**按引用**接收参数（`fn f(x: &T)`）= 借用，调用方保留所有权
- 函数**按可变引用**接收参数（`fn f(x: &mut T)`）= 独占可变借用，函数能改，调用方保留所有权
- 函数的**返回值**把所有权交回调用方
- **`Copy` 类型**在函数边界不 move，按值传入后调用方仍可用
- 函数签名是 API 契约：`T` 表示"消费"；`&T` 表示"只读"；`&mut T` 表示"允许修改"

---

## 与其他语言对比

| 语言 | 函数参数的默认行为 |
|-----|---------------|
| Java | 对象引用传递（调用方仍持有引用，函数和调用方"共享"对象）|
| Python | 对象引用传递（同 Java）|
| C++ | 值拷贝（默认）或引用（显式写 `T&`）|
| Go | 值复制（基础类型）或指针（`*T`）|
| Rust | **默认 move**（非 Copy 类型）；引用传递需显式写 `&T` 或 `&mut T` |

Java 程序员转 Rust 的第一大困惑：Java 的"传参"是把引用（指针）复制一份传进去，对象本身没有被移走。Rust 的按值传参是真正的所有权转移。

## 核心概念与心智模型

### 三种函数签名的语义

```
┌───────────────────────────────────────────────────────────┐
│ fn take(x: T)         "我要拥有这个值，你就别用了"        │
│   调用方失去所有权                                         │
│   适合：消费/变换/销毁，不需要归还的场景                    │
└───────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────┐
│ fn borrow(x: &T)      "我只是借来看看，一会还给你"        │
│   调用方保留所有权                                         │
│   适合：只读操作，计算长度、打印、检查等                    │
└───────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────┐
│ fn mutably_borrow(x: &mut T)  "借来改改，改完还给你"      │
│   调用方保留所有权，但函数期间有独占访问权                   │
│   适合：push/pop、排序、追加内容等修改操作                   │
└───────────────────────────────────────────────────────────┘
```

### 函数签名就是 API 合同

```rust
// 合同 1：我要消费你的字符串，你之后就别用了
fn consume(s: String) -> String {
    format!("consumed: {s}")
}

// 合同 2：我只看，你放心用完还能继续用
fn measure(s: &str) -> usize {
    s.len()
}

// 合同 3：我会改它，改完所有权还给你
fn append(s: &mut String, extra: &str) {
    s.push_str(extra);
}
```

## 详细原理

### 1. 按值传入 = Move（非 Copy 类型）

```rust
fn eat_string(s: String) {
    println!("函数内使用: {s}");
    // 函数结束时，s 被 drop
}

fn main() {
    let greeting = String::from("hello");
    eat_string(greeting);   // greeting 的所有权 move 进函数
    // println!("{greeting}"); // ❌ E0382：greeting 已被 move 给函数参数
}
```

### 2. 返回值把所有权交回调用方

```rust
fn make_greeting() -> String {
    String::from("Hello, Rust!")  // 所有权跟着返回值走
}

fn take_and_return(mut s: String) -> String {
    s.push_str(" world");
    s  // 所有权返回给调用方
}

fn main() {
    let s1 = make_greeting();
    println!("{s1}");  // ✅ s1 是新的所有者

    let s2 = String::from("hello");
    let s3 = take_and_return(s2);  // s2 move 进函数，s3 接住返回值
    // println!("{s2}"); // ❌ s2 已被 move
    println!("{s3}");  // ✅ s3 是新的所有者，值是 "hello world"
}
```

### 3. 按引用传 = 借用（推荐的设计）

```rust
fn calculate_length(s: &String) -> usize {
    s.len()
    // s 是借用，函数结束时不 drop 原数据
}

fn main() {
    let s = String::from("hello");
    let len = calculate_length(&s);  // 传引用，不 move
    println!("s = {s}, len = {len}"); // ✅ s 仍然有效！
}
```

### 4. 对比：take-and-give vs 借用

```rust
// 写法 A：先拿走，改完再还（笨重）
fn add_exclamation_v1(mut s: String) -> String {
    s.push('!');
    s  // 必须显式返回，否则 s 在函数里被 drop
}

// 写法 B：借用来改（推荐）
fn add_exclamation_v2(s: &mut String) {
    s.push('!');
    // 不需要返回，调用方自然保留所有权
}

fn main() {
    // 写法 A
    let s1 = String::from("hello");
    let s1 = add_exclamation_v1(s1); // 必须重新绑定
    println!("{s1}");

    // 写法 B
    let mut s2 = String::from("hello");
    add_exclamation_v2(&mut s2); // 原地修改，s2 不变
    println!("{s2}");
}
```

### 5. Copy 类型在函数边界的行为

```rust
fn double(x: i32) -> i32 {
    x * 2  // x 是 i32（Copy），所以函数拿到的是副本，不影响调用方
}

fn main() {
    let n = 21;
    let result = double(n);
    println!("n = {n}");        // ✅ n 仍然有效（Copy 类型不 move）
    println!("result = {result}");
}
```

### 6. 元组返回多个值（早期 Rust 风格）

```rust
fn min_and_max(v: &[i32]) -> (i32, i32) {
    let min = *v.iter().min().unwrap();
    let max = *v.iter().max().unwrap();
    (min, max)
}

let nums = [3, 1, 4, 1, 5, 9, 2, 6];
let (min, max) = min_and_max(&nums);
println!("min={min}, max={max}");
```

## 完整运行示例

```rust
// 工具函数：演示不同函数签名的语义
fn takes_ownership(v: Vec<i32>) {
    // v 在这里被消费，函数结束时 drop
    println!("  [takes_ownership] 接收到 {v:?}，即将释放");
}

fn gives_ownership() -> Vec<i32> {
    // 工厂函数：创建并返回
    vec![4, 5, 6]
}

fn takes_and_gives_back(mut v: Vec<i32>) -> Vec<i32> {
    v.push(99);
    v  // 所有权归还
}

fn borrows_read_only(v: &[i32]) -> i32 {
    v.iter().sum()
}

fn borrows_mutably(v: &mut Vec<i32>, item: i32) {
    v.push(item);
}

fn works_with_copy(n: i32) -> i32 {
    n * 2
}

fn main() {
    println!("=== 1. 按值传入（Move）===");
    let v1 = vec![1, 2, 3];
    takes_ownership(v1);
    // println!("{v1:?}"); // ← 取消注释 → E0382

    println!();

    println!("=== 2. 工厂函数（返回值带着所有权）===");
    let v2 = gives_ownership();
    println!("  v2 = {v2:?}（从工厂函数得到所有权）");

    println!();

    println!("=== 3. take-and-give-back ===");
    let v3 = takes_and_gives_back(v2);  // v2 move 进函数，v3 接住返回
    // println!("{v2:?}"); // ← v2 已被 move
    println!("  v3 = {v3:?}");

    println!();

    println!("=== 4. 借用（推荐：调用后仍可用）===");
    let v4 = vec![10, 20, 30];
    let sum = borrows_read_only(&v4);
    println!("  sum = {sum}");
    println!("  v4 仍然有效: {v4:?}");  // ✅

    println!();

    println!("=== 5. 可变借用（修改但不 move）===");
    let mut v5 = vec![1, 2, 3];
    borrows_mutably(&mut v5, 100);
    println!("  v5 追加后: {v5:?}");  // ✅ v5 仍然是所有者

    println!();

    println!("=== 6. Copy 类型在函数边界 ===");
    let n = 21;
    let doubled = works_with_copy(n);
    println!("  n = {n}（仍然有效）, doubled = {doubled}");
}
```

## 编译器错误分析

### ❌ E0382：函数调用后继续使用 moved 变量

```rust
fn consume(v: Vec<i32>) { println!("{v:?}"); }

let data = vec![1, 2, 3];
consume(data);
println!("{data:?}"); // ❌
```

```text
error[E0382]: borrow of moved value: `data`
  |
4 | consume(data);
  |         ---- value moved here
5 | println!("{data:?}");
  |            ^^^^ value borrowed here after move
```

**修复方案**：

```rust
// A. 函数签名改成借用（通常是最好的选择）
fn consume(v: &[i32]) { println!("{v:?}"); }
// 现在 consume(data); println!("{data:?}"); 都合法

// B. 传 clone（有拷贝成本）
consume(data.clone());
println!("{data:?}");

// C. 不再需要原值的话，就直接让它 move 进去
consume(data);
// 不再使用 data 就没问题
```

## 实际工程场景

### 1. 链式 Builder API：按值传参让链式调用自然

```rust
#[derive(Debug)]
struct QueryBuilder {
    table: String,
    conditions: Vec<String>,
}

impl QueryBuilder {
    fn new(table: impl Into<String>) -> Self {
        Self { table: table.into(), conditions: vec![] }
    }

    // 按值接收 self，改完返回 Self——链式调用的基础
    fn where_clause(mut self, cond: impl Into<String>) -> Self {
        self.conditions.push(cond.into());
        self  // 返回 self
    }

    fn build(self) -> String {
        if self.conditions.is_empty() {
            format!("SELECT * FROM {}", self.table)
        } else {
            format!("SELECT * FROM {} WHERE {}", self.table, self.conditions.join(" AND "))
        }
    }
}

let sql = QueryBuilder::new("users")
    .where_clause("age > 18")
    .where_clause("active = 1")
    .build();
println!("{sql}");
// SELECT * FROM users WHERE age > 18 AND active = 1
```

### 2. 函数不应该要求比实际需要更强的所有权

```rust
// ❌ 不好：要求拥有 String，但实际只需要读
fn print_name(name: String) {
    println!("Name: {name}");
}
// 调用方的 name 被消费，即使只是打印

// ✅ 好：只需要只读访问，接受最通用的 &str
fn print_name(name: &str) {
    println!("Name: {name}");
}
// 调用方可以传 String, &str, &String 等，且调用后原值仍可用
```

### 3. 实现 Iterator 的 `next(self)` 和 `by_ref()`

```rust
let numbers = vec![1, 2, 3, 4, 5];

// into_iter() 消费 Vec，产生所有权迭代器
for n in numbers.into_iter() {
    println!("{n}");
}
// numbers 被消费，不能再用

// iter() 借用，产生引用迭代器
let numbers = vec![1, 2, 3, 4, 5];
for n in numbers.iter() {
    println!("{n}"); // n 是 &i32
}
println!("numbers 仍然有效: {numbers:?}"); // ✅
```

## 性能影响

| 传参方式 | 调用成本 | 函数内成本 |
|---------|--------|---------|
| `fn f(x: T)` | Move 栈上数据（通常 8-24 字节） | 全权掌控，可 in-place 修改 |
| `fn f(x: &T)` | 复制一个指针（8 字节） | 只读，不能修改 |
| `fn f(x: &mut T)` | 复制一个指针（8 字节） | 可独占写，不能传给其他线程 |

所有方式都**不复制堆数据**，代价都很小。选择哪种主要根据语义需求，不是性能需求。

## 注意点与陷阱

### 陷阱 1：`for x in vec` 会 move 整个 Vec

```rust
let words = vec!["hello", "world"];
for w in words { // ← words 被 move 进 for 循环（into_iter()）
    println!("{w}");
}
// println!("{words:?}"); // ❌ words 已被消费

// 用 &words 保留所有权：
let words = vec!["hello", "world"];
for w in &words { // ← 借用，不 move
    println!("{w}");
}
println!("{words:?}"); // ✅
```

### 陷阱 2：`collect()` 后忘记绑定

```rust
let v = vec![1, 2, 3, 4, 5];
v.into_iter().filter(|&x| x > 2).collect::<Vec<_>>();
// 结果被丢弃了！因为没有绑定到变量

let big: Vec<_> = v.into_iter().filter(|&x| x > 2).collect();
// 现在 v 被 move 进 into_iter，big 持有结果
```

## 我的理解与记忆方法

**函数签名的语义速记**：

```
fn f(x: T)        → 我要这个值，你别用了（消费）
fn f(x: &T)       → 我只借看看（只读）
fn f(x: &mut T)   → 我来帮你改（改完还你）
fn f(x: T) -> T   → 我改完帮你改好（take-and-give，笨方法，通常 &mut 更好）
```

**读 Rust 函数签名的习惯**：

第一眼看函数签名，先看参数是 `T`、`&T` 还是 `&mut T`——这决定了调用方能不能继续使用被传入的值。
